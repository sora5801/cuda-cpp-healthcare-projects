This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which
is a hard requirement for every project (CLAUDE.md §8). It is **synthetic** and
**not** real structural data — it must never be read as a real binding affinity.

### What the sample represents

A single protein–ligand **complex** plus a short **MD trajectory** of it:

- a **rigid receptor** of `R` atoms (a crude charged "pocket": a line of
  alternating partial charges along the x-axis), held fixed across all frames;
- `S` **snapshots** of an `L`-atom **ligand**, where the ligand is displaced a
  little further along **+z** in each successive frame.

The geometry is engineered to be **interpretable** (see `docs/PATTERNS.md §6`):
because the ligand drifts out of the pocket frame by frame, the per-snapshot
binding energy ΔG **climbs toward zero** as the complex "unbinds". You can see
that trend directly in the demo output — a built-in sanity check on the physics.

### File format

```
line 1 (header):   R  L  S  minus_TdS
next R lines:      receptor atoms     x y z   q   sigma eps   born
next S*L lines:    ligand atoms, grouped by snapshot (frame 0's L atoms first,
                   then frame 1's L atoms, ...)
```

`#` begins a comment (to end of line); blank lines are ignored. Per-atom fields:

| Field | Meaning | Units |
|---|---|---|
| `x y z` | atom position | ångström (Å) |
| `q` | partial charge | elementary charge `e` |
| `sigma` | Lennard-Jones distance parameter (where the 12-6 potential is 0) | Å |
| `eps` | Lennard-Jones well depth | kcal/mol |
| `born` | effective Generalized-Born radius (larger ⇒ more buried) | Å |

`minus_TdS` is the single constant entropy penalty `−T·ΔS` (kcal/mol) added to
every snapshot's ΔG. See `../THEORY.md` for why a constant is used here instead
of a per-frame normal-mode estimate.

Regenerate (or make a larger synthetic problem) with:

```
python scripts/make_synthetic.py --snapshots 64
```

## Full dataset

Real MM-GBSA inputs are **MD trajectories of protein–ligand complexes** plus a
matching force field (charges, LJ parameters, Born radii). `scripts/download_data.*`
prints pointers; it never bypasses any registration.

- **PDBbind** — curated protein–ligand complexes with measured affinities:
  <http://www.pdbbind.org.cn> (registration required; respect its license).
- **CASF-2016** — the PDBbind "core set" scoring benchmark:
  <http://www.pdbbind.org.cn/casf.php>.
- **ChEMBL** — bioactivity data for building rescoring benchmarks:
  <https://www.ebi.ac.uk/chembl/>.
- **AMBER MM-GBSA tutorials** — ready-to-run trajectories + `MMPBSA.py` inputs:
  <https://ambermd.org/tutorials/>.

To use a real trajectory you would: (1) run/obtain an MD trajectory of the
complex (e.g. `pmemd.cuda`), (2) extract per-frame coordinates and the topology's
charges/LJ/Born parameters, and (3) flatten them into the format above (one
receptor block + one ligand block per frame). Real workflows (`MMPBSA.py`,
`gmx_MMPBSA`) automate this from AMBER/GROMACS topologies.

## Provenance & honesty

The committed sample is **synthetic and labeled synthetic** everywhere it
appears (CLAUDE.md §8). Its numbers are **not** experimentally measured binding
free energies and carry **no clinical or predictive validity** — they exist only
to exercise the CPU/GPU code path and to teach the MM-GBSA pipeline.
