### Layout

```
# ... comment lines (ignored) ...
n_train n_generate seed          <- header: three integers
<SMILES line 1>                  <- n_train training strings follow
<SMILES line 2>
...
```

- **`n_train`** — number of training SMILES strings that follow (here 16).
- **`n_generate`** — how many novel molecules the demo samples on CPU and GPU
  (here 4096). Raise it (e.g. `--n-generate 65536`) for a larger parallel run.
- **`seed`** — base RNG seed; molecule *i* is generated from
  `rng_seed(seed, i)`, so the whole run is reproducible and CPU/GPU agree
  bit-for-bit.
- Each **training SMILES** uses only the project's 13-character alphabet
  (`C O N c n ( ) = 1 2 3 4 #`). Characters outside it are simply skipped during
  training. These toy strings are chosen to make a first-order Markov model
  produce a healthy mix of chain- and ring-like outputs; **read no chemistry
  into the exact strings.**

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8).

## Regenerating / enlarging the sample

```bash
python scripts/make_synthetic.py                       # the committed sample
python scripts/make_synthetic.py --n-generate 65536    # a bigger parallel run
python scripts/make_synthetic.py --seed 7              # a different random seed
```

## Real datasets (not committed — see `scripts/download_data.*`)

This is a **reduced-scope teaching version** (a first-order Markov language
model). Production de-novo design trains deep generative nets on large public
corpora. The catalog names these; none are redistributed here, and several are
license-restricted:

| Dataset | What it is | Access |
|---|---|---|
| **ChEMBL** | 2M+ bioactive molecules | <https://www.ebi.ac.uk/chembl/> — CC-BY-SA 3.0 (attribution + share-alike) |
| **ZINC20** | 1.4B purchasable compounds | <https://zinc20.docking.org> — free for academic use |
| **MOSES** | molecular-generation benchmark | <https://github.com/molecularsets/moses> — MIT |
| **GuacaMol** | distribution-learning & goal-directed benchmarks | <https://github.com/BenevolentAI/guacamol> — MIT |

`scripts/download_data.ps1` / `.sh` print these links and instructions; they do
**not** download anything automatically (the full corpora are large and the
teaching demo does not need them — the synthetic sample is sufficient).

## Honesty note

The scoring function in `src/generator.h` is a **toy integer drug-likeness
proxy**, not a validated medicinal-chemistry score. Real pipelines use QED, the
SA (synthetic-accessibility) score, and docking. See `THEORY.md` §"Where this
sits in the real world".
