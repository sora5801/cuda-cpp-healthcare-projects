This tiny config lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **not** a molecular dataset —
it is the parameter set for a deliberately simple 1-D model (a double-well
potential) on which the *GaMD algorithm itself* (boost + cumulant reweighting) is
demonstrated. This is the reduced-scope teaching choice (CLAUDE.md §13); the full
method runs on all-atom systems (next section).

### Field layout (read in this exact order by `load_config`)

The single whitespace-separated record holds 15 fields:

```
u_barrier  kT  gamma_fric  dt  steps  equil_steps
e_threshold  v_min  v_max  k0
n_walkers  x_lo  x_hi  n_bins  seed
```

| Field | Meaning | Units | Sample |
|---|---|---|---|
| `u_barrier` | double-well barrier height, `U(x)=u_barrier*(x²−1)²` | kT | 3 |
| `kT` | thermal energy `k_B·T` (sets equilibrium populations) | energy | 1 |
| `gamma_fric` | overdamped Langevin friction (drag) | — | 1 |
| `dt` | Langevin timestep | model time | 0.005 |
| `steps` | timesteps per walker | — | 8000 |
| `equil_steps` | leading steps NOT tallied (forget the start) | — | 2000 |
| `e_threshold` | GaMD boost energy ceiling `E` (boost ON when `U<E`) | kT | 3 |
| `v_min` | lowest potential on the grid (for the force constant) | kT | 0 |
| `v_max` | highest relevant potential (= `e_threshold` here) | kT | 3 |
| `k0` | dimensionless boost force-constant knob, `0<k0≤1` | — | 0.15 |
| `n_walkers` | independent walkers (= GPU threads) | — | 512 |
| `x_lo`, `x_hi` | PMF histogram range on `x` | — | −2, 2 |
| `n_bins` | number of PMF histogram bins | — | 40 |
| `seed` | base RNG seed (per-walker streams derived from it) | — | 12345 |

All numbers are **dimensionless / in units of kT** — this is a model system, not a
physical molecule, so there are no Ångströms or kcal/mol. Nothing here is clinical
or patient-derived.

## Full / real datasets (for a production GaMD study)

Real GaMD does **not** read a config like this — it boosts a full molecular-
mechanics force field and reads standard MD inputs. To study the real method:

- **AMBER GaMD tutorials** — https://www.med.unc.edu/pharm/miaolab/resources/gamd/
  (Miao lab): worked GaMD runs with `pmemd.cuda`, including the input keywords
  (`igamd`, `iE`, `sigma0P/sigma0D`) that map onto this project's `E`/`k0`.
- **GPCRmd** — https://gpcrmd.org : curated GPCR MD systems (prmtop/inpcrd, PDB) —
  classic GaMD drug-target targets.
- **D. E. Shaw Research** long-timescale benchmark trajectories (folding,
  binding) — useful as ground truth for what enhanced sampling tries to reach.
- **RCSB PDB** — https://www.rcsb.org : starting structures of drug targets to
  build your own system with `tleap`.

`scripts/download_data.ps1` / `.sh` print these links and instructions; they do
**not** download or bypass any registration (some systems require accounts). The
committed synthetic config is always enough to build and run the demo.

## Honesty / safety

- The committed sample is **synthetic** and labeled synthetic everywhere.
- The model potential is a textbook stand-in for a two-state conformational change;
  it makes **no** claim about any specific molecule and is **not for clinical use**.
- No patient data is involved.
