### File format

Whitespace-separated text, parsed by `load_trajectory()` in `src/reference_cpu.cpp`:

```
<n_frames> <n_atoms> <ref_index>      # header line
x y z                                 # frame 0, atom 0
x y z                                 # frame 0, atom 1
...                                   # n_atoms lines per frame
x y z                                 # frame 1, atom 0
...                                   # n_frames blocks total
```

- `n_atoms` **must equal** the compile-time `N_ATOMS` (16) in `src/rmsd_core.h`;
  the loader throws if they differ (the layout is fixed so a frame maps cleanly to
  one GPU thread and the inner loops unroll).
- `ref_index` selects the reference frame for RMSD and native contacts (we use 0).
- Coordinates are written with full `repr()` precision so the C++ loader parses the
  **exact** same doubles every run → byte-identical, reproducible output.

### What the synthetic data represents (and what it does NOT)

Frame 0 is a compact **helix** (16 atoms on a tight spiral). Successive frames
progressively **unfold** the helix toward a straight chain, driven by a fixed
three-state schedule so the trajectory dwells in a **folded**, an **intermediate**,
and an **unfolded** basin. Consequently:

- the optimal-superposition **RMSD** rises from `0` (frame 0) to `~10.8`, and
- the **native-contact fraction Q** decays `1.0 → 0.52 → 0.0`, and
- the **RMSD clustering** histogram shows three populated shells (the three states).

> **This data is SYNTHETIC.** It is a geometric construction with **no physical or
> biological meaning** beyond the helix-to-coil morph we impose to make the metrics
> interpretable. It is **not** a real protein, ligand, or simulation. No clinical or
> scientific conclusion may be drawn from it (CLAUDE.md §8). Units are arbitrary
> (Angstrom-like); the contact cutoff is 8.0 of those units.

## Full dataset

Real molecular-dynamics trajectories you can analyze with the same pipeline (after
adapting the loader to the real file format and atom count):

- **MDCATH** — curated all-atom trajectories: <https://huggingface.co/datasets/compsciencelab/mdcath>
- **GPCRmd** — GPCR molecular-dynamics database: <https://gpcrmd.org>
- **MDDB** — the molecular-dynamics database: <https://www.mddbr.eu>
- **PDB trajectory depositions** — e.g. via RCSB / PDB-Dev.

`scripts/download_data.ps1` / `.sh` print the source URLs and the idempotent fetch
pattern. They **never** bypass any registration or credentials; where an account is
required they print instructions only, and `make_synthetic.py` provides the offline
stand-in used by the demo.
