### File format

```
M  tol  max_iter            # M systems; CG relative-residual tol; CG iteration cap
<repeated M times:>
  n                         # number of atoms in this system (1..32)
  x y z  Ex Ey Ez  alpha    # one line per atom (see field meanings below)
```

| Field | Meaning | Units |
|---|---|---|
| `x y z` | atom position | Ångström |
| `Ex Ey Ez` | **permanent** electric field at the atom (the right-hand side `b`) | reduced field units |
| `alpha` | atomic (isotropic) polarizability | Ångström³ |

The induced dipoles `mu_i` we solve for satisfy
`mu_i = alpha_i (E_i^perm + sum_{j≠i} T_ij · mu_j)` — see [`../THEORY.md`](../THEORY.md).

### What the sample encodes (so the result is interpretable)

Each of the 8 systems is a 3-atom trio: a central polarizable site at the origin
and two partners at `(±d, 0, 0)`. The half-separation `d` is swept from **4.0 Å
down to 2.0 Å** across the ensemble. As the partners approach, the dipole–dipole
coupling (which scales like `1/d³`) strengthens, so the induced dipoles and the
(negative, stabilizing) polarization energy **grow monotonically** — the headline
physics of a polarizable force field. The demo's per-member table makes this
trend visible directly.

The committed file is written at full floating-point precision so a file-driven
run reproduces the in-memory builder (`make_synthetic_ensemble` in
`src/reference_cpu.cpp`) bit-for-bit. Regenerate or resize it with:

```bash
python scripts/make_synthetic.py --members 8     # default; writes data/sample/
```

## Full / real dataset

There is no single "download and run" dataset for this teaching model; the real
ingredients of a production AMOEBA run are **force-field parameter files** plus
**molecular geometries**. `scripts/download_data.ps1` / `.sh` print pointers to:

- **AMOEBA / AMOEBA+ parameters** — Tinker `.prm`/`.key` files:
  <https://github.com/TinkerTools/tinker> (see `params/`) and
  <https://github.com/TinkerTools/poltype2> (AMOEBA+ parameterization).
- **NIST thermophysical properties** — water dielectric/dipole benchmarks for
  validating a polarizable water model: <https://webbook.nist.gov>.
- **BindingDB** experimental affinities (for AMOEBA-FEP validation):
  <https://www.bindingdb.org>.

These are large and/or license-restricted, so we **do not redistribute** them.
Respect each source's license. The scripts never bypass credentials — they print
instructions and links only (CLAUDE.md §8).

## Honesty note

The sample is **synthetic** and labeled synthetic everywhere it appears. The
polarizabilities, field magnitude, and geometry are chosen for didactic clarity,
not chemical accuracy. Nothing here is validated for any real-world or clinical
use.
