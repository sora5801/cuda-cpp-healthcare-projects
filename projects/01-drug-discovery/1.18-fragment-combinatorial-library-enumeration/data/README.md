### File format

```
# comment lines start with '#'
<N_SLOTS>                                  # must equal the compiled N_SLOTS (= 3)
SLOT 0 <size0>                             # header announcing slot 0
<name> <MW> <cLogP> <TPSA> <HBD> <HBA>     # size0 synthon rows
...
SLOT 1 <size1>
...
SLOT 2 <size2>
...
```

Each **synthon** (building block) carries five **additive group-contribution
descriptors** (the order matches `DescIndex` in `src/product_core.h`):

| Field | Meaning | Units |
|---|---|---|
| `MW`    | molecular-weight contribution | g/mol |
| `cLogP` | Crippen octanol–water log-partition contribution | dimensionless |
| `TPSA`  | Ertl topological polar surface area contribution | Å² |
| `HBD`   | hydrogen-bond **donor** count | integer |
| `HBA`   | hydrogen-bond **acceptor** count | integer |

A **product** is one synthon per slot; its descriptors are the **sum** of its
synthons' contributions (additivity — see `THEORY.md`). The library is the
Cartesian product of the slots, so `216 = 6 × 6 × 6` products here.

The loader (`src/reference_cpu.cpp::load_synthons`) rejects a file whose
`N_SLOTS` differs from the compiled constant.

## Full dataset

Real synthon descriptors are computed with **RDKit** from building-block SMILES:

- **Enamine building blocks** — <https://enamine.net/building-blocks>
- **Enamine REAL Space** — multi-billion make-on-demand library — <https://enamine.net>
- **ChemSpace** — commercial building blocks — <https://chem-space.com>
- **Sigma-Aldrich** building-block catalog (search the vendor site for the
  current building-block listing; the historical URL has moved).

`scripts/download_data.ps1` / `.sh` print the RDKit recipe (SMILES → `Descriptors.MolWt`,
`Crippen.MolLogP`, `rdMolDescriptors.CalcTPSA`, `NumHDonors`, `NumHAcceptors` →
this row format). For a larger synthetic library instead:

```
python scripts/make_synthetic.py --per-slot 40     # 40^3 = 64 000 products
```
