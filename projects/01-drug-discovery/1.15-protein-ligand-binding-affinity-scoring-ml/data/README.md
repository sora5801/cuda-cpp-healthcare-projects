## Committed sample (`sample/complexes_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/complexes_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, seed 11) |
| License | Public domain (CC0) — it is synthetic |
| Size | 6 complexes, ~200 atoms total (~4 KB) |
| Why synthetic | The real benchmarks (PDBbind, CASF) require registration and have redistribution terms. A synthetic batch keeps the demo offline, reproducible, and license-free. |

The committed sample is **chemically meaningless**. It exists only to exercise and
**verify** the 3D-CNN forward pass (CPU vs GPU). It lets `demo/run_demo` run
**offline, with zero downloads** — a hard requirement (CLAUDE.md §8).

### File format

```
<n>                                  # number of complexes (poses), e.g. 6
# then, repeated n times:
<m> <label_pKd>                      # m = atom count of this complex; label is a TOY pKd
<x> <y> <z> <type> <is_ligand>       # m atom lines follow
...
```

Per-field meaning:

| Field | Meaning | Units / range |
|---|---|---|
| `n` | number of complexes in the batch | integer ≥ 1 |
| `m` | atoms in this complex | integer ≥ 1 |
| `label_pKd` | **synthetic** "true" affinity (interpretability only; the model never sees it) | pKd, ~[2, 11] |
| `x y z` | atom position | angstroms, inside the `[0, 16)` grid box |
| `type` | element index | `0=C 1=N 2=O 3=S` |
| `is_ligand` | which molecule the atom belongs to | `0` = protein, `1` = ligand |

The loader (`src/reference_cpu.cpp::load_complexes`) rejects out-of-range `type` or
`is_ligand` and any truncated record, so a malformed file fails loudly.

To regenerate (or make a large "rescore at scale" batch):

```
python scripts/make_synthetic.py                 # default n=6
python scripts/make_synthetic.py --n 100000      # stress the GPU batch path
```

## Full dataset

Real ML scoring functions are trained and benchmarked on measured complexes. None
are auto-downloaded — they require registration and/or have redistribution terms.
`scripts/download_data.ps1` / `.sh` print the links and instructions:

- **PDBbind v2020** — 19,443 protein-ligand complexes with measured `Kd`/`Ki`
  (the standard training set): <http://www.pdbbind.org.cn> (registration required).
- **CASF-2016** — the community scoring/ranking/docking benchmark:
  <http://www.pdbbind.org.cn/casf.php>.
- **ChEMBL** — bioactivity database: <https://www.ebi.ac.uk/chembl/>.
- **BindingDB** — 2.8M measured binding affinities: <https://www.bindingdb.org>.

To turn a real complex (a `.pdb` protein + a `.sdf`/`.mol2` ligand) into this
format you would parse atom coordinates and elements (e.g. with **RDKit** or
**Biopython**), center the box on the binding pocket, and emit the `x y z type
is_ligand` rows. The download script documents this recipe.

## Provenance & honesty

The committed sample is **synthetic** and labeled as such everywhere. The predicted
pKd values carry **no chemical meaning** (the network is untrained — see THEORY §7),
and the `label_pKd` column is a toy value the model never uses. No clinical or
chemical conclusion may be drawn from this data.
