This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is **not real chemistry data**
and must never be presented as such. It encodes a *didactic* covalent-docking
problem engineered so the energy landscape is smooth and has a single clear,
favorable (negative-energy) minimum.

### What the synthetic system is

A reactive **warhead** atom (the *anchor*) already covalently bonded to a
cysteine **S-gamma** sulfur, with a short flexible ligand chain (3 rotatable
torsions → 4 atoms) hanging off it, sitting in a rigid pocket of 6 protein atoms.
The program searches all `36³ = 46 656` torsion combinations for the lowest-
energy pose. See `THEORY.md` for the science and math.

### Field order (matches `load_problem()` in `src/reference_cpu.cpp`)

| Group | Fields | Units / meaning |
|---|---|---|
| anchor | `x y z` | warhead carbon position (Å); bonded to the cysteine sulfur |
| S-gamma | `x y z` | cysteine sulfur position (Å); the covalent bond partner |
| covalent | `bond_len_ideal angle_ideal k_bond k_angle` | ideal C–S length (Å), ideal Sγ–anchor–firstatom angle (rad), two harmonic force constants |
| ligand chain | `seg_len bond_angle first_dir.x first_dir.y first_dir.z` | bond length (Å), valence angle (rad), unit direction of segment 0 |
| ligand nonbonded | `lig_sigma lig_epsilon lig_charge` | LJ σ (Å), LJ ε (kcal/mol), partial charge (e) |
| pocket × 6 | `x y z sigma epsilon charge` | each fixed protein atom: position (Å), LJ σ (Å), LJ ε (kcal/mol), charge (e) |

The pocket atom positions are **not arbitrary**: each is placed ≈ 3.82 Å (the
Lennard-Jones energy minimum, `2^(1/6)·σ`) from the closest point the flexible
ligand tip can reach, so every pocket atom is attractive but **never clashes** —
which keeps the energy finite and the GPU/CPU agreement at machine precision (see
`THEORY.md` §Numerical considerations).

### Regenerate it

```bash
python scripts/make_synthetic.py        # writes data/sample/covalent_sample.txt
```

The generator and `src/main.cu`'s built-in fallback encode the **same** system,
so the program prints the same docked pose with or without the file.

## Full / real datasets

No real dataset is required to build, run, or learn from this project. Covalent
docking in practice consumes real protein–ligand structures and curated covalent
inhibitor sets. `scripts/download_data.{ps1,sh}` prints links to these and never
bypasses any registration. The catalog notes (verbatim):

> CovDocker benchmark (2025, verify URL); ChEMBL covalent inhibitor set
> (https://www.ebi.ac.uk/chembl/); PDB covalent complex structures
> (https://www.rcsb.org); BindingDB covalent entries (https://www.bindingdb.org).

Each source has its **own license** — respect it. Many require registration and
forbid redistribution, which is exactly why the committed sample is synthetic.

## Safety

Educational only. **Not for clinical use.** Nothing here is a validated force
field or a real binding affinity; the energies are illustrative, not predictive.
