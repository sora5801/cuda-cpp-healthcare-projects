# Data — 1.2 Particle-Mesh Ewald Electrostatics

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/charges_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 4 KB (64 ions) |
| Layout | line 1: `<n> <box>`; then `n` rows of `x y z q` |

The sample is a **synthetic NaCl-like ionic crystal**: a 4×4×4 simple-cubic
lattice of 64 point charges in a periodic cubic box of side 8.0 (reduced units),
with charges alternating +1 / −1 in the rock-salt (checkerboard) pattern. It is
**charge-neutral**, which PME requires (a net charge in a periodic box has an
ill-defined, infinite Coulomb energy).

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is engineered to be
*interpretable*: the alternating-charge lattice has a well-known electrostatic
(Madelung) energy, the demo recovers a stable total energy of ≈ −27.96 reduced
units, and that energy is shown to be **invariant to the Ewald splitting
parameter β** — the textbook self-consistency check of the method.

### Field meanings

| Token | Meaning | Units |
|---|---|---|
| `n` | number of point charges | — |
| `box` | cubic periodic box side length | reduced length unit |
| `x y z` | Cartesian position of a charge (wrapped into `[0, box)`) | same length unit |
| `q` | charge | elementary charge `e` |

> Units are **reduced**: Coulomb's constant `1/(4πε₀)` is set to 1, so energies
> come out in a self-consistent reduced unit (THEORY explains the kcal/mol factor
> a production MD code would apply). **Synthetic, educational — not clinical.**

Regenerate or resize the sample:

```bash
python scripts/make_synthetic.py --reps 4 --box 8.0      # the committed 64-ion sample
python scripts/make_synthetic.py --reps 8 --box 16.0     # a larger 512-ion system
```

## Full dataset

There is no single "official" PME dataset; the method runs on any **periodic
molecular system with per-atom partial charges**. Realistic benchmark systems
(from the catalog) and how to get them are printed by
`scripts/download_data.ps1` / `.sh`:

- **CHARMM-GUI Archive** — pre-built solvated protein–water boxes (PSF/PDB).
  <https://charmm-gui.org/?doc=archive> *(free account required)*.
- **MemProtMD** — membrane-protein systems in periodic boxes.
  <https://memprotmd.bioch.ox.ac.uk/>.
- **D. E. Shaw Research Anton trajectories** — ms-scale MD archives, available by
  request from DE Shaw Research.

**Licensing:** these require registration and/or are not redistributable, so the
committed sample is synthetic by design (CLAUDE.md §8). The download scripts
**only print instructions** — they never bypass credentials. A production loader
would read charges, coordinates, and the periodic box vectors out of PSF/PDB/DCD
files; our `load_system` uses the plain text format above to keep the focus on
the PME algorithm rather than file parsing.
