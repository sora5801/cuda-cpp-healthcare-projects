# Data — 1.17 Markov State Models from MD

## Committed sample (`sample/trajectory_sample.txt`)

| Field | Value |
|---|---|
| Origin | **Synthetic** featurized MD trajectory (`scripts/make_synthetic.py`, seed 7) |
| License | Public domain (CC0) — synthetic, no physical/clinical meaning |
| Contents | 12,000 frames × 3 features, generated by a hidden 3-basin Markov chain |
| Size | ~290 KB |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**
(CLAUDE.md §8).

### File format

```
<N> <D> <K> <lag>        # frames, features/frame, microstates to find, MSM lag (frames)
<frame 0: D floats>      # one row per frame, time-ordered, values normalized to [0,1]
<frame 1: ...>
... (N rows)
```

The rows are a **single continuous trajectory** (frame `t` precedes frame `t+1`
in time), which is what lets us count `s(t) -> s(t+lag)` transitions. Each frame
is a `D`-dimensional feature vector — a stand-in for tICA / collective-variable
coordinates of a real MD run. Values are normalized to **[0,1]** so the
fixed-point centroid accumulation in `msm.h` is exact.

### How the sample is engineered (so the answer is checkable)

`make_synthetic.py` runs a hidden **3-state Markov chain** with a known, sticky
transition matrix and emits a Gaussian "blob" around each basin center:

```
basin 0  center (0.20, 0.20, 0.25)     ground-truth T (row=from):
basin 1  center (0.80, 0.30, 0.75)       0.94 0.05 0.01
basin 2  center (0.50, 0.85, 0.40)       0.04 0.93 0.03
                                          0.02 0.04 0.94
```

Because the basins are well separated, k-means recovers one microstate per basin
and the MSM's estimated transition matrix should **approximate** the ground-truth
matrix above. The high self-transition probabilities create a clear **slow
process**, whose implied timescale the demo reports.

## Full dataset

Real MSMs are built from long or many-short MD trajectories, then featurized and
reduced (tICA/PCA) before clustering:

- **mdCATH** — 5 µs MD for 272 proteins
  (<https://huggingface.co/datasets/compsciencelab/mdcath>).
- **Fast-folder benchmark trajectories** (chignolin, Trp-cage, Villin — shared by
  Piana/Shaw).
- **GPCRmd** (<https://gpcrmd.org>) — curated GPCR MD.
- **D. E. Shaw** millisecond trajectories (via RCSB deposition).

Featurize and tICA-reduce a trajectory (e.g. with **PyEMMA** /
**deeptime**), scale a few leading components to `[0,1]`, and write them into the
format above (one row per frame, in time order). `scripts/download_data.*` prints
these pointers; it never bypasses credentials.

Bigger synthetic trajectory (no download):
`python scripts/make_synthetic.py --frames 50000`.

## Provenance & honesty

The sample is a **synthetic** hidden-Markov trajectory, not real MD, and carries
no physical or clinical meaning. It exists to make the MSM result interpretable
(the 3 basins and their slow interconversion are recovered) and the GPU/CPU
comparison verifiable.
