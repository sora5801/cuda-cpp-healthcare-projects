# Data — 1.33 Interaction Fingerprinting & Binding-Mode Clustering

## Committed sample (`sample/ifp_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/ifp_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, seed 7) |
| License | Public domain (CC0) — it is synthetic, no real protein/ligand |
| Size | ~4 KB |
| Contents | A 24-residue model pocket + 120 ligand poses drawn from **4 planted binding modes** (30 poses each) |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is engineered so the answer is
*knowable*: the four modes sit in four corners of the pocket, leave four distinct
interaction fingerprints, and the demo recovers them at **100% purity** — so you
can confirm the pipeline works, not just that it ran (PATTERNS.md §6).

> **SYNTHETIC — not a real complex.** No coordinates here come from a deposited
> structure. Nothing in this project may inform a real binding-mode call.

### File format (parsed by `load_dataset()` in `src/reference_cpu.cpp`)

```
line 1            : P K                                  # P poses, K clusters
next 24 lines     : x y z can_hbond can_aromatic can_ionic   # one per residue
next P lines      : x y z has_donor has_aromatic has_charge true_mode
```

Per-field meaning:

| Column | Meaning | Units |
|---|---|---|
| `x y z` | interaction-center coordinates (residue) / representative atom (pose) | Ångström |
| `can_hbond` / `has_donor` | residue / ligand can form a hydrogen bond here | 0 or 1 |
| `can_aromatic` / `has_aromatic` | residue / ligand carries an aromatic ring | 0 or 1 |
| `can_ionic` / `has_charge` | residue / ligand is charged (salt-bridge capable) | 0 or 1 |
| `true_mode` | ground-truth binding mode the pose was generated from | integer (synthetic only) |

`true_mode` is used **only** to report how well clustering rediscovered the
planted modes (the `mode recovery` line). The clustering algorithm never reads it.

Regenerate or enlarge the sample:

```bash
python scripts/make_synthetic.py                 # default 120-pose sample
python scripts/make_synthetic.py --per-mode 500  # 2000 poses (timing studies)
```

## Full / real datasets

Real interaction fingerprints are computed from **docking poses** or **MD
trajectory frames** of actual protein–ligand complexes. `scripts/download_data.*`
prints pointers to the public sources from the catalog; it never bypasses any
registration:

- **PDBbind** — curated protein–ligand complexes with affinities: <http://www.pdbbind.org.cn>
- **KLIFS** — kinase–ligand structures + interaction features: <https://klifs.net>
- **ChEMBL** — bioactivities with structures: <https://www.ebi.ac.uk/chembl/>
- **BindingDB** — measured binding data: <https://www.bindingdb.org>

To go from such structures to the bit-vectors this project clusters, a tool like
**ProLIF** or **ODDT** detects the interactions per frame; you would emit one row
per pose in the format above. That conversion is out of scope here (it needs a
full chemistry toolkit); the synthetic generator stands in so the demo is
self-contained. License note: respect each source's terms — none of the real
coordinates are redistributed in this repo.
