# Data — 1.1 Molecular Dynamics Engine

## Committed sample (`sample/lj_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/lj_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no real structure or patient data |
| Size | < 2 KB (27 atoms) |
| Units | **LJ reduced units** (mass = ε = σ = 1) throughout |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is a deterministic
**Lennard-Jones fluid**: 27 identical particles on a 3×3×3 simple-cubic lattice
inside a periodic cubic box, given small deterministic starting velocities.

### File layout

```
line 1 : n  box  dt  steps  eps  sigma  rcut  mass
n lines: x  y  z  vx  vy  vz          (one atom per line)
```

| Symbol | Meaning | Units (reduced) |
|---|---|---|
| `n` | number of atoms | — |
| `box` | cubic periodic box edge length L | σ |
| `dt` | velocity-Verlet timestep | σ·√(m/ε) |
| `steps` | number of integration steps | — |
| `eps` (ε) | Lennard-Jones well depth | ε (= 1) |
| `sigma` (σ) | LJ zero-crossing distance | σ (= 1) |
| `rcut` | force/energy cutoff radius | σ |
| `mass` (m) | particle mass | m (= 1) |
| `x y z` | atom position | σ |
| `vx vy vz` | atom velocity | σ/time |

The sample uses `n=27, box=3.6, dt=0.004, steps=50, rcut=2.5` — a near-equilibrium
lattice whose total energy is **conserved** by the integrator (the headline result;
see `demo/expected_output.txt`). Regenerate it (or make a larger system) with:

```bash
python scripts/make_synthetic.py            # 27-atom default sample
python scripts/make_synthetic.py --side 8   # 512-atom larger synthetic system
```

The generator uses an integer hash (not a floating-point RNG) for the velocities,
so it produces **byte-identical** atoms on every platform and matches the C++
built-in fallback `make_default_system()`.

## Full / real datasets

This is a **teaching engine for the LJ force field**, so it does not need a real
dataset to run — the synthetic LJ fluid is the intended input. Production
biomolecular MD instead reads force-field parameter sets and starting structures.
The catalog points at the standard ones (study these to see what a real engine
consumes; do not commit them here):

- **CHARMM36m** force field — proteins/lipids/nucleic acids/carbohydrates:
  https://mackerell.umaryland.edu/charmm_ff.shtml
- **AMBER ff19SB** — protein force field with improved backbone torsions:
  https://ambermd.org
- **GPCRmd** — curated GPCR MD trajectories: https://gpcrmd.org
- **MoDEL** — molecular-dynamics extended library of protein simulations:
  https://mmb.irbbarcelona.org/MoDEL/

`scripts/download_data.ps1` / `.sh` print these pointers and **never** bypass any
registration. They defer to `make_synthetic.py` for the offline stand-in.

## Provenance & honesty

The committed sample is **synthetic** and labeled synthetic everywhere it appears.
It models a noble-gas-like LJ fluid, not a biomolecule, and carries **no clinical
meaning** — this project is educational (CLAUDE.md §1, §8).
