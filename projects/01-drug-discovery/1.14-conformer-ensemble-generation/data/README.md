# Data — 1.14 Conformer Ensemble Generation

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/conformer_params.txt` |
| Origin | **Synthetic** (two demo run-parameters; generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB |
| Layout | one line, two whitespace-separated numbers: `<rmsd_threshold_angstrom> <num_representatives_to_print>` |

**Why the sample is so small.** Unlike most projects, the "dataset" here is *not* a molecule — the molecule
is defined at **compile time** in [`src/conformer.h`](../src/conformer.h) (a fixed flexible chain of 8 atoms).
The program enumerates and embeds every conformer internally. The only thing an external file needs to supply
is a couple of **run knobs** for the pruning step:

| Field | Meaning | Units | Default |
|---|---|---|---|
| `rmsd_threshold` | conformers closer than this RMSD are treated as duplicates and pruned | Ångström | `1.0` |
| `num_representatives` | how many lowest-energy cluster representatives to print | count | `5` |

`1.0 Å` is a typical RDKit conformer-deduplication threshold. With the committed value the demo enumerates
243 conformers and prunes them to **81 distinct** shapes.

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is a hard requirement for
every project (CLAUDE.md §8). Regenerate or customize it with:

```bash
python scripts/make_synthetic.py --rmsd 1.0 --top 5
```

## Full / real datasets

This teaching demo needs **no external dataset** — it is self-contained. The real-world datasets below are
what you would validate a *production* conformer generator against (e.g. compare your generated conformers and
their energies to crystallographic / DFT references). They are large and/or license-restricted;
`scripts/download_data.ps1` / `.sh` print pointers and do **not** attempt to bypass any registration.

Catalog dataset notes (verbatim):

> GEOM — 37M conformers of drug-like molecules with DFT energies (https://github.com/learningmatter-mit/geom);
> CSD torsion library (https://www.ccdc.cam.ac.uk); COD (Crystallography Open Database) — crystal structures
> for torsion validation (https://www.crystallography.net); PDB small molecule conformations
> (https://www.rcsb.org).

| Dataset | What it provides | License / access |
|---|---|---|
| **GEOM** | 37M conformers of drug-like molecules with DFT energies | MIT-licensed data release; large download (tens of GB) |
| **CSD torsion library** | experimental torsion-angle preferences (the "ET" in ETKDGv3) | CCDC — **commercial / academic license required** |
| **COD** | open crystal structures for torsion validation | open access |
| **PDB** | small-molecule conformations from deposited structures | open access |

## Provenance, safety & honesty

- The committed `conformer_params.txt` is **synthetic** — it is two demo parameters, not measured data.
- Nothing here is patient-derived; there are no clinical claims. The molecule is a generic synthetic chain.
- The force-field constants in `src/conformer.h` are *illustrative* (physically reasonable, not parameterized
  to reproduce real alkane energetics). Energies are in kcal/mol but are teaching values, not predictions.
