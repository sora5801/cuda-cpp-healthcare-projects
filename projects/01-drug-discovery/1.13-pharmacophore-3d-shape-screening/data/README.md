# Data — 1.13 Pharmacophore & 3D Shape Screening

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/conformers_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic; **NOT real molecules** |
| Size | ~1.6 KB |
| Contents | 1 query molecule + 9 library conformers, heavy-atom 3D coordinates |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which
is a hard requirement for every project (CLAUDE.md §8). Everything in it is
synthetic geometry engineered to make the result interpretable — see "How the
sample is engineered" below.

### File format

Whitespace-separated text; lines beginning with `#` are comments and are
skipped by the loader (`src/reference_cpu.cpp`). Layout:

```
N                       # number of library conformers
<QUERY block>
<library block 0>
...
<library block N-1>
```

Every **block** is:

```
M                       # number of heavy atoms in this molecule (1..MAX_ATOMS=64)
label                   # one whitespace-free token (e.g. QUERY, lib_00_self)
x y z radius            # one line per atom; M such lines
```

### Per-field meaning & units

| Field | Meaning | Units |
|---|---|---|
| `x`, `y`, `z` | atom-center Cartesian coordinates in the molecule's frame | ångström (Å) |
| `radius` | atom van der Waals radius (defines the Gaussian width via `atom_alpha`) | ångström (Å) |
| `label` | human-readable conformer name (used in the report) | — |

The loader converts each `radius` to a Gaussian width parameter `alpha`
(`1/Å²`) at load time using the Grant–Pickett single-Gaussian model (see
`src/shape_overlap.h`), so the hot loop never touches a radius again.

> **Rigid-overlay assumption.** The library conformers are treated as already
> aligned to the query (a rigid overlay). Real shape screening *optimizes* the
> alignment first; that simplification is documented in `THEORY.md` §7.

### How the sample is engineered (why the result is interpretable)

`make_synthetic.py` builds the query as a benzene-like hexagon (6 atoms at the
1.39 Å aromatic C–C distance) plus one substituent atom, then derives library
conformers spanning the whole similarity range so the ranking recovers a
**known answer**:

- `lib_00_self` is an exact copy ⇒ ShapeTanimoto = **1.000000** (built-in check).
- graded perturbations (`jitter`, `shift05`, `shift15`, `rot`, `grow`, `shrink`)
  fall in physically sensible order.
- `lib_07_far` (8 Å away) and `lib_08_line` (a different, linear shape) score
  near zero.

## Full dataset

This teaching project ships only the synthetic sample. To experiment at realistic
scale, point the program at a larger file in the same format, derived from public
3D conformer libraries:

- **ZINC20** — purchasable-compound 3D conformers: <https://zinc20.docking.org>
- **DUD-E** — Directory of Useful Decoys, Enhanced (actives + decoys, with 3D
  conformers): <https://dude.docking.org>
- **Enamine REAL** — make-on-demand library conformers: <https://enamine.net>

**License:** each source has its own terms — respect them. None are redistributed
here. ZINC and DUD-E are freely downloadable for research; Enamine REAL access is
via their site. You must convert their SDF/MOL2 conformers into this project's
simple `x y z radius` text format yourself (a small RDKit script: read 3D coords,
look up each element's van der Waals radius). `scripts/download_data.{ps1,sh}`
print these links and instructions; they do **not** bypass any registration.

For a larger **synthetic** problem with no downloads at all, regenerate the
sample (the generator is deterministic, so output stays reproducible):

```
python scripts/make_synthetic.py
```

> Synthetic data is labeled synthetic everywhere it appears (CLAUDE.md §8). None
> of this is real chemistry, and no output here has any clinical meaning.
