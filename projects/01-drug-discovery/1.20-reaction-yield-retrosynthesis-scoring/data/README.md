### File format

```
# comment lines start with '#' and are ignored
<n> <MAX_STEPS> <NUM_FEATURES>          # e.g. "24 6 4"  (last two MUST match the build)
<w0 w1 ... w{F-1}> <bias>               # the shared logistic model (F weights + 1 bias)
# then n route blocks:
<num_real_steps> <availability>         # availability in [0,1]
<step 0: NUM_FEATURES feature values>   # one line per real step
<step 1: ...>
...
```

Each **step** is a feature vector (all features scaled to roughly `[0,1]`):

| slot | feature | meaning | good when |
|---|---|---|---|
| 0 | `template_prior` | historical reliability of the reaction template | high |
| 1 | `precedent_count` | `log10(1+#literature precedents)`, normalized | high |
| 2 | `condition_penalty` | harshness of required conditions (temp/pressure/catalyst) | **low** |
| 3 | `selectivity` | expected regio/stereo selectivity | high |

The per-step yield is `sigmoid(w·x + b)`; a route's score is the **product** of
its step yields times its building-block `availability` (see `src/route_score.h`
and `THEORY.md`). Steps beyond `num_real_steps` are padded internally with the
`STEP_ABSENT` sentinel and skipped. The loader
(`src/reference_cpu.cpp::load_routes`) rejects a file whose `MAX_STEPS` or
`NUM_FEATURES` differs from the compiled constants.

## Full / real dataset

Real per-step features are derived from **atom-mapped reaction SMILES** with a
learned model. Public sources of reactions:

- **USPTO-50k** — 50k atom-mapped reactions: <https://github.com/connorcoley/rexgen_direct>
- **USPTO-MIT** — 479k reactions: <https://github.com/wengong-jin/nips17-rexgen>
- **Open Reaction Database (ORD)** — open-access reaction data: <https://open-reaction-database.org>
- **Reaxys / CAS** — commercial, license-restricted (do **not** redistribute).

A real pipeline (AiZynthFinder, ASKCOS) would (1) run a retrosynthesis planner to
emit candidate routes, (2) featurize each step with a transformer/GNN yield model,
and (3) score every route — which is exactly the **batched scoring** step this
project parallelizes. `scripts/download_data.ps1` / `.sh` print that recipe and
the links; they download nothing and bypass no credentials (CLAUDE.md §8).

For a planner-scale synthetic batch instead:

```
python scripts/make_synthetic.py --n 1000000
```
