### Why this sample is *interpretable* (a baked-in known answer)

The generator embeds a **recoverable ground truth** (PATTERNS.md §6). It chooses a
small set of "true target" proteins and constructs the drug embedding `h` and the
`TARGETS` relation embedding `r` so that, under the TransE rule `h + r ≈ tail`,
those true targets sit at (near-)zero distance — i.e. they are the highest-scoring
candidates. Every other protein is a random "decoy" placed far away. The demo then
uses the GPU to **recover exactly those targets** at the top of the ranking, so you
can *see* the method work. For this committed sample the ground-truth targets are
proteins `6`, `15`, `62` (with `6` the exact match), and the demo reports
`recovery: 3 / 3 ground-truth targets in top-5`.

### File layout (whitespace text, parsed by `src/reference_cpu.cpp`)

```
line 1            : n  dim                         # n candidate tails, embedding dim
line 2            : head embedding                 # dim floats  (the query drug h)
line 3            : relation embedding             # dim floats  (the TARGETS relation r)
line 4            : n_true idx0 idx1 ...           # ground-truth target indices (0-based)
next n lines      : one candidate tail embedding   # dim floats each, row-major
```

Field meanings:

- **head** `h` — the query drug's d-dimensional TransE embedding (unitless latent
  coordinates; a trained model would learn these from the knowledge graph).
- **relation** `r` — the `TARGETS` relation's translation vector. TransE models a
  fact `(drug, TARGETS, protein)` as `h + r ≈ tail`.
- **tails** — the candidate protein embeddings we score the drug against. Tail `j`
  occupies the `j`-th of the final `n` lines.
- **true_targets** — used only to *report recovery*; never fed to the scorer.

Regenerate (or make a bigger synthetic problem) with:

```bash
python scripts/make_synthetic.py                       # the committed 64×16 sample
python scripts/make_synthetic.py --n 100000 --dim 64   # a larger synthetic graph
```

## Full datasets (real polypharmacology knowledge graphs)

`scripts/download_data.ps1` / `.sh` print the sources below. They do **not**
auto-download: several require registration or have licenses that forbid
redistribution, so we never bypass that (CLAUDE.md §8).

| Source | What it provides | Access / license |
|---|---|---|
| **STRING** (https://string-db.org) | Protein–protein interaction (PPI) edges with confidence scores | CC BY 4.0; bulk download available |
| **DrugBank** (https://go.drugbank.com) | FDA-approved drugs and their known protein targets | Free **academic registration** required; redistribution restricted |
| **STITCH** (http://stitch.embl.de) | Chemical (drug)–protein interactions | Check per-use terms before redistribution |
| **DrugComb** (https://drugcomb.fimm.fi) | Drug-combination synergy measurements (for DeepSynergy-style tasks) | Cite the publication; check terms |

These are large (STRING alone spans billions of scored interactions across
thousands of organisms), which is exactly why GPU graph learning is needed.

## Provenance & honesty

The committed sample is **synthetic** and is labeled synthetic everywhere it
appears. No real patient, drug, or interaction data is included. This project is
**educational only** — its output is not a prediction about any real drug and must
not be used for any clinical or therapeutic decision.
