### File format

```
<n> <D> <M>                                   # e.g. "24 64 12"  (D must be 64, M must be 12)
<endpoint_name> <bias> <w_0 ... w_{D-1}>      # M lines, one logistic-regression model each
... (M endpoint lines total)
<mol_name> <x_0 ... x_{D-1}>                  # n lines, one descriptor vector each
... (n molecule lines total)
```

- **D** = descriptor length (= compiled `ADMET_D`); **M** = number of toxicity
  endpoints (= compiled `ADMET_M`). The loader
  (`src/reference_cpu.cpp::load_admet`) rejects a file whose `D`/`M` differ from
  the compiled constants, so a mismatched file fails loudly rather than reading
  garbage.
- All floating-point values are written at full **`%.17g`** precision so the
  doubles the C++ loader reads are **bit-for-bit identical** to those the Python
  generator used — there is no Python↔C++ rounding gap, which is part of why the
  CPU and GPU agree to ~1e-16.

Per molecule *i* and endpoint *t* the program computes
`p = sigmoid(bias_t + Σ_d w_{t,d} · x_{i,d})`, the predicted probability that
molecule *i* is "positive" (toxic / fails) for endpoint *t*.

## Full / real datasets

Real ADMET labels (and the models trained on them) come from:

- **Tox21** — 12 toxicity endpoints, ~8k compounds:
  <https://tripod.nih.gov/tox21/> (the M=12 here mirrors Tox21's 12 assays).
- **Therapeutics Data Commons (TDC) — ADMET benchmark group**:
  <https://tdcommons.ai/benchmark/admet_group/overview/>
- **ClinTox** — FDA-approved vs. failed-for-toxicity drugs (MoleculeNet):
  <https://moleculenet.org>
- **DILI** (drug-induced liver injury) databases — search current literature for
  a redistributable release (URL not pinned here on purpose).

`scripts/download_data.ps1` / `.sh` print the access recipe (these sets are
public but you fetch them yourself; the script never bypasses any
registration/credential step). To turn molecules into the descriptor format
this project expects you would, in production, use **RDKit** descriptors or
**Chemprop** D-MPNN features (see README "Prior art"); here we use a synthetic
stand-in instead. For a larger synthetic screen:

```
python scripts/make_synthetic.py --n 1000000
```

## Provenance & honesty

The committed sample is **synthetic** and labeled as such everywhere. The
endpoint names (`hERG_block`, `Ames_mutagen`, …) are illustrative labels only;
the weights are random and the probabilities are chemically meaningless. **No
clinical, chemical, or safety conclusion may be drawn from this sample or this
project** (CLAUDE.md §8). It exists solely to exercise and verify the GPU kernel.
