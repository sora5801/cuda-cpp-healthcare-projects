# Data — 1.35 QMMM/ML Potential Hybrid MD

> **Reduced-scope teaching version.** This project does **not** train a neural
> network and needs **no measured dataset to run**. Its ML-potential weights are
> small, **fixed, synthetic surrogate** values baked into `src/nnpmm.h` (clearly
> labeled there), standing in for a model trained on QM data. The only committed
> "data" is a tiny **run configuration** for the MD ensemble.

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/ensemble_params.txt` |
| Origin | **Synthetic** (a run spec, generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB |
| Layout | one line: `M  dt  steps  amp` |

### Field meanings

| Symbol | Meaning | Units | Sample value |
|---|---|---|---|
| `M` | number of ensemble members (independent MD trajectories) | count | 64 |
| `dt` | velocity-Verlet integration timestep | time units | 0.005 |
| `steps` | integration steps per trajectory | count | 300 |
| `amp` | max ± perturbation applied to the link atom across the ensemble | distance units | 0.20 |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). Regenerate or resize it with:

```bash
python scripts/make_synthetic.py --M 4096      # a larger ensemble
```

The physical system itself (a 1-D hybrid NNP/MM chain: 4 MM atoms, a link atom,
3 ML atoms) is defined in code (`src/nnpmm.h`), not in this file — the config
only chooses how many trajectories to run and how.

## Full dataset — the real ML-potential training sets

To build a *genuine* NNP/MM hybrid (a trained MACE/NequIP potential rather than
the fixed surrogate here), you would train on QM/DFT reference data. Use
`scripts/download_data.*`, which prints pointers to (and **respects the license
of**) the catalog's recommended sets:

- **Transition1x** — ~10M DFT calculations along reaction paths.
  <https://zenodo.org/record/5781475>
- **SPICE** — drug-like + biomolecular DFT energies/forces.
  <https://github.com/openmm/spice-dataset>
- **ANI-1ccx** reactive extensions — CCSD(T)\*-quality energies (verify URL).

Catalog dataset notes (verbatim):

> ANI-1ccx reactive extensions (verify URL); DFT reaction pathway datasets from QM/MM studies; Transition1x — 10M DFT calculations along reaction paths (https://zenodo.org/record/5781475); SPICE dataset (https://github.com/openmm/spice-dataset).

## Honesty & safety

Everything here is **synthetic and educational**. The fixed NNP weights are not a
trained model and the energies are not physical quantities for any real molecule.
Nothing in this project may be used for diagnosis, treatment, or any real medical
or chemical decision (CLAUDE.md §1, §8).
