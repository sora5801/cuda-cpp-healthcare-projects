### File format

```
# comment lines start with '#'
<n>                         # atom count, e.g. "27"
<element> <x> <y> <z>       # one atom per line; coords in Angstrom
...                         # exactly n atom lines
```

`element` is a one- or two-letter symbol (`C`, `O`, `N`, `S`, ...); the loader
keys the **van der Waals radius** on the first letter (Bondi 1964 radii, the set
FreeSASA defaults to). Unknown elements fall back to carbon (1.70 Å) with a
warning on stderr. See `src/reference_cpu.cpp::vdw_radius` /
`load_molecule`.

## Full dataset

Real SASA is computed on actual structures:

- **PDB (RCSB)** — experimental protein/ligand structures: <https://www.rcsb.org>
  (download a `.pdb`/`.cif`, e.g. `1CRN` crambin, a common small-protein test case).
- **ASA benchmark / FreeSASA test set** — for validating an implementation against
  a reference tool: <https://github.com/mittinatten/freesasa>.
- **MD trajectory ensembles** — SASA per frame (e.g. MDTraj `shrake_rupley`):
  <https://github.com/mdtraj/mdtraj>.

`scripts/download_data.ps1` / `.sh` print how to obtain a PDB file and convert it
to this simple `<element> x y z` format (one option: extract `ATOM`/`HETATM`
records, or use Biopython / MDTraj to write coordinates). They download nothing
that needs credentials and never bypass registration (CLAUDE.md §8).

To make a larger synthetic input instead:

```
python scripts/make_synthetic.py --out data/sample/molecule_sample.xyz
```

## Provenance & honesty

The committed sample is **synthetic** and labeled as such in the file header and
here. The coordinates are not a real molecule; the SASA values are meaningful as
a geometric exercise only. **No clinical, chemical, or structural conclusion** may
be drawn from this sample (CLAUDE.md §8).
