This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **synthetic**: it does not
contain any real protein, trajectory, or measurement — it is a set of parameters
for the toy REST2 run described in [../THEORY.md](../THEORY.md).

### File format

```
line 1:  n_replicas  sweeps_per_round  n_rounds
line 2:  barrier_h  tilt  k_bond  k_pw  x_solvent  step_size
line 3:  T0  Tmax
```

| Symbol | Meaning | Units | Sample value |
|---|---|---|---|
| `n_replicas` | replicas in the temperature ladder (≥ 2) | — | 8 |
| `sweeps_per_round` | Monte-Carlo sweeps between exchange attempts | — | 200 |
| `n_rounds` | number of (sample, exchange) rounds | — | 60 |
| `barrier_h` | double-well barrier height | `kT` (kT≡1) | 5 |
| `tilt` | linear bias lowering the **right** well (the global minimum) | `kT`/x | 2 |
| `k_bond` | bead–bead bond stiffness (solute-internal energy) | `kT`/x² | 0.5 |
| `k_pw` | solute–solvent coupling stiffness (the √λ term) | `kT`/x² | 0.2 |
| `x_solvent` | position of the implicit solvent field | x | 0 |
| `step_size` | Metropolis trial-move half-width | x | 0.35 |
| `T0` | cold (physical) temperature → λ = 1 | K | 300 |
| `Tmax` | hottest effective solute temperature | K | 900 |

The sample is **engineered to be interpretable** (PATTERNS.md §6): the barrier is
high enough that a 300 K replica started in the left well would stay trapped, but
the ladder + REST2 swaps let the cold replica reach the global **right** well.
Regenerate or sweep with, e.g.:

```bash
python scripts/make_synthetic.py --barrier-h 9 --n-replicas 16
```

## Full datasets (for validating a *real* REST2 implementation)

This teaching version uses synthetic parameters, **not** the datasets below — they
are listed so you know where a production REST2 study gets its validation data.
`scripts/download_data.ps1` / `.sh` print these pointers and **never bypass any
credential or license**.

- **Shaw (D. E. Shaw Research) millisecond folding trajectories** — long-timescale
  reference trajectories used to validate enhanced sampling. Access by request /
  collaboration; not redistributable here.
- **SAMPL challenges** — https://github.com/samplchallenges/SAMPL — blind
  binding/affinity challenge sets (open). Used to benchmark REST2-driven FEP.
- **GPCRmd** — https://gpcrmd.org — curated GPCR MD trajectories, some with REST2
  enhanced sampling. Web-accessible; respect site terms.
- **Chignolin / Trp-cage fast-folder benchmarks** — small fast-folding peptides
  (sequences are public) used as the standard REST2/REMD test systems.
