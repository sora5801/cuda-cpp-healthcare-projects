This tiny file lets `demo/run_demo` run **offline, with zero downloads** (a hard
requirement, CLAUDE.md §8). It is **synthetic** and carries **no clinical or
scientific validity** — it exists only to exercise the 3DVA pipeline.

### File format (whitespace-separated text; `#` starts a comment)

```
# two comment lines documenting the file
N G D                         # line 1: ints — N particles, grid edge G, D = G³ voxels
v0 v1 ... v(D-1)   # volume 0  # one flattened G×G×G density cube per particle
v0 v1 ... v(D-1)   # volume 1  #   (row-major voxel order: idx = (z*G + y)*G + x)
...                            # ... N such rows ...
t0 t1 ... t(N-1)   # truth     # last line: ground-truth conformational coordinate t[p]
```

### What each field means

- **A "volume"** is one particle reconstructed into a small 3-D density map and
  flattened to a length-`D` vector. Voxel value = local density (arbitrary units).
- **The heterogeneity:** every volume is the *same* Gaussian density blob, but
  the blob has **slid along the z-axis** by an amount set by `t[p] ∈ [−1, +1]`.
  That single sliding axis is the molecule's one "conformational degree of
  freedom" — the thing 3DVA must rediscover.
- **`t[p]` (ground truth)** is the hidden coordinate used to render particle `p`.
  The algorithm never sees it; `main.cu` only uses it *after the fact* to report
  how well the recovered PC1 latent coordinate correlates with it (≈ 0.997 here).
  Embedding a known answer in synthetic data is the pattern in PATTERNS.md §6.

Regenerate (deterministic) or make a bigger synthetic problem:

```bash
python scripts/make_synthetic.py                 # the committed 24×6³ sample
python scripts/make_synthetic.py --n 64 --g 8    # larger: more particles, finer grid
```

## Full / real datasets

This is a **reduced-scope teaching version** (see `../THEORY.md` → "Where this
sits in the real world"). The catalog points at real heterogeneous cryo-EM data;
fetching/preprocessing them (CTF correction, pose assignment, per-particle
back-projection) is out of scope for a single didactic project, so the committed
sample is synthetic. `scripts/download_data.ps1` / `.sh` print the pointers
below and never bypass any registration.

Catalog dataset notes (verbatim):

> EMPIAR-10180 (spliceosome), EMPIAR-10076 (80S ribosome), EMPIAR-10028 (TRPV1)
> (all at https://www.ebi.ac.uk/empiar/); cryoDRGN benchmark datasets
> (https://github.com/ml-struct-bio/cryodrgn); simulated heterogeneous datasets
> from IgG/spike protein.

- **EMPIAR** (https://www.ebi.ac.uk/empiar/) hosts raw movie/particle stacks; most
  entries are openly downloadable but are tens to hundreds of GB.
- **cryoDRGN** (https://github.com/ml-struct-bio/cryodrgn) ships small benchmark
  sets and tooling to turn a particle stack + poses into latent embeddings — the
  modern, nonlinear cousin of the linear 3DVA implemented here.
- **License:** respect each dataset's terms; none are redistributed in this repo.
