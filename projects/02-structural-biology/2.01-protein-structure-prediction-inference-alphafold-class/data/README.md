# Data — 2.1 Protein Structure Prediction Inference (AlphaFold-class)

> **Reduced-scope teaching version.** This project implements **one self-attention
> head** (the core Evoformer/transformer building block), not the full AlphaFold
> pipeline. Its input is therefore a tiny set of **synthetic** Q/K/V matrices, not
> a real protein, MSA, or set of trained weights. See the project
> [README "Limitations & honesty"](../README.md#limitations--honesty).

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/attention_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (deterministic) |
| License | Public domain (CC0) — it is synthetic, not derived from any dataset |
| Size | < 4 KB |
| Shape | `L = 6` residues, `d = 32` feature channels |

### File layout

```
<L> <d>                 # header: sequence length and feature width (d must == D_MODEL = 32)
# comment lines start with '#' and are skipped by the loader
<L rows of d floats>    # Q (query)  matrix, row-major: residue r is row r
<L rows of d floats>    # K (key)    matrix, row-major
<L rows of d floats>    # V (value)  matrix, row-major
```

Each row is one residue's length-`d` feature vector. `Q`, `K`, `V` are the three
linear projections a real attention layer would learn; here they are constructed
directly (no learned weights).

### What the synthetic numbers mean (and the embedded known answer)

The sample is engineered to have a **verifiable** answer (see
[`docs/PATTERNS.md` §6](../../../../docs/PATTERNS.md)). Residue `r` places a large
"identity" peak in feature channel `r % d` of its `Q` and `K` vectors, plus a
small deterministic ramp elsewhere. Consequently the affinity `Q[i]·K[j]` is
largest when `i == j`, so **every residue attends most strongly to itself** — the
identity-like baseline of self-attention, which the demo prints and checks.
Channel 0 of `V` stores the residue index `r+1`, making the mixed output easy to
read. **None of this is biological** — it is a reproducible toy to exercise the
kernel.

Regenerate it (or make a bigger one) with:

```bash
python scripts/make_synthetic.py --L 6      # the committed sample
python scripts/make_synthetic.py --L 64     # a larger synthetic problem
```

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8).

## Full / real-world data

A real AlphaFold-class prediction does **not** consume a Q/K/V text file — it
consumes an amino-acid sequence, builds a multiple-sequence alignment (MSA) from
huge sequence databases, and runs trained model weights. None of that is needed
(or shipped) for this teaching kernel, but for further study:

- **AlphaFold Protein Structure Database** — 200M+ predicted structures:
  <https://alphafold.ebi.ac.uk/>
- **RCSB PDB** — 227k+ experimental structures: <https://www.rcsb.org>
- **UniProt / UniRef90** — sequence databases used to build MSAs:
  <https://www.uniprot.org>
- **CAMEO / CASP15** — structure-prediction benchmarks: <https://www.cameo3d.org>

`scripts/download_data.ps1` / `.sh` print these pointers. They deliberately do
**not** download model weights or MSA databases (multi-gigabyte, and some require
registration) — respecting licenses and credentials (CLAUDE.md §8).

## Honesty

The committed data is **synthetic and labeled synthetic** everywhere it appears.
Nothing here is a real protein structure, and no output of this project may be
used for any clinical or scientific decision (educational use only).
