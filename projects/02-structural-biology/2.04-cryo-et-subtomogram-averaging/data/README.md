# Data — 2.4 Cryo-ET Subtomogram Averaging

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/subtomograms_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | ~260 KB |
| Generator seed | `12345` (fixed → byte-identical every run) |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is **synthetic** and carries no
biological meaning; it exists only to exercise the alignment + averaging pipeline
with a *known* answer.

### File layout

Whitespace-separated floats:

```
line 1 (header):  n_sub  d  n_angles
then:             ref cube   ( d*d*d floats, one line )
then:             cand cube 0 ( d*d*d floats, one line )
                  cand cube 1
                  …
                  cand cube (n_sub-1)
```

| Symbol | Meaning | Units | Sample value |
|--------|---------|-------|--------------|
| `n_sub` | number of candidate subtomograms | count | `6` |
| `d` | cube edge length | voxels | `16` |
| `n_angles` | trial in-plane rotation angles in the search | count | `12` |
| `ref` | the reference cube (the ideal "average"/template) | arbitrary density | `d³` floats |
| `cand[s]` | candidate `s` = the motif rotated by a planted angle + Gaussian noise | arbitrary density | `d³` floats each |

Voxels are stored **z-major then y then x**: `vox[(z*d + y)*d + x]`. The loader
(`src/reference_cpu.cpp::load_subtomograms`) subtracts each cube's mean
(zero-meaning) so cross-correlation peaks are meaningful.

### What is planted (the known answer)

Candidate `s` is the clean motif rotated so that the alignment search should
recover **trial angle index `(2s+1) mod n_angles`** = `[1, 3, 5, 7, 9, 11]` for
the default sample, plus mild Gaussian noise (`sigma = 0.04`). The demo prints
these recovered indices — recovering the planted answer is the project's
science-level correctness check (PATTERNS.md §6). Regenerate with:

```bash
python scripts/make_synthetic.py            # default 6 cubes, 16^3, 12 angles
python scripts/make_synthetic.py --d 32     # a bigger problem
```

## Full dataset

Real cryo-ET subtomograms are large, require heavy preprocessing (tomogram
reconstruction + particle picking), and are generally **not redistributable** —
so this project ships a synthetic sample and points you to the real sources:

- **EMDB STA deposits** — https://www.ebi.ac.uk/emdb/ (deposited STA maps).
- **EMPIAR** — https://www.ebi.ac.uk/empiar/ (raw tilt series & subtomograms;
  e.g. EMPIAR-10064 and related deposits). Large (GBs–TBs).
- **SHREC subtomogram challenge** — synthetic-but-realistic benchmark volumes
  with ground-truth particle locations (search "SHREC cryo-ET"; verify the
  current URL, which moves year to year).
- **CryoDRGN-ET benchmark** — https://github.com/ml-struct-bio/cryodrgn
  (heterogeneity benchmarks).

**License:** respect each source's terms. EMDB/EMPIAR data are openly available
for research but have their own citation/usage policies; this repo does **not**
redistribute them. `scripts/download_data.ps1` / `.sh` print links and
instructions only — they never bypass any registration or license gate
(CLAUDE.md §8).

## Provenance & field meanings

The committed sample is **100% synthetic** (a 3-blob Gaussian motif, rotated and
noised) and is labeled synthetic everywhere it appears. It implies **no clinical
or biological validity**. To work with real data, fetch a SHREC or EMPIAR set via
the download script, extract `d³` cubes around picked particles, write them in
the layout above, and point the demo at that file.
