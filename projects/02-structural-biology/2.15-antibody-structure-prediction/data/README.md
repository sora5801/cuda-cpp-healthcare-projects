# Data — 2.15 Antibody Structure Prediction (reduced-scope: CDR screening)

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/antibodies_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (seed 215) |
| Content | 1 query antibody + 24 library antibodies, each described by its six CDR loops |
| License | Public domain (CC0) — it is synthetic; **not real antibodies, not for clinical use** |
| Size | < 2 KB |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8).

### File format

Plain text. Lines beginning with `#` are comments. There is exactly one `QUERY`
line and then one line per library antibody:

```
QUERY <name> <H1> <H2> <H3> <L1> <L2> <L3>
<name> <H1> <H2> <H3> <L1> <L2> <L3>
...
```

- **`<name>`** — a human-readable identifier (e.g. `mAb_07`). Reported, not scored.
- **`<H1> <H2> <H3>`** — the three heavy-chain CDR loops as amino-acid strings
  (one-letter codes). **CDR-H3** is the hypervariable, antigen-specificity loop;
  the screen weights it **×3**.
- **`<L1> <L2> <L3>`** — the three light-chain CDR loops, same encoding.

The loader (`src/reference_cpu.cpp`) encodes each residue to a 0–20 integer
(20 amino acids + gap), right-pads each CDR to 24 residues, and stores the six
fields contiguously (144 residues per antibody record). Unknown letters and `-`
map to the gap symbol.

### The planted answer (so the demo is interpretable)

The generator embeds a **known result** (PATTERNS.md §6) so you can verify the
screen recovers it:

- **`mAb_07`** is a **near-copy of the query** (each CDR is the query's CDR with
  two point mutations) → it should rank **#1**.
- **`mAb_18`** shares the query's **exact CDR-H3** but has unrelated other CDRs →
  it should rank **#2** (driven almost entirely by the ×3-weighted CDR-H3).

Every other antibody has unrelated random CDRs and scores near zero. The demo's
`expected_output.txt` shows mAb_07 (516) and mAb_18 (373) far above the rest.

## Full dataset

The committed sample is synthetic. To screen **real** antibody CDRs, obtain them
from the sources below (each has its own license — respect it) and convert to the
format above. `scripts/download_data.ps1` / `.sh` print the URLs and the
conversion recipe; they do **not** download or bypass any registration.

- **SAbDab — Structural Antibody Database** (IMGT-numbered antibody structures):
  <https://opig.stats.ox.ac.uk/webapps/newsabdab/sabdab/>
- **Thera-SAbDab — therapeutic antibodies:**
  <https://opig.stats.ox.ac.uk/webapps/newsabdab/therasabdab/>
- **OAS — Observed Antibody Space** (~2 billion antibody sequences):
  <https://opig.stats.ox.ac.uk/webapps/oas/>
- **CASP-Ab** antibody-modeling benchmarks (community assessments).

**Conversion recipe:** IMGT-number each Fv (e.g. with **ANARCI**) to delimit the
six CDR loops, then emit one line per antibody. Pick one antibody as the `QUERY`.
