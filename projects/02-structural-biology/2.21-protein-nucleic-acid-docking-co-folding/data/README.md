## Committed sample (`sample/complex_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/complex_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` |
| License | Public domain (CC0) — it is synthetic, derived from no real structure |
| Size | < 2 KB (25 protein atoms + 9 ligand atoms) |
| Units | Fixed-point **milli-Ångström** integers (`1000` = `1.000 Å`) |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is **engineered to have a known,
unique answer** (a planted "native" pose), so the demo is interpretable: the
search must rank that pose #1.

### File format

Lines beginning with `#` and blank lines are ignored. The meaningful lines are, in
order:

```
Np Nl                                         # atom counts
tx0 ty0 tz0 step nx ny nz                      # translational pose lattice (milli-Å)
clash_r2 contact_r2 clash_pen contact_w elec_w # scoring params (see below)
x y z charge        × Np                        # protein atoms
x y z charge        × Nl                        # ligand (nucleic-acid) atoms
```

Per-field meaning:

| Field | Meaning | Units |
|---|---|---|
| `Np`, `Nl` | number of protein / ligand atoms | count |
| `tx0,ty0,tz0` | translation lattice origin | milli-Å |
| `step` | lattice spacing | milli-Å |
| `nx,ny,nz` | lattice point counts per axis | count |
| `clash_r2` | squared distance below which a pair "clashes" | milli-Å² |
| `contact_r2` | squared distance below which a pair is "in contact" | milli-Å² |
| `clash_pen` | penalty subtracted per clashing pair | score units |
| `contact_w` | bonus added per contacting pair | score units |
| `elec_w` | electrostatic weight (per contacting pair) | score units |
| `x y z` | atom position | milli-Å (int) |
| `charge` | formal charge **sign** ∈ {−1, 0, +1} | — |

Orientations are **not** in the file — they are the fixed 24-element cube rotation
group, built in code (`cube_rotations()` in `src/reference_cpu.cpp`).

### What the planted complex is (and why)

- The protein is a flat **5×5 plane** of atoms at `z = 0`; its central **3×3 patch**
  carries a deliberately **chiral "L"-shaped** pattern of ±1 charges (the rim is
  neutral). The chirality makes the native pose the **unique** maximum — a
  symmetric checkerboard would tie under several cube rotations.
- The ligand is a **3×3 plane** whose charges are the **opposite sign** of the
  protein patch directly below — the complementary "key".
- The **native pose** is identity rotation translated to `(0, 0, +step)`: the ligand
  sits one contact shell above the patch, every charge pair is attractive, nothing
  clashes → the global-best score (340 in the committed sample).

Regenerate (e.g. with a different spacing) via:

```bash
python scripts/make_synthetic.py --spacing 3500
```

> **Synthetic, not real.** These coordinates correspond to **no** real molecule and
> imply **no** biological or clinical validity. They exist only to exercise and
> verify the docking kernel.

## Full dataset (real complexes)

Real protein-nucleic-acid complexes come from the **Protein Data Bank (PDB)** and
RNA-structure benchmarks. `scripts/download_data.ps1` / `.sh` print fetch
instructions and example URLs; they **never** bypass any access controls. Turning a
downloaded `.pdb`/`.cif` into this loader's integer format (extract atoms, assign
charge signs, scale to milli-Å) is left as an exercise — the point of this project
is the GPU search, not a production PDB parser.

Catalog dataset notes (verbatim):

> PDB protein-nucleic acid complexes (https://www.rcsb.org); RNA structure
> benchmarks from RNA-Puzzles (https://github.com/RNA-Puzzles); PDB-NA complex
> benchmark sets; Rfam RNA family database (https://rfam.org).

- **License:** PDB data is freely available (CC0); respect each entry's citation
  guidance. RNA-Puzzles / Rfam carry their own terms — check before redistributing.
- **Credentialed sets:** none required for the PDB; if a benchmark needs
  registration, the script prints the link and does not attempt to bypass it.
