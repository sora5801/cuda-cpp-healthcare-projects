# Data — 2.32 Protein Folding Pathway Extraction (Transition Path Sampling)

## Committed sample (`sample/tps_params.txt`)

| Field | Value |
|---|---|
| File | `sample/tps_params.txt` |
| Origin | **Synthetic** simulation parameters (`scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB (one line) |
| Contents | the double-well landscape + Brownian-dynamics + shooting-run parameters |

This project's "data" is the **simulation parameters**, not a measured
trajectory. The dynamics (the folding/unfolding shooting trajectories) are
generated by the program itself from these parameters and a fixed RNG seed, so
the demo runs **offline with zero downloads** (CLAUDE.md §8).

### File format (one whitespace-separated line)

```
barrier  x0  w  D  dt  basin_tol  max_steps  n_shooters  n_bins  seed
```

| Field | Meaning | Units |
|---|---|---|
| `barrier` | height of the double-well folding barrier | kT (thermal energy) |
| `x0` | landscape centre = the transition-state position on the reaction coordinate | reduced (≈ fraction of native contacts) |
| `w` | basin half-separation: unfolded basin A at `x0 − w`, folded basin B at `x0 + w` | reduced |
| `D` | diffusion constant on the reaction coordinate | reduced |
| `dt` | Brownian-dynamics integration timestep | reduced |
| `basin_tol` | distance from a basin minimum that counts as "arrived" (must be `< w`) | reduced |
| `max_steps` | per-leg step budget (rare-event safety net so a leg always terminates) | steps |
| `n_shooters` | number of independent TPS shooting moves to run (one GPU thread each) | count |
| `n_bins` | committor-histogram resolution along the reaction coordinate | bins |
| `seed` | base RNG seed (shooter `i` uses the reproducible stream `(seed, i)`) | integer |

Default sample: `5.0 0.5 0.4 1.0 0.0005 0.05 20000 4096 20 20240517`.

Regenerate or rescale it with, e.g.:

```bash
python scripts/make_synthetic.py --shooters 16384 --barrier 6.0
```

## "Full dataset" / realistic inputs

A research-grade TPS study does **not** use a 1-D toy landscape — it shoots
**all-atom molecular-dynamics** trajectories of a real protein and uses long
reference trajectories as a source of shooting points. The catalog points to:

- **Anton / D. E. Shaw millisecond folding trajectories** — used as TPS starting
  configurations (access is by request to D. E. Shaw Research).
- **GPCRmd** (<https://gpcrmd.org>) — curated GPCR MD trajectories & pathways.
- **Folding benchmarks** — Trp-cage, chignolin, and the WW domain are the
  standard small fast-folders (structures via the **PDB**, <https://www.rcsb.org>;
  e.g. chignolin `5AWL`, Trp-cage `1L2Y`).
- **SAMPL host–guest kinetics challenges** — binding/unbinding kinetics datasets
  (URL changes per round; search "SAMPL challenge").

`scripts/download_data.ps1` / `.sh` print these pointers and **never bypass any
registration**; they defer to `make_synthetic.py` for the offline stand-in.

## Provenance & honesty

This is a **deliberately simplified, synthetic** teaching model: a **1-D
reaction coordinate** on an analytic **double-well free-energy surface**,
**overdamped Langevin (Brownian) dynamics**, and a **simplified shooting move**
(the backward leg is an independent forward BD leg rather than a rigorous
momentum-reversal). There is **no force field, no solvent, no atoms**. It teaches
the *structure* of Transition Path Sampling — shooting, acceptance, committor —
not quantitative folding kinetics, and must never be used for any biological or
clinical claim.
