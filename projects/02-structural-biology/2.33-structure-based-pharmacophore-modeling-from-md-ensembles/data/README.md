This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard

no real ligands, receptors, or MD trajectories in it.

### File format

A whitespace/newline-separated text file:

```
N  n_query  target            # header: N library molecules, query feature count, planted-target index
<query block>                 # n_query rows, each:  type x y z weight
<molecule 0>                  # a line "m" (feature count) then m rows: type x y z weight
<molecule 1>
...
<molecule N-1>
```

Each **feature row** is `type x y z weight`:

| Token | Meaning | Units / range |
|---|---|---|
| `type` | feature class (see below) | integer `0..5` |
| `x y z` | 3-D center of the feature | angstroms |
| `weight` | feature importance (consensus features from many MD frames → ~1) | `[0,1]` |

Feature **types** mirror the common ROCS/Pharmer "color" types (see
`src/pharmacophore.h`, enum `FeatureType`):

| `type` | Meaning |
|---|---|
| 0 | hydrogen-bond donor |
| 1 | hydrogen-bond acceptor |
| 2 | hydrophobe |
| 3 | aromatic ring centroid |
| 4 | positively ionizable |
| 5 | negatively ionizable |

### How the synthetic answer is planted

`make_synthetic.py` builds the query pharmacophore, then makes library molecule
`target` (default index **7**) a near-copy of the query: the **same feature types**
with **sub-angstrom position jitter** (±0.25 Å) plus one or two random decoy
features. Every other molecule has random typed features at random positions. So
the demo has a **known ground truth**: molecule 7 should score far above the
decoys and rank **#1**. Regenerate (optionally larger) with:

```
python scripts/make_synthetic.py                 # the committed 512-molecule sample
python scripts/make_synthetic.py --N 100000      # a bigger SYNTHETIC screen
```

## Full / real datasets

`scripts/download_data.ps1` / `.sh` print pointers and instructions only — they do
**not** download or bypass any registration. The catalog names these real sources;
turning them into pharmacophore feature points requires a feature-extraction step
(e.g. RDKit/Pharmer pharmacophore typing, fpocket pocket detection), which is out
of scope for this teaching version (described in `THEORY.md` §Where this sits in
the real world):

- **GPCRmd trajectory archive** — https://gpcrmd.org — GPCR MD trajectories (the
  "ensemble" a real consensus pharmacophore would be derived from).
- **DUD-E** — https://dude.docking.org — actives/decoys for screening validation.
- **RCSB PDB** — https://www.rcsb.org — experimental structures of target classes.
- **ZINC** — https://zinc20.docking.org — drug-like library for screening.

**Licensing:** respect each source's terms. None of that data is redistributed
here; the committed sample is synthetic precisely so this repo stays license-clean
and runnable offline. Nothing here is for clinical or real screening use.
