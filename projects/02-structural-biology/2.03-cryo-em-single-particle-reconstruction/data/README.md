## Committed sample (`sample/cryoem_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/cryoem_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (seed 11) |
| License | Public domain (CC0) — it is synthetic, no real specimen involved |
| Size | ~145 KB |
| Geometry | 64×64 image, 60 reference angles over [0, π), 64-sample projections |
| Particles | 120 noisy 1-D projections at random (recorded) orientations |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **100% synthetic** — there is
no real cryo-EM micrograph, particle, or specimen anywhere in this repository.

### File layout (what the loader in `src/reference_cpu.cpp` reads)

All tokens are whitespace-separated ASCII:

```
line 1            :  n  IMG_SIZE  N_ANGLES  PROJ_LEN        (header + geometry self-check)
IMG_SIZE*IMG_SIZE :  the ground-truth density (row-major floats; the "molecule")
N_ANGLES*PROJ_LEN :  the reference projection bank (row a = projection at angle a)
per particle (n)  :  one int  true_angle_index            (synthetic ground truth)
                     then PROJ_LEN floats  the noisy 1-D projection profile
```

### Per-field meaning

- **`IMG_SIZE` (64):** side length, in pixels, of the square density grid. The
  "molecule" is a sum of Gaussian blobs placed **asymmetrically** so that its
  projections differ across angles — that asymmetry is what makes orientation
  recovery possible (a radially symmetric blob would project identically at every
  angle).
- **`N_ANGLES` (60):** number of reference projection directions, evenly spaced
  over `[0, π)` (angle `a` → `θ_a = a·π/60`, i.e. 3° apart). Half a circle
  suffices because the projection at `θ` and `θ+π` are mirror images.
- **`PROJ_LEN` (64):** number of detector samples in each 1-D projection.
- **`true_angle_index`:** the angle each particle was generated at. The matcher
  **never sees this** — it is used only by `main.cu` to *report* recovery accuracy.
- **Profile floats:** the line integrals of the density along parallel rays at
  the particle's angle, **plus Gaussian noise** (σ ≈ 15% of the profile RMS) to
  mimic cryo-EM's notoriously low signal-to-noise ratio.

Regenerate (deterministic; identical bytes for the same seed):

```bash
python scripts/make_synthetic.py --n 120          # the committed sample
python scripts/make_synthetic.py --n 100000       # a larger stress set
```

## Full / real datasets

This project is a **reduced-scope teaching model** in 2-D. Real single-particle
cryo-EM works on stacks of large 2-D micrographs and 3-D density maps. To explore
real data (none of it is redistributed here — respect every license):

| Source | What it is | URL |
|---|---|---|
| **EMDB** | 50,000+ deposited 3-D cryo-EM density maps (`.map`/MRC) | https://www.ebi.ac.uk/emdb/ |
| **EMPIAR** | Raw cryo-EM particle image stacks (large; tens of GB) | https://www.ebi.ac.uk/empiar/ |
| **RCSB PDB** | Atomic models fit into cryo-EM maps | https://www.rcsb.org |
| **cryoDRGN** | Benchmark heterogeneous-reconstruction datasets | https://github.com/ml-struct-bio/cryodrgn |

See `scripts/download_data.ps1` / `.sh` for pointers. Those scripts **print
instructions only** — EMPIAR entries are large and some require acknowledging a
data-use agreement; the scripts never bypass that. For an offline stand-in at any
size, use `make_synthetic.py`.

## Honesty note

The data here is synthetic and the pipeline is a simplified 2-D analogue (no CTF,
no Bayesian weighting, known reference angles). Nothing in this project is
suitable for any real structural-biology or clinical conclusion — it exists to
teach the **projection-matching + back-projection** GPU pattern. See
`../THEORY.md` §"Where this sits in the real world".
