### Scene file format

A whitespace-separated text format the loader (`src/reference_cpu.cpp`,
`load_scene`) understands:

```
<n_atoms> <width> <height> <ao_samples>      # header line
<x> <y> <z> <radius> <color>                 # one line per atom, n_atoms of them
...
```

| Field | Units | Meaning |
|---|---|---|
| `n_atoms` | count | number of atom lines that follow |
| `width`, `height` | pixels | rendered image resolution |
| `ao_samples` | count | ambient-occlusion rays cast per pixel (more = smoother, slower) |
| `x`, `y`, `z` | Ångström (Å) | atom centre in world coordinates |
| `radius` | Ångström (Å) | van-der-Waals radius of the sphere |
| `color` | id | CPK colour id: `0`=C grey, `1`=O bright, `2`=N dark, `3`=H near-white, `4`=S |

The camera is **auto-fitted** to the molecule's bounding box at load time, so any
valid scene frames itself. The synthetic helix is a 5-turn backbone of carbon
spheres with outward nitrogen side chains and bright oxygen caps — chosen so the
ambient-occlusion shading has deep grooves to darken, making 3-D shape legible.

To regenerate or resize:

```
python scripts/make_synthetic.py --turns 6 --width 320 --height 320 --ao 64
```

## Full / real datasets

The catalog lists these real sources. Our renderer reads the simple `.scene`
format above; to render a **real** structure, convert a PDB/EMDB file into that
format (extract each ATOM/HETATM record's coordinates, assign a VDW radius and a
CPK colour id by element). A converter is left as an **Exercise** in the README.

- **RCSB PDB** — molecular structures (atoms): <https://www.rcsb.org>
- **EMDB** — cryo-EM density maps (volumes; this project renders atoms, not
  volumes — volume rendering is the natural extension): <https://www.ebi.ac.uk/emdb/>
- **GPCRmd** — molecular-dynamics trajectories: <https://gpcrmd.org>
- **CHARMM-GUI** — example membrane/protein systems: <https://charmm-gui.org>

`scripts/download_data.ps1` / `.sh` print pointers to these sources; they do
**not** download anything automatically (PDB/EMDB entries are large and each has
its own terms — always check the per-entry license). None of these are needed
for the demo, which runs entirely on the committed synthetic sample.

## Honesty & safety

The committed sample is **synthetic** and labelled synthetic everywhere. Nothing
here is a real patient- or specimen-derived structure, and no rendered image
implies any clinical or biological conclusion (CLAUDE.md §8). This project is
**educational**: it teaches the GPU ray-tracing pattern, not production molecular
graphics.
