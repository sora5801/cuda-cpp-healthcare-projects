# Demo — 2.22 Electron Density Map Analysis & Model Validation

## What this demonstrates

Running `run_demo.ps1` (Windows) or `run_demo.sh` (Linux/CMake) will:

1. **Build** the project if the executable is missing.
2. **Run** it on the committed `data/sample/map_sample.txt` — a synthetic pair of
   16³ electron-density maps.
3. **Compute** two model-validation scores, on both the CPU (reference) and the
   GPU (cuFFT):
   - **RSCC** — the real-space correlation coefficient of the two maps.
   - **FSC** — the Fourier shell correlation curve, plus the resolution where it
     crosses the 0.143 (gold-standard half-map) and 0.5 (map-vs-model) cutoffs.
4. **Verify** the GPU result against the CPU reference and print `PASS`/`FAIL`.
5. **Time** the GPU (CUDA events) and the CPU baseline — a *teaching artifact*,
   not a benchmark claim.

The program splits its output deliberately:

- **stdout** is byte-for-byte deterministic and is diffed against
  [`expected_output.txt`](expected_output.txt). It prints the CPU's
  double-precision RSCC, FSC curve, and resolution estimates.
- **stderr** carries the timing and the GPU-vs-CPU numeric error (which vary run
  to run), so it is shown but never diffed.

## How to read the result

The synthetic maps share the same low-frequency structure (a few Gaussian
"atoms") but map B has extra **high-frequency** noise. So the FSC curve is the
teaching payoff:

- FSC ≈ **1.0** in the **low-frequency** shells (the big features agree).
- FSC **decays toward 0** at **high frequency** (the fine detail disagrees).
- The shell where FSC drops below **0.143** is the estimated **resolution** — for
  this sample, **8.0 Å**. That is exactly how cryo-EM software reports map
  resolution.
