## File format (the loader in `src/reference_cpu.cpp`)

Whitespace/newline separated, `#` begins a comment to end-of-line:

```
# header line:
n_beads n_chains box sigma epsilon r_cut k_bond r0 mass dt n_steps
# then n_beads rows, one bead per row (chains are consecutive blocks of rows):
x y z vx vy vz lambda
```

| Field | Meaning | Units (reduced LJ units) |
|-------|---------|--------------------------|
| `n_beads` | total coarse-grained beads (residues) | — |
| `n_chains`| number of IDP chains (`n_beads % n_chains == 0`) | — |
| `box` | cubic box side L (periodic in x,y,z) | length |
| `sigma` | bead diameter (LJ length scale) | length |
| `epsilon` | LJ well depth (energy scale) | energy |
| `r_cut` | non-bonded cutoff radius | length |
| `k_bond` | harmonic backbone bond stiffness | energy / length^2 |
| `r0` | bond rest length | length |
| `mass` | bead mass | mass |
| `dt` | velocity-Verlet time step | time |
| `n_steps`| number of MD steps to run | — |
| `x y z` | bead position | length |
| `vx vy vz` | bead velocity | length / time |
| `lambda`| per-residue stickiness in [0,1] | — |

Chain membership is implied by row order: the first `chain_len = n_beads/n_chains`
rows are chain 0, the next block is chain 1, and so on. Consecutive beads of the
same chain are bonded (harmonic spring); all other pairs use the HPS / Ashbaugh-
Hatch non-bonded potential.

## Regenerate / scale the synthetic system

```bash
python scripts/make_synthetic.py                                    # the committed default
python scripts/make_synthetic.py --chains 16 --len 12 --box 12.0    # bigger droplet
python scripts/make_synthetic.py --lam 0.2                          # weak stickiness: stays dilute
```

A fixed RNG seed makes the output byte-identical, so `demo/expected_output.txt`
stays stable. Lowering `--lam` toward 0 is the most instructive change: the
chains no longer stick and the system fails to condense — the "soluble" side of
the phase boundary.

## Real-world data (for the curious; not needed for the demo)

`scripts/download_data.{ps1,sh}` prints pointers to the actual LLPS / IDP
resources. They are **databases**, not ready-to-run particle inputs:

- **PhaSePro** — proteins undergoing LLPS — https://phasepro.elte.hu
- **PhaSepDB** — phase-separation database — http://db.phasep.pro
- **DisProt** — intrinsically disordered proteins — https://disprot.org
- **FuzDB** — fuzzy protein complexes — https://fuzdb.org
- **CALVADOS** — residue-level IDP force field + lambda tables — https://github.com/KULL-Centre/CALVADOS
- human-proteome LLPS predictors: catGRANULE, PScore

To turn a real sequence (e.g. the FUS low-complexity domain) into a runnable
system you map each residue to its HPS/CALVADOS lambda, lay the beads on a chain,
and emit the loader format above. We use synthetic lambda here to keep the repo
free of licensing entanglements and to make the demo self-contained.
