### File format

```
<n_recv> <n_lig> <N> <spacing>  <ans_x> <ans_y> <ans_z>   # header
<x y z>        # receptor atom 0  (Angstrom)
...            # n_recv receptor atom lines
<x y z>        # ligand   atom 0  (Angstrom)
...            # n_lig ligand atom lines
```

- `n_recv`, `n_lig` — number of receptor / ligand atoms.
- `N` — grid edge length in voxels (the search grid is `N×N×N`).
- `spacing` — Ångström per voxel (grid resolution).
- `ans_x ans_y ans_z` — **optional** known-answer translation, in signed voxels:
  the docking translation the FFT search should recover. Present only for
  synthetic samples; omit these three numbers for a real complex (there is no
  pre-known rigid answer). The loader falls back to "no truth" when absent.

### How the sample is built (and why the answer is known)

`make_synthetic.py` builds the receptor as the union of a few overlapping atom
spheres (a lumpy, non-symmetric blob), then makes the **ligand a whole copy of
the receptor displaced by `D = (3, 2, −1)` voxels**. Both are voxelized with the
same core/skin shape rule, so the docking score is the *autocorrelation* of the
shape function. By Cauchy-Schwarz that autocorrelation has a **unique** maximum
exactly where the copies re-register — at `t = −D = (−3, −2, 1)` — which is the
value recorded in the header and what the program recovers. This "embed a known
answer" trick (cf. project 12.01) makes the demo result verifiable.

> The sample is **synthetic** and labeled as such everywhere. It is a geometric
> test object, not a real protein complex, and carries no biological or clinical
> meaning. It exists to make the FFT-docking result interpretable and checkable.

For a different size: `python scripts/make_synthetic.py --N 48 --spacing 1.5`.

## Full / real datasets

Real rigid-body docking benchmarks (download separately; see
`scripts/download_data.*`):

- **Docking Benchmark 5.5** — 230 non-redundant protein complexes, with the
  bound and unbound structures used to score docking methods
  (<https://zlab.umassmed.edu/benchmark/>).
- **SAbDab** — the structural antibody database, for antibody-antigen docking
  (<https://opig.stats.ox.ac.uk/webapps/newsabdab/sabdab/>).
- **PDB** — any deposited protein complex (<https://www.rcsb.org>); split a
  complex into its chains to make a receptor/ligand pair.

To dock real structures with this code you would parse the PDB atom records into
`x y z` triples (one block per chain) and drop the known-answer header fields.
Production pipelines additionally search **thousands of ligand rotations** and
add electrostatics — see `THEORY.md` "Where this sits in the real world".

## Provenance & honesty

Everything in `sample/` is generated by `make_synthetic.py` and is labeled
synthetic. No real patient- or structure-derived data is committed. Respect the
license of any real dataset you download (the benchmarks above are freely
available for research; cite them).
