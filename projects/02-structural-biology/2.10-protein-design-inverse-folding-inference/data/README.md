```bash
python scripts/make_synthetic.py            # exactly reproduces the committed file
```

### What each field means

- `L` — the number of residues in the backbone.
- `x y z` — the **Cα atom coordinate** of that residue, in **ångström (Å)**. (The
  Cα is the central backbone carbon; its position is the standard one-point
  summary of where a residue sits.)
- `native_letter` — the **native amino acid** at that residue as a standard
  one-letter code (one of `ARNDCQEGHILKMFPSTWYV`). This is the "ground truth"
  sequence the design's *recovery* is scored against; it is **never** used to
  produce the design.

### How the synthetic sample is constructed (and why)

`make_synthetic.py` places residues on **concentric shells**, from a tightly
packed core (small radius → many neighbours → buried) out to a sparse surface
(large radius → few neighbours → exposed), so the per-residue **burial** spans a
realistic range rather than two extremes. The native residue at each position is
then chosen to *mostly fit* that position's burial (hydrophobic in the core, polar
on the surface), with ~25% of positions randomly "mutated" so that native
sequence recovery lands at a believable, **less-than-100%** value. This makes the
demo's headline metric meaningful and the buried/exposed split visible. The whole
thing is driven by a fixed RNG seed, so the committed file — and therefore
`demo/expected_output.txt` — is byte-stable.

> **Labeled synthetic.** This is **not** a real protein. It is a geometric toy
> that exercises the burial→hydrophobicity rule cleanly. Do not interpret the
> coordinates as a real structure or the designed sequence as a real protein.

## Full dataset (real backbones)

Real backbones and native sequences come from public structural databases. None of
these are redistributed here; use the pointers below (and the
`scripts/download_data.*` helpers, which only print instructions — they never
bypass any registration):

- **PDB — RCSB Protein Data Bank** (<https://www.rcsb.org>): the primary source of
  experimentally determined protein structures (`.pdb` / `.cif`). The Cα
  coordinates and the native sequence for any chain can be parsed from a structure
  file. License: most PDB data is freely redistributable (CC0-like); check each
  entry.
- **CATH** (<https://www.cathdb.info>): 500k+ classified domain structures — the
  kind of curated, non-redundant backbone set ProteinMPNN-style models train on.
- **ProteinGym** (<https://github.com/OATML-Markslab/ProteinGym>): mutational
  fitness benchmarks, useful for evaluating how design scores track function.
- **CAMEO** (<https://www.cameo3d.org>): continuous structure-prediction
  validation; a source of fresh backbones.

To use a real backbone with this teaching code you would parse a `.pdb` file's
`CA` atom records into the `x y z native_letter` format above (one residue per
line). That parser is left as an exercise — the committed synthetic sample is
enough to run and verify the demo.
