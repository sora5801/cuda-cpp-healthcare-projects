### File format

```
# free-form comment lines (begin with '#')
# SITE_ALLO   <int>      <- residue index of the (engineered) allosteric site
# SITE_ACTIVE <int>      <- residue index of the (engineered) active site
N T                      <- two ints: number of residues, number of frames
x y z                    <- N coordinate lines per frame, T frames (frame-major)
... (T*N coordinate lines total)
```

Coordinates are in arbitrary "angstrom-like" units (the demo is dimensionless). The
order is **frame-major**: all N residues of frame 0, then all N of frame 1, etc. —
matching `coord_index()` in `src/dcc_core.h`.

### What the synthetic model contains (so the demo is interpretable)

- **Geometry:** an extended single chain at 5 Å spacing. With the 8 Å contact
  cutoff, each residue contacts only its immediate sequence neighbors → the contact
  graph is a clean *path graph*, so the recovered pathway spans the whole chain.
- **Dynamics:** the chain is split into three "breathing" domains — an allosteric
  domain, a middle **hinge**, and the active domain. A single global collective
  coordinate drives all three along the same axis, so the two end domains move *in
  phase* (strongly correlated). The hinge couples only partially and carries its own
  independent wiggle, so the **domain-boundary edges are the weakest links** — that
  is why the reported bottleneck hop lands at the hinge.
- **Sites:** `SITE_ALLO = 2` and `SITE_ACTIVE = N-3`, one near each end, distant in
  both sequence and space — the hallmark geometry of allostery.

Regenerate (or resize) it with:

```bash
python scripts/make_synthetic.py                       # the committed 30×120 sample
python scripts/make_synthetic.py --residues 200 --frames 1000   # a bigger problem
```

## Full dataset (real allosteric trajectories)

Real allosteric MD trajectories are large and account-gated; `scripts/download_data.*`
prints instructions and **never** bypasses any registration. Sources (from the
catalog):

- **GPCRmd allosteric trajectory archive** — <https://gpcrmd.org> — GPCR MD
  trajectories (free, registration required).
- **MDAnalysis** test trajectories — <https://github.com/MDAnalysis/mdanalysis> —
  also the recommended tool to convert a real `.xtc/.dcd` + topology into this
  project's plain-text Cα format.
- **ProDy** benchmark structures/ensembles — <https://github.com/prody/ProDy> — the
  reference implementation of DCC/ANM-based allosteric analysis.
- **Allosteric Database (ASD)** — <http://mdl.shsmu.edu.cn/ASD/> — curated
  allosteric sites and modulators.

**Licensing:** respect each source's terms; none of that data is redistributed here.
The committed sample is synthetic precisely so the repo carries no third-party data.
