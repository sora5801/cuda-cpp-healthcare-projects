This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is **not real experimental
data** — it is a small "protein-like" point cloud plus a SAXS curve computed *from
that same cloud* with the Debye formula and 1% Gaussian noise, so the demo's fit is
meaningful and self-checking (the recovered Guinier `Rg` matches the structure's
true geometric `Rg`).

### File layout (parsed by `load_model()` in `../src/reference_cpu.cpp`)

```
# comment lines start with '#'
n_atoms  n_q  true_rg          # header: counts + the synthetic structure's true Rg (Å)
x  y  z  f                     # one line per atom (repeated n_atoms times)
...
q  I_exp  sigma                # one line per q-point (repeated n_q times)
...
```

| Symbol | Units | Meaning |
|---|---|---|
| `x, y, z` | Ångström (Å) | atom coordinates of the model structure |
| `f` | electrons | per-atom scattering strength (point-atom approximation; constant 6 e⁻ ≈ carbon) |
| `q` | 1/Å | momentum transfer, `q = 4π·sin(θ)/λ` |
| `I_exp` | arbitrary | synthetic "experimental" intensity = Debye(model) + noise |
| `sigma` | arbitrary | 1σ error bar on `I_exp` (here 1% of the noiseless intensity) |
| `true_rg` | Å | the model's exact scattering-weighted radius of gyration |

## Full / real datasets

Real SAXS data is **1-D scattering curves** `I(q)` (often with associated PDB models)
deposited in public banks. None are redistributed here; fetch them yourself:

- **SASBDB** — Small-Angle Scattering Biological Data Bank: <https://www.sasbdb.org>.
  Open-access curated SAXS/SANS curves + fitted models (`.dat` two/three-column
  `q I sigma` files — the same shape as our synthetic sample).
- **PDB / RCSB** — atomic models to forward-model: <https://www.rcsb.org> (`.pdb`/`.cif`).
- **BIOISIS** — SAXS benchmark database (verify the current URL before scripting).
- **Simulated SAXS from MD trajectories** — generate curves from your own structures.

`scripts/download_data.ps1` / `.sh` print these links and instructions; they never
bypass any registration. To convert a real `.pdb` into our text format you would
parse atom records, map each element to an electron count (or a proper `q`-dependent
form factor — see `THEORY.md`), and pair it with a `.dat` curve from SASBDB.

## Honesty / safety

The committed sample is **synthetic and labeled synthetic everywhere**. Nothing here
is a real measurement, and no output of this project is suitable for any structural-
biology conclusion about a real molecule, let alone any clinical use (CLAUDE.md §1, §8).
