## Committed sample (`sample/profile_db_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/profile_db_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (seed 2025) |
| License | Public domain (CC0) — it is fabricated, not derived from any database |
| Size | ~3 KB |
| Contents | one query **profile HMM** (L = 10 columns) + **24** database sequences |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). **Everything in it is synthetic and
carries no biological meaning** — it exists purely to exercise the Viterbi search.

### File format (parsed by `src/reference_cpu.cpp` → `load_problem`)

Whitespace-separated tokens; lines beginning with `#` are comments and skipped.

| Line(s) | Meaning |
|---|---|
| `L N` | profile length (match columns) and number of database sequences |
| 7 ints | transition log-odds `t_mm t_mi t_im t_ii t_md t_dm t_dd` (scaled ×1000) |
| next `L` lines | 21 emission log-odds per match column (20 amino acids in `ARNDCQEGHILKMFPSTWYV` order, then a catch-all `X` slot), scaled ×1000 |
| next `N` lines | one database sequence each, as amino-acid **letters** |

**Why scaled integers?** All log-odds are pre-multiplied by `SCORE_SCALE = 1000`
and rounded to integers (see `src/hmm_core.h`). The whole Viterbi dynamic program
is then integer max/add, so the CPU and GPU results are **bit-for-bit identical**
(no floating-point drift) and the demo's stdout is reproducible.

### The embedded "known answer"

The generator **plants** the query motif `WYGGFPKDEC` in the first three
sequences so the demo's result is verifiable (PATTERNS.md §6):

- `seq[0]`: clean motif → should be the **top** hit.
- `seq[1]`: motif with one point mutation → slightly lower.
- `seq[2]`: motif with a 2-residue insertion → lower still (insert-state cost).
- `seq[3..23]`: pure random "noise" sequences → low background scores.

Regenerate (deterministic) or scale up with:

```
python scripts/make_synthetic.py                 # default: L=10, N=24
python scripts/make_synthetic.py --n 256 --seed 7 # bigger / different draw
```

## Full dataset (real MSA databases)

Real MSA generation searches the query profile against enormous protein
databases. **None are committed here** (size + license); `scripts/download_data.*`
only prints pointers and never bypasses any registration:

| Database | What it is | Link |
|---|---|---|
| **UniRef90** | ~210 GB clustered UniProt sequences (the AF2 default) | <https://www.uniprot.org/help/uniref> |
| **UniClust30 / UniRef30** | clustered sequences for HHblits profiles | <https://uniclust.mmseqs.com> |
| **MGnify** | metagenomic protein sequences (deep MSAs) | <https://www.ebi.ac.uk/metagenomics/> |
| **BFD** | the "Big Fantastic Database" used by AlphaFold2 | <https://bfd.mmseqs.com> |

To run a *real* search you would build a profile HMM from your query with
HMMER/HHblits and scan one of the above; this project instead teaches the GPU
inner loop on a tiny synthetic stand-in.
