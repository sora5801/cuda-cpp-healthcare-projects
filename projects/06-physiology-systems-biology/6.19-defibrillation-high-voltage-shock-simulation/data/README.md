# Data — 6.19 Defibrillation & High-Voltage Shock Simulation

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/defib_sweep.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, not patient data |
| Size | < 1 KB |
| Layout | three whitespace-separated lines (see below) |

This tiny file is **not** patient data — it is the *setup* for a simulation: the
parameters of a 1-D cardiac cable and the ladder of shock amplitudes to sweep.
It lets `demo/run_demo` run **offline, with zero downloads** (a hard requirement,
CLAUDE.md §8).

### File format (parsed by `load_sweep` in `src/reference_cpu.cpp`)

```
line 1:  ncell nsteps dt dx D a eps gamma
line 2:  initial_excited shock_start shock_len biphasic success_thresh
line 3:  namp  a0 a1 a2 ...            (namp ascending shock amplitudes)
```

The committed sample is:

```
100 2000 0.1 1.0 0.6 0.13 0.008 1.0
30 800 10 0 0.01
10 0 0.05 0.1 0.15 0.2 0.25 0.3 0.4 0.6 1
```

### Per-field meaning (all quantities are DIMENSIONLESS — FitzHugh-Nagumo units)

| Field | Meaning |
|---|---|
| `ncell` | number of cells along the 1-D cable (100) |
| `nsteps` | number of forward-Euler time steps (2000) |
| `dt` | time step; must satisfy the diffusion stability limit `dt ≤ dx²/(2D)` (0.1) |
| `dx` | cell-to-cell spacing (1.0) |
| `D` | diffusion / gap-junction coupling coefficient (0.6) |
| `a` | FitzHugh-Nagumo excitation threshold, `0<a<1` (0.13) |
| `eps` | recovery-variable rate; small ⇒ slow recovery / long refractory period (0.008) |
| `gamma` | recovery coupling (sets the resting equilibrium) (1.0) |
| `initial_excited` | number of left-hand cells pre-excited to `V=1` — seeds the travelling wave the shock must terminate (30) |
| `shock_start` | time step the defibrillation shock turns on (800) |
| `shock_len` | shock duration in steps (10) |
| `biphasic` | `0` = monophasic (sample), `1` = polarity-reversing biphasic |
| `success_thresh` | residual-activity level below which a shock counts as "defibrillated" (0.01) |
| `namp`, `a…` | the ascending shock amplitudes to sweep (10 values, 0 … 1.0) |

Regenerate or retune with:

```
python scripts/make_synthetic.py                 # defaults (as committed)
python scripts/make_synthetic.py --biphasic 1     # try a biphasic protocol
python scripts/make_synthetic.py --D 0.4 --shock-len 20
```

## Full / real datasets

This project is a **reduced-scope teaching model** (a 1-D FitzHugh-Nagumo cable),
so it does not consume a real dataset. Real defibrillation research uses 3-D
bidomain meshes and recorded electrograms; the catalog points to these sources,
none of which are redistributed here — respect each one's license and access
terms (`scripts/download_data.*` prints links and instructions only, and never
bypasses credentials):

Catalog dataset notes (verbatim):

> PhysioNet fibrillation/defibrillation recordings (https://physionet.org); openCARP defibrillation tutorial cases (https://opencarp.org); Cardioid (https://github.com/llnl/cardioid) — bidomain shock examples; patient-specific ICD placement datasets (verify institutional access).

- **PhysioNet** recordings: openly available but governed by PhysioNet's data-use
  agreement; register and download from the site, do not commit the raw files.
- **openCARP / Cardioid / Chaste** meshes and tutorial cases: open-source project
  data under their own licenses — fetch from the upstream repos as needed.
- **Patient-specific ICD placement** data: institutional / IRB access only; never
  redistributed. Use the synthetic sample instead.

## Safety / honesty

Everything here is **synthetic and educational**. The FitzHugh-Nagumo cable is a
qualitative caricature of cardiac excitation, not a validated model of any
patient's heart. Nothing in this project may be used for diagnosis, device
programming, or any clinical decision (CLAUDE.md §1, §8).
