# Data — 6.14 Multi-Scale Physiological Modeling

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/cable.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no real patient data |
| Size | < 1 KB (a single line of numbers) |
| Layout | one line: `n dx dt steps stim_nodes a eps b D` |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **not** measured data — it is
the *configuration* of a self-contained multi-scale toy simulation (a 1-D cardiac
cable). Everything downstream is deterministic given this one line.

### Field meanings (units in brackets)

| Field | Meaning |
|---|---|
| `n`          | number of tissue nodes on the cable (mesh resolution) |
| `dx`         | node spacing [space units] |
| `dt`         | global (operator-split) time step [time units] |
| `steps`      | number of global steps (total time = `steps*dt`) |
| `stim_nodes` | the first this-many left-end nodes are held excited (`v=1`) at `t=0` to launch the action potential |
| `a`          | FitzHugh-Nagumo excitation threshold [dimensionless] |
| `eps`        | FHN recovery time-scale [dimensionless] (`eps<<1` = slow recovery) |
| `b`          | FHN recovery coupling [dimensionless] |
| `D`          | tissue diffusion coefficient [space²/time] (larger = faster conduction) |

The committed values (`128 0.5 0.02 5000 5 0.13 0.005 0.5 2`) are tuned so the
left-end stimulus launches a **self-sustaining traveling wave** that crosses the
whole cable — the phenomenon the demo recovers (an increasing activation map and
a well-defined conduction velocity). The explicit-diffusion stability number
`D·dt/dx² = 0.16 ≤ 0.5`, so the forward-Euler diffusion step is stable.

Regenerate or resize with:

```bash
python scripts/make_synthetic.py                    # default demo cable
python scripts/make_synthetic.py --n 512 --steps 20000   # a longer/finer cable
```

## Full dataset (real multi-scale physiology models)

This teaching project is **simulation-only**, so there is no measured dataset to
download to run it. The catalog points to the real-world model repositories that
a production Virtual Physiological Human (VPH) workflow would draw its cell and
tissue models from:

- **Physiome Model Repository** — VPH-standard CellML cell models:
  <https://models.physiomeproject.org>
- **BioModels Database** — curated systems-biology / ODE models:
  <https://www.ebi.ac.uk/biomodels>
- **OpenCMISS examples** — multi-scale FEM problem setups:
  <https://github.com/OpenCMISS/examples>
- **UK Biobank** — multi-modal phenotyping (credentialed; see below):
  <https://www.ukbiobank.ac.uk>

`scripts/download_data.ps1` / `.sh` print pointers to these; they do **not** try
to bypass any registration (UK Biobank in particular requires an approved
application — the script only links to it). Because none of these are needed for
the demo, the committed sample stays synthetic.

Catalog dataset notes (verbatim):

> Physiome Model Repository — VPH-standard CellML models (https://models.physiomeproject.org); BioModels Database (https://www.ebi.ac.uk/biomodels); UK Biobank multi-modal phenotyping (https://www.ukbiobank.ac.uk); OpenCMISS examples (https://github.com/OpenCMISS/examples).

## Honesty & safety

The sample is **synthetic** and labeled as such everywhere it appears. The FHN
cell model and the 1-D cable are **didactic reductions** of real cardiac
electrophysiology (ten-Tusscher / O'Hara-Rudy cell models on a 3-D FEM mesh);
the numbers here have **no clinical meaning** and must not be used for diagnosis
or treatment (CLAUDE.md §8).
