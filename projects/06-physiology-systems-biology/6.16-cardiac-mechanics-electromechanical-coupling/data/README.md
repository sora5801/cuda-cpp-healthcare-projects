The "data" is the **model setup**, not measured input: each virtual heart's two
swept parameters are derived from the sweep grid, so the whole ensemble is fully
reproducible. This tiny file lets `demo/run_demo` run **offline, with zero
downloads** (CLAUDE.md §8).

### File format (whitespace-separated, in this block order)

```
t_activate
Ca_rest  Ca_amp  tau_rise  tau_decay
Tref_baseline  Ca50  nH  k_xb
Emin  V0
R_ao  R_mv  P_ven
R_sys_baseline  C_art  P_art_dias
dt  steps_per_beat  n_beats
nT  nR  Tref_lo  Tref_hi  R_lo  R_hi
```

| Field | Meaning | Units |
|---|---|---|
| `t_activate` | time within a beat at which the cell fires | ms |
| `Ca_rest`, `Ca_amp` | diastolic calcium; peak calcium added by release | µM |
| `tau_rise`, `tau_decay` | calcium upstroke / re-uptake time constants (need `rise < decay`) | ms |
| `Tref_baseline` | baseline active-elastance scale (**overridden** by the sweep) | mmHg/mL |
| `Ca50`, `nH` | Hill half-activation calcium and cooperativity | µM, — |
| `k_xb` | cross-bridge recruitment/relaxation rate | 1/ms |
| `Emin`, `V0` | diastolic (passive) elastance; unloaded chamber volume | mmHg/mL, mL |
| `R_ao`, `R_mv` | aortic (ejection) and mitral (filling) valve resistances | mmHg·ms/mL |
| `P_ven` | venous filling (preload) pressure | mmHg |
| `R_sys_baseline` | baseline systemic vascular resistance = afterload (**overridden** by sweep) | mmHg·ms/mL |
| `C_art`, `P_art_dias` | arterial (Windkessel) compliance; diastolic arterial floor | mL/mmHg, mmHg |
| `dt`, `steps_per_beat`, `n_beats` | RK4 timestep; steps per cardiac cycle; beats (warm-up + 1 recorded) | ms, —, — |
| `nT`, `nR` | sweep size: `nT` contractility × `nR` afterload = `nT·nR` hearts | — |
| `Tref_lo..hi` | contractility (active-elastance) sweep range | mmHg/mL |
| `R_lo..hi` | afterload (`R_sys`) sweep range | mmHg·ms/mL |

Default sample: `dt=0.1`, `8000` steps/beat (≈ 800 ms ≈ 75 bpm), `10` beats, a
`6×6 = 36`-heart sweep with contractility `1.5..3.5` mmHg/mL and afterload
`0.7..2.2` mmHg·ms/mL. That spread produces ejection fractions from ~36% (a weak
/ failing ventricle) to ~65% (healthy) — a clean, interpretable teaching result.

Enlarge it: `python scripts/make_synthetic.py --nT 32 --nR 32` (1024 hearts).

## "Full dataset" / realistic models

There is **no raw dataset to download** for this reduced-scope teaching version
(the model is self-contained). Real cardiac-electromechanics work is driven by:

- **UK Biobank CMR + strain imaging** (<https://www.ukbiobank.ac.uk>) — geometry
  and motion to fit patient-specific mechanics (registration required).
- **Zenodo cardiac-mechanics emulation dataset** (<https://zenodo.org/records/7075055>).
- **ACDC segmentation challenge** (<https://www.creatis.insa-lyon.fr/Challenge/acdc/>).
- **MICCAI STACOM cardiac-mechanics challenge** (verify URL on grand-challenge.org).

`scripts/download_data.*` prints these pointers and never bypasses any account
or license.

## Provenance & honesty

The configuration is **synthetic**; every parameter is illustrative and in a
physiological ballpark, **not fitted to any patient**. Outputs (PV loops,
ejection fraction, wall-stress proxy) are a software demonstration of
electromechanical coupling and ensemble ODE integration, **not** a diagnosis,
forecast, or any clinically valid quantity.
