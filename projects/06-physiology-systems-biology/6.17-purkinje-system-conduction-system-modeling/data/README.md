| Not clinical | A didactic toy tree; **not** an anatomically calibrated model |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is deliberately
engineered so the result is interpretable: the diffusion coefficient `D` varies
across cables, so the demo shows a clear spread of conduction velocities, and one
branch (cable 2, `D=1.5`) is deliberately slow.

### File format

```
line 1 : N  dt_ms  n_steps                       (global integration clock)
next N : n_nodes length_mm D stim_amp stim_dur_ms stim_width thresh parent delay_ms
```

Per-cable field meanings:

| Field | Meaning | Units |
|---|---|---|
| `n_nodes` | spatial grid points along the cable (2 … `PK_MAX_NODES`=256) | — |
| `length_mm` | physical fibre length | mm |
| `D` | diffusion (axial) coefficient; ∝ fibre diameter; larger → faster CV | mm²/ms |
| `stim_amp` | pacing-electrode current added to the proximal nodes | dimensionless V/ms |
| `stim_dur_ms` | how long the stimulus is held | ms |
| `stim_width` | how many proximal nodes are stimulated | — |
| `thresh` | activation threshold used to timestamp the front | dimensionless V |
| `parent` | index of the parent cable in the tree (`-1` = His-bundle root) | — |
| `delay_ms` | fixed gap-junction / PMJ delay to *enter* this cable | ms |

`parent` must be `< index` so the tree is topologically ordered (a single
forward pass computes activation times). The transmembrane potential `V` is the
nondimensional Aliev-Panfilov variable in `[0,1]`; `D` and `dt_ms` are chosen to
satisfy the explicit-stability limit `dt ≤ dx² / (2 D)` (see `THEORY.md`).

## Full / real datasets

This project ships a **reduced-scope teaching model**, so no real dataset is
required to run it. To go further with genuine Purkinje geometries and
electrograms (see `scripts/download_data.*` for links and instructions — the
script prints pointers and never bypasses any registration):

- **openCARP community Purkinje experiments** — https://opencarp.org/community/community-experiments
- **MonoAlg3D_C Purkinje examples** — https://github.com/rsachetto/MonoAlg3D_C
- **NeuroMorpho** (morphological analogy for branching-tree datasets) — https://neuromorpho.org
- **PhysioNet** His-bundle electrogram databases — https://physionet.org

Catalog dataset notes (verbatim):

> openCARP community Purkinje experiments (https://opencarp.org/community/community-experiments); MonoAlg3D_C Purkinje examples (https://github.com/rsachetto/MonoAlg3D_C); NeuroMorpho (morphological analogy for tree datasets) (https://neuromorpho.org); PhysioNet His-bundle electrogram databases (https://physionet.org).

## Provenance & honesty

The committed sample is **synthetic**, generated deterministically by
`scripts/make_synthetic.py`. It is labeled synthetic everywhere it appears and
carries no clinical validity. Real Purkinje meshes are large research artefacts;
adapting this loader to read a real geometry (edges + per-segment diameters) is
left as an exercise in the project `README.md`.
