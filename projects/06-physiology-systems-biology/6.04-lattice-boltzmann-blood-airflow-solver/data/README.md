# Data — 6.04 Lattice-Boltzmann Blood/Airflow Solver

## Committed sample (`sample/channel_params.txt`)

| Field | Value |
|---|---|
| Origin | **Synthetic** simulation parameters (`scripts/make_synthetic.py`) |
| License | Public domain (CC0) |
| Setup | Periodic channel, no-slip walls (bounce-back), body-force-driven Poiseuille flow |

This project's "data" is the **simulation setup**, not measured input — the flow
is generated by the solver. The geometry here is the simplest physiological-flavor
case (a straight channel); real blood/airflow uses a vessel/bronchial geometry.

### File format (one line)

```
nx  ny  steps  tau  gx
```

| Field | Meaning |
|---|---|
| `nx` | lattice width (flow direction; periodic) |
| `ny` | lattice height (across the channel; walls top & bottom) |
| `steps` | number of collide+stream iterations |
| `tau` | BGK relaxation time; viscosity `nu=(tau-0.5)/3`, must be `>0.5` |
| `gx` | body force per unit mass in +x (keep velocities `<< 1` for stability) |

Default sample: `16 24 6000 0.8 0.00001` → develops a parabolic profile, `u_max ≈ 0.0072`.

## "Full dataset" / realistic geometry

Real LBM hemodynamics/airflow uses a 3-D voxelized vessel or bronchial-tree
geometry (segmented from CT/MRI) and a D3Q19/D3Q27 stencil. Reference frameworks:

- **HemeLB** (<https://github.com/hemelb-codes/hemelb>) — sparse-geometry vascular LBM.
- **PALABOS** (<https://gitlab.com/unigespc/palabos>) — full C++ LBM framework.
- **USERMESO-2.0** (<https://github.com/AnselGitAccount/USERMESO-2.0>) — GPU red-blood-cell hemodynamics.

Bigger 2-D grid: `python scripts/make_synthetic.py --nx 128 --ny 64 --steps 20000`.

## Provenance & honesty

The setup is **synthetic** and the geometry is a **simplified 2-D straight
channel** (not a real vessel). It demonstrates the LBM/GPU pattern; it is not a
validated hemodynamics simulation and is not for any clinical use.
