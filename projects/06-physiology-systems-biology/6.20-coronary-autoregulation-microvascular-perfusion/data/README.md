## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/coronary_network.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, deterministic — no RNG) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB |
| Contents | An 8-node, 8-segment coronary microvascular network with one stenosis |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is a hard requirement for
every project (CLAUDE.md §8). It is **synthetic teaching data — not patient data and not clinically valid.**

### File format

Whitespace-delimited, line-oriented; lines starting with `#` are comments. Parsed by
`src/reference_cpu.cpp::load_network`.

```
# header
n_nodes  n_segs  hct  aortic_p
# then n_nodes lines, one per node id 0..n_nodes-1:
is_fixed  fixed_pressure_mmHg
# then n_segs lines, one per segment:
a  b  radius_um  length_um  target_flow
# finally one line for the FFR read-out:
ffr_seg  ffr_prox  ffr_dist
```

**Field meanings (units):**

| Field | Meaning | Units |
|---|---|---|
| `n_nodes`, `n_segs` | count of junction nodes / vessel segments | — |
| `hct` | hematocrit fraction (feeds the Fåhræus–Lindqvist viscosity) | fraction 0–1 |
| `aortic_p` | inlet (aortic) pressure, used to normalize FFR | mmHg |
| `is_fixed` | 1 = Dirichlet boundary node (prescribed pressure); 0 = interior | flag |
| `fixed_pressure_mmHg` | prescribed pressure at a fixed node (ignored if interior) | mmHg |
| `a`, `b` | endpoint node indices of the segment | node id |
| `radius_um`, `length_um` | segment radius and length | µm |
| `target_flow` | metabolic set-point flow the autoregulation drives toward | µm³/s |
| `ffr_seg` | segment index treated as the stenosis for the FFR read-out | segment id |
| `ffr_prox`, `ffr_dist` | proximal / distal nodes across the stenosis | node id |

### What the sample encodes (engineered to be interpretable)

- Node 0 is the **aortic inlet** (fixed at 100 mmHg); nodes 6 and 7 are **venous outlets** (fixed at 20 mmHg).
- Segment 2 (node 1→3) is a **stenosis** (radius 8 µm vs. 20 µm on the healthy branch). Because conductance
  ∝ r⁴, this branch is resistance-limited, so the computed virtual FFR across it comes out flow-limiting
  (`< 0.80`). Segment 7 is a thin collateral cross-link. The two territories are deliberately asymmetric so
  the stenosis is visible in the nodal pressures.

Regenerate with `python scripts/make_synthetic.py` (writes exactly this file).

## Full dataset

This teaching project runs entirely on the synthetic sample. Real coronary geometries/waveforms come from
the catalog's datasets; `scripts/download_data.ps1` / `.sh` print how to obtain them (they never bypass
credentialed access):

- **UK Biobank coronary CTA (subset)** — <https://www.ukbiobank.ac.uk> — requires an approved application;
  the script prints instructions only. License forbids redistribution ⇒ our sample is synthetic.
- **PhysioNet coronary pressure/flow waveforms** — <https://physionet.org> — some sets require
  credentialing + a data use agreement.
- **Vascular Model Repository** — <http://www.vascularmodel.com> — open cardiovascular geometries usable to
  build a real network (extract centerlines + radii into this file's format).
- **MICCAI coronary artery tracking challenges** — <https://grand-challenge.org> — registration required.

To use real data, convert a centerline+radius model into the `coronary_network.txt` format above (nodes =
branch points, segments = centerline edges with mean radius/length), pin the inlet and outlets, and run.

## Provenance & safety

- The committed sample is **100% synthetic** and labeled as such everywhere (header comment lines in the
  file, this README, `make_synthetic.py`). It contains no patient-derived information.
- **No clinical validity.** The virtual FFR and perfusion numbers are didactic outputs on invented geometry;
  they must never be used for diagnosis or treatment (CLAUDE.md §8).
