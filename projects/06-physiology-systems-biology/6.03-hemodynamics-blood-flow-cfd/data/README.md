# Data — 6.3 Hemodynamics / Blood-Flow CFD

## Committed sample (`sample/channel_params.txt`)

| Field | Value |
|---|---|
| File | `sample/channel_params.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (no patient data) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB |
| Layout | one line: `nx ny steps p_iters h dt rho gx nu0 nu_inf lambda n_cy a_cy` |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **not** patient data — it is a
set of parameters for a synthetic straight-channel flow that the solver drives
toward the analytic **Poiseuille** solution, giving the demo a known, checkable
answer.

### Field meanings (dimensionless "lattice" units)

Units are kept dimensionless so the printed numbers are clean; THEORY.md §2 shows
how to map them to SI (cm, s, Pa) for real blood.

| Field | Meaning | Sample value |
|---|---|---|
| `nx` | grid cells in the streamwise (flow) direction | 32 |
| `ny` | grid cells across the channel (odd → a true centre row) | 17 |
| `steps` | number of fractional-step time steps | 4000 |
| `p_iters` | Jacobi iterations per pressure Poisson solve | 40 |
| `h` | uniform cell spacing (x and y) | 1 |
| `dt` | time-step size (obeys `dt < h²/(4·ν)` stability limit) | 0.02 |
| `rho` | fluid density | 1 |
| `gx` | streamwise body force (stand-in for the driving `dp/dx`) | 1e-4 |
| `nu0` | Carreau-Yasuda zero-shear kinematic viscosity | 0.1 |
| `nu_inf` | Carreau-Yasuda infinite-shear viscosity (`==nu0` ⇒ **Newtonian**) | 0.1 |
| `lambda` | Carreau-Yasuda relaxation time | 1 |
| `n_cy` | power-law index (`<1` ⇒ shear thinning; ignored when Newtonian) | 0.5 |
| `a_cy` | Yasuda transition exponent | 2 |

The sample is **Newtonian** (`nu0 == nu_inf`) on purpose: only then does the exact
Poiseuille parabola apply, so the demo can check the science, not just CPU==GPU.
Enable shear thinning (blood's real behaviour) with, e.g.,
`python scripts/make_synthetic.py --nu-inf 0.03`.

## Full / real datasets (from the catalog)

The real, research-grade version of this project consumes patient-specific
geometries and physiological waveforms. These are **credential-gated or
license-restricted** and are **not** redistributed here; `scripts/download_data.*`
print instructions and links only (never bypassing registration, CLAUDE.md §8):

- **Vascular Model Repository** — patient-specific vascular geometries (aorta,
  coronary, cerebral): <http://www.vascularmodel.com>
- **PhysioNet MIMIC-III waveforms** — invasive pressure/flow recordings
  (credentialed): <https://physionet.org/content/mimiciii/1.4/>
- **Zenodo Cardiac Mechanics Emulation dataset**:
  <https://zenodo.org/records/7075055>
- **UK Biobank aortic 4D-flow MRI subset** (application required):
  <https://www.ukbiobank.ac.uk>

Turning those into a runnable simulation requires the full image-to-mesh pipeline
(SimVascular/svFSI) — far beyond this teaching project's scope. See THEORY.md
"Where this sits in the real world".

## Provenance & honesty

Everything in `sample/` is **synthetic** and labelled as such. No output of this
project is clinically valid or usable for diagnosis or treatment (CLAUDE.md §1, §8).
