# Data — 6.2 Whole-Heart Digital Twin

> **Reduced-scope teaching version.** This project simulates a spatially-lumped
> (0-D) closed-loop heart from a tiny **synthetic** parameter file. It uses **no
> patient images**. The real-world datasets a full twin is built from are listed
> below for study; none are required to build or run the demo.

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/heart_ensemble.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB |
| Layout | one whitespace-separated record (see below) |

This one-line file lets `demo/run_demo` run **offline, with zero downloads** — a
hard requirement for every project (CLAUDE.md §8).

### Field meanings (in on-disk order)

```
n emax_lo emax_hi dt_ms beats target_sv bcl_ms E_min V0 Rp C_art
```

| Field | Meaning | Units |
|---|---|---|
| `n`         | number of virtual hearts in the ensemble (the contractility sweep) | — |
| `emax_lo`   | lowest systolic elastance (contractility) in the sweep | mmHg/mL |
| `emax_hi`   | highest systolic elastance in the sweep | mmHg/mL |
| `dt_ms`     | RK4 integration timestep | ms |
| `beats`     | cardiac cycles simulated (transient wash-out + measurement) | — |
| `target_sv` | **synthetic** clinical target stroke volume the twin is fit to | mL |
| `bcl_ms`    | basic cycle length / beat period (800 ms = 75 bpm) | ms |
| `E_min`     | diastolic (relaxed) elastance | mmHg/mL |
| `V0`        | unstressed ventricular volume | mL |
| `Rp`        | peripheral (systemic) resistance | mmHg·s/mL |
| `C_art`     | arterial compliance | mL/mmHg |

The committed sample is `12 1.2 3.4 0.1 6 70 800 0.06 10 1 1.6`: a 12-member
contractility sweep from 1.2 to 3.4 mmHg/mL, six 800-ms beats at a 0.1-ms step,
fit to a synthetic target stroke volume of 70 mL. The remaining physiology
(FitzHugh-Nagumo constants, valve resistances, characteristic resistance) uses
the documented defaults in `src/heart.h::HeartParams`.

Regenerate or resize it:

```bash
python scripts/make_synthetic.py --n 256 --emax-lo 1.0 --emax-hi 4.0
```

## Full / real-world datasets

A *real* patient-specific whole-heart twin is built from imaging (for geometry
and fiber orientation) and clinical measurements (for calibration targets).
`scripts/download_data.{ps1,sh}` prints these links; **none are fetched
automatically** and most require registration (CLAUDE.md §8 forbids bypassing
credentials):

| Dataset | What it provides | Access |
|---|---|---|
| [UK Biobank Cardiac MRI](https://www.ukbiobank.ac.uk) | 100 000+ cine CMR (geometry, function targets) | Application required |
| [Zenodo Synthetic Biventricular Heart Meshes](https://zenodo.org/records/4506930) | 1 000 virtual-cohort tetrahedral meshes | Open (CC-BY) |
| [Visible Human Project](https://www.nlm.nih.gov/research/visible/visible_human.html) | Full-body CT/MRI/cryosection | License/registration |
| [ACDC MICCAI](https://www.creatis.insa-lyon.fr/Challenge/acdc/) | 100-patient CMR segmentations | Registration |

## Provenance & honesty

- The committed sample is **synthetic** and is labeled synthetic everywhere it
  appears (file, README, program stdout).
- The model outputs (stroke volume, ejection fraction, pressures) are the result
  of a simplified 0-D ODE model on made-up parameters. They are **not** derived
  from any real patient and **must not** be used for diagnosis or treatment.
