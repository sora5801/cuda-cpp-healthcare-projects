## Committed sample (`sample/tissue_params.txt`)

| Field | Value |
|---|---|
| Origin | **Synthetic** simulation parameters (`scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB (one line) |
| Setup | Tumor nodule at the domain centre secreting a chemokine; immune cells scattered around it, chemotaxing inward |

This project's "data" is the **simulation setup**, not measured input — the
tissue state (cell positions, chemokine field) is generated by the solver. The
individual cell positions are placed **deterministically** inside the C++ loader
from the `seed` field (a fully-specified LCG, not `<random>`), so the whole run
is reproducible from this one line on any machine.

### File format (one line, 14 whitespace-separated fields)

```
gx gy dx steps dt D decay secretion radius k_rep chemotaxis seed n_tumor n_immune
```

| Field | Meaning |
|---|---|
| `gx`, `gy` | substrate (chemokine) grid dimensions in cells |
| `dx` | grid spacing (domain units per grid cell); domain is `gx*dx` × `gy*dx` |
| `steps` | number of timesteps (secrete → diffuse → move) |
| `dt` | timestep |
| `D` | chemokine diffusion coefficient (needs `D*dt/dx^2 ≤ 0.25` for stability) |
| `decay` | first-order chemokine decay rate |
| `secretion` | chemokine secreted per tumor cell per unit time |
| `radius` | cell radius; soft-sphere repulsion acts within `2*radius` |
| `k_rep` | repulsion stiffness (Hookean overlap spring) |
| `chemotaxis` | immune migration speed per unit chemokine gradient |
| `seed` | RNG seed for the deterministic initial cell layout |
| `n_tumor` | number of tumor cells (secretors, mechanically passive) |
| `n_immune` | number of immune cells (chemotactic) |

Default sample: `32 32 1.0 300 0.1 0.4 0.02 1.0 0.6 1.0 2.0 12345 40 30`
→ a chemokine plume forms over the tumor and immune cells migrate inward, so the
**mean immune→tumor distance shrinks** over the run.

## "Full dataset" / calibrating against real data

There is nothing to download to run the demo. To *calibrate* an agent-based
model against real tissue/immune measurements, the catalog points to (each needs
its own registration / license — `scripts/download_data.*` prints the links and
does **not** bypass credentials):

- **CancerSEA** single-cell functional states — <http://biocc.hrbmu.edu.cn/CancerSEA/>
- **TCGA** pan-cancer immune landscape — <https://portal.gdc.cancer.gov>
- **MIBI/IMC** imaging mass cytometry — various Zenodo deposits
- **TCIA** immunotherapy imaging — <https://www.cancerimagingarchive.net>

Bigger synthetic scenario:
`python scripts/make_synthetic.py --gx 64 --gy 64 --n-tumor 400 --n-immune 300 --steps 800`.

## Provenance & honesty

The setup is **synthetic** and the model is a **reduced-scope 2-D teaching
version** of PhysiCell-style agent-based tissue simulation (soft-sphere
mechanics + one diffusing substrate + chemotaxis). It demonstrates the
hybrid GPU pattern (atomic scatter + stencil PDE + spatial-binning neighbour
search); it is **not** a validated tumor-immune model and is **not for any
clinical use**.
