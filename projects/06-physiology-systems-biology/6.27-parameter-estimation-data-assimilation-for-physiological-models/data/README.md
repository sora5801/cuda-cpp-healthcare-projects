# Data — 6.27 Parameter Estimation & Data Assimilation for Physiological Models

## Committed sample (`sample/enkf_config.txt`)

| Field | Value |
|---|---|
| File | `sample/enkf_config.txt` |
| Origin | **Synthetic** twin-experiment config (`scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | < 1 KB (a single line) |
| Contents | assimilation settings + a KNOWN true patient + a prior |

The "data" here is **not a measured waveform** — it is the *configuration* of a
synthetic **twin experiment**. The program uses the true `(R, C)` in this file to
generate a noisy pressure waveform internally, then runs the Ensemble Kalman Filter
to try to recover those parameters. This makes the demo self-contained, offline, and
verifiable (we can score the estimate against the known truth). Everything is
**synthetic**; nothing derives from a real patient.

### File format (one whitespace-separated line)

```
m  n_obs  dt  substeps  T  t_sys  Q_peak  R_true  C_true  P0  obs_noise  R_prior  C_prior  logR_std  logC_std  seed
```

| Field | Meaning | Units |
|---|---|---|
| `m` | ensemble size (number of members) | — |
| `n_obs` | number of observation windows to assimilate | — |
| `dt` | RK4 sub-step | s |
| `substeps` | RK4 sub-steps per window (window length = `substeps·dt`) | — |
| `T` | cardiac cycle length (0.8 s = 75 bpm) | s |
| `t_sys` | systolic ejection duration | s |
| `Q_peak` | peak ventricular inflow | mL/s |
| `R_true` | **true** peripheral resistance (target to recover) | mmHg·s/mL |
| `C_true` | **true** arterial compliance (target to recover) | mL/mmHg |
| `P0` | initial aortic pressure | mmHg |
| `obs_noise` | std dev of measurement noise on pressure | mmHg |
| `R_prior` | prior mean guess for `R` (deliberately off) | mmHg·s/mL |
| `C_prior` | prior mean guess for `C` (deliberately off) | mL/mmHg |
| `logR_std` | prior std dev of `log R` (initial ensemble spread) | — |
| `logC_std` | prior std dev of `log C` (initial ensemble spread) | — |
| `seed` | RNG seed → fully reproducible ensemble & noise | — |

Default sample: `256 40 0.005 16 0.8 0.3 500.0 1.0 1.5 80.0 1.0 1.4 1.0 0.3 0.3 20260701`
→ 256 members, 40 windows of 0.08 s, true `(R,C) = (1.0, 1.5)`, prior `(1.4, 1.0)`.

Regenerate or rescale: `python scripts/make_synthetic.py --ensemble 1024 --windows 80`.

## Full dataset

Real patient-specific calibration uses measured clinical waveforms and cardiac
parameters. All of the catalog's sources require **registration / credentialed
access**, so this project ships a synthetic stand-in and `scripts/download_data.*`
only prints pointers (it never bypasses a credential wall — CLAUDE.md §8):

- **PhysioNet MIMIC** clinical waveforms — <https://physionet.org> (credentialed).
- **UK Biobank** cardiac functional parameters — <https://www.ukbiobank.ac.uk> (application).
- **Zenodo** cardiac mechanics emulation dataset — <https://zenodo.org/records/7075055>.
- **openCARP** community experiments — <https://opencarp.org/community/community-experiments>.

## Provenance & honesty

The sample is **synthetic** and labeled synthetic everywhere it appears. The true
`(R, C)`, the inflow waveform, and the measurement noise are all generated by the
program from this config; no output implies clinical validity. This is a software
demonstration of ensemble data assimilation, not a patient measurement or a
diagnostic tool.
