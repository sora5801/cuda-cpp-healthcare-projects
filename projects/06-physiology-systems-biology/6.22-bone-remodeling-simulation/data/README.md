| Field | Value |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It contains no measurements
of any kind — it is just the set of **dimensionless knobs** for the mechanostat
remodeling model in `src/`. It implies **no clinical validity** whatsoever.

### Field-by-field meaning (the loader reads them in this exact order)

`src/reference_cpu.cpp::load_bone` reads these with `operator>>`, so the file is a
pure list of numbers (comments live here, not in the file):

| # | Field | Sample value | Meaning / units |
|---|-------|-------------|-----------------|
| 1 | `nx` | 24 | grid columns (x, across the load) |
| 2 | `ny` | 16 | grid rows (y, along the load; **row 0 = loaded top**, row `ny-1` = supported base) |
| 3 | `remodel_steps` | 60 | number of remodeling iterations (each loosely ~ "one month") |
| 4 | `relax_iters` | 80 | Jacobi sweeps per step used to settle the mechanical-stimulus field |
| 5 | `load` | 4.0 | mechanical stimulus injected under the loaded footprint (dimensionless, >0) |
| 6 | `load_x0` | 10 | first column of the loaded footprint on the top edge (inclusive) |
| 7 | `load_x1` | 13 | last column of the loaded footprint on the top edge (inclusive) |
| 8 | `setpoint` | 0.55 | homeostatic target `k` for the remodeling signal `phi = S/rho` |
| 9 | `lazy` | 0.20 | half-width `w` of the lazy zone (dead band) around `k` |
| 10 | `rate` | 0.05 | remodeling gain (how fast density chases the stimulus) |
| 11 | `rho_min` | 0.05 | density floor (bone never fully vanishes; keeps `S/rho` finite) |
| 12 | `rho_init` | 0.50 | uniform initial density everywhere (the "unremodeled" blank) |

The chosen values encode a **compression specimen with a localized load** — a
joint or implant contact patch pressing on the center of the top edge (columns
10–13), base supported. Over the run the mechanostat carves an **oriented
trabecular strut down the load path** and resorbs the lightly-loaded flanks; the
per-column mass profile in the demo output peaks under the footprint. This is a
qualitative illustration of **Wolff's law**, not a calibrated tissue model.

## Full / real datasets

There is **no single canonical dataset** for this teaching model — it runs on the
synthetic parameter file above. The catalog lists real bone-imaging datasets you
could adapt a production voxel-FEM pipeline to (segment a µCT stack into a bone /
marrow voxel mask, then drive remodeling on that geometry). `scripts/download_data.*`
print the pointers below and never bypass any registration:

- **OsteoArthritis Initiative (OAI)** µCT + radiographs — <https://nda.nih.gov/oai/>
  (registration required; do not attempt to bypass credentials).
- **PhysioNet** bone-related datasets — <https://physionet.org> (credentialed use
  agreement for some collections).
- **BoneJ** morphometric example datasets — <https://bonej.org>.
- **MICCAI** bone-segmentation challenge data — <https://grand-challenge.org>.

Respect every dataset's license. If a license forbids redistribution, do **not**
commit the data — the committed sample here stays synthetic (CLAUDE.md §8).
