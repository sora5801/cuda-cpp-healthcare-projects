This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is

imply any clinical validity.

### File format

Whitespace-separated tokens; lines beginning with `#` are comments (ignored by the
loader in `src/reference_cpu.cpp`). The layout is:

```
# comment lines...
nx ny nz spacing po2_inflow m0 km p50 hill_n n_src
# then one line per capillary segment:
x  y  z  blood_po2
...
```

| Token | Meaning | Units |
|---|---|---|
| `nx ny nz` | tissue-grid dimensions (points per axis) | — |
| `spacing` | grid spacing (points are `spacing` apart, origin at 0,0,0) | um |
| `po2_inflow` | baseline / arterial tissue PO2 the field starts from | mmHg |
| `m0` | maximal Michaelis-Menten O2 consumption rate | sink strength |
| `km` | Michaelis constant (PO2 at half-maximal consumption) | mmHg |
| `p50` | Hill half-saturation PO2 of hemoglobin | mmHg |
| `hill_n` | Hill cooperativity coefficient | — |
| `n_src` | number of capillary-segment sources that follow | — |
| `x y z` | position of a capillary segment (source) | um |
| `blood_po2` | blood PO2 in that segment (sets its O2 source strength via the Hill curve) | mmHg |

### What the sample encodes (an engineered "known answer")

Three straight capillaries run along the x-axis at low-to-mid (y, z), each sampled
into 6 segments whose blood PO2 falls from ~95 mmHg (arterial inlet) to ~45 mmHg
(venous outlet). They deliberately leave the **far corner** (max x, max y, max z)
distant from every vessel — an engineered **hypoxic pocket**. The demo's reported
`min PO2`, `hypoxic fraction`, and `far-corner` sample recover this on purpose
(this is the "embed a known answer" idea from `docs/PATTERNS.md` §6).

Regenerate or resize it:

```bash
python scripts/make_synthetic.py                       # the committed default
python scripts/make_synthetic.py --nx 24 --ny 24 --nz 16   # a bigger block
```

## Full / real datasets

There is no single canonical "microvascular oxygen-transport" input; real studies
build the geometry from imaged vascular networks and the physiology from the
literature. `scripts/download_data.ps1` / `.sh` print pointers and instructions but
**never** bypass any registration. Candidate sources (from the catalog):

- **Vascular Model Repository** — <http://www.vascularmodel.com> — vascular
  geometries (CFD-oriented); check per-model license.
- **Allen Institute two-photon microscopy** microvascular datasets —
  <https://portal.brain-map.org> — imaged cortical microvasculature.
- **PhysioNet** oxygen-saturation waveforms — <https://physionet.org> — requires a
  credentialed account for some collections; respect the data-use agreement.
- **Secomb group** published microvascular networks — verify at
  <https://secomb.org> — the reference Green's-function networks; check terms.

Respect every license. Where redistribution is forbidden or credentials are
required, the committed sample stays **synthetic** and `make_synthetic.py` provides
a stand-in so the project still runs.

## Honesty note

The committed data is synthetic and the model is a **reduced-scope teaching
version** of the Green's-function O2 transport method (see `../THEORY.md`). It is
for learning the GPU pattern and the physiology, not for any diagnostic or
therapeutic use.
