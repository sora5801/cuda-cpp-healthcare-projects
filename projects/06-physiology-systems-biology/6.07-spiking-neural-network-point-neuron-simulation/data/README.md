There is **no patient data here at all.** A point-neuron "dataset" is just the
network's structural parameters (sizes, connection density, weights) plus the
neuron biophysics; the connectivity itself is generated deterministically from a
hash of the seed (see `src/lif.h::synapse_target`), so the file only needs the
scalar parameters below. This lets `demo/run_demo` run **offline, with zero
downloads** (CLAUDE.md §8).

### File format (4 whitespace-separated lines)

```
line 1:  n_exc n_inh out_degree
line 2:  w_exc w_inh ext_kick ext_every
line 3:  steps seed
line 4:  v_rest v_reset v_thresh tau_m tau_syn r_m refractory_ms dt
```

The committed sample is:

```

```

### Field meanings (units)

| Field | Meaning | Units |
|---|---|---|
| `n_exc` / `n_inh` | number of excitatory / inhibitory neurons (ids `[0,n_exc)` are E, rest I) | count |
| `out_degree` | synapses each neuron sends (sparse: `<<` total) | count |
| `w_exc` / `w_inh` | outgoing synaptic weight of an E / I neuron (I is negative) | drive units |
| `ext_kick` | fixed external drive delivered to a rotating subset each step | drive units |
| `ext_every` | on step `t`, neurons with `id % ext_every == t % ext_every` get the kick | count |
| `steps` | number of timesteps to simulate (duration = `steps*dt`) | count |
| `seed` | seeds the deterministic wiring + the initial-voltage jitter | integer |
| `v_rest` / `v_reset` | leak-reversal / post-spike reset potential | mV |
| `v_thresh` | spike threshold | mV |
| `tau_m` / `tau_syn` | membrane / synaptic time constants | ms |
| `r_m` | membrane resistance factor (converts conductance `g` to a voltage drive) | — |
| `refractory_ms` | dead time after a spike | ms |
| `dt` | integration timestep | ms |

The default parameters put the network in a **balanced, recurrence-driven**
regime: the external drive reliably fires a rotating seed subset, recurrent
excitation recruits ~60% more spikes than the drive alone, and inhibition reins
activity back (removing inhibition roughly doubles the spike count). See
`../THEORY.md` for the numbers.

## Full / real datasets (optional, for validation)

This project needs **no** external data. If you want to compare a point-neuron
model against real recordings, or drive it with a real structural connectome, the
catalog points to (each under its own terms — the download scripts print links
only and never bypass registration):

> Allen Brain Observatory — visual-cortex spiking from Neuropixels
> (https://portal.brain-map.org); DANDI Archive — electrophysiology in NWB format
> (https://dandiarchive.org); OpenNeuro — EEG/MEG for network-model validation
> (https://openneuro.org); Human Connectome Project — structural connectivity
> matrices (https://db.humanconnectome.org).

See `scripts/download_data.ps1` / `.sh` for the same links and a note on how to
swap the hash-based wiring for a loaded adjacency list.
