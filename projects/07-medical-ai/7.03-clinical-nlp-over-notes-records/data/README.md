This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard

embeddings are a fixed integer recipe, not learned weights.

### File format

Whitespace-separated text; lines beginning with `#` are comments. Parsed by
`src/reference_cpu.cpp::load_notes`.

```
V D H S B                      # header: vocab size, model dim, heads, seq len, notes
tok: <id> <string>             # V lines: the vocabulary (id -> readable token)
note: <len> <id0> <id1> ...    # B lines: one note; `len` real tokens, padded to S with [PAD]
emb: <V doubles>               # D lines: row d = embedding dimension d for every token
proj: <tag>                    # recipe id; Wq/Wk/Wv are rebuilt in code, not stored
```

Field meanings:

- **V, D, H, S, B** — vocabulary size, model/embedding dimension, number of
  attention heads (`D % H == 0`), sequence length (all notes padded to this), and
  number of notes in the batch.
- **Token ids 0 and 1** are the special tokens `[CLS]` (sequence summary) and
  `[PAD]` (padding, masked out of attention). All other ids are toy clinical
  terms (`patient`, `he`, `denies`, `chest`, `pain`, …).
- **Embeddings** are stored dimension-per-row (compact to read); the loader
  transposes them to token-per-row internally.
- **The planted signal** (see `scripts/make_synthetic.py`): token `he` is given
  the same embedding as `patient`, so a correct self-attention block makes `he`
  attend most strongly to `patient` — a toy stand-in for the **coreference
  resolution** the catalog lists as a task. The demo reports whether that link is
  recovered (it is, in 4/4 notes).

Regenerate or resize with:

```
python scripts/make_synthetic.py                 # default 4 notes, D=8, H=2
python scripts/make_synthetic.py --dim 16 --heads 4
```

## Full dataset(s)

The real clinical-note corpora named in the catalog are **all credentialed** (they
contain de-identified but still legally protected patient text). `scripts/download_data.*`
**does not** and **must not** bypass their registration — it prints how to obtain
each legally:

- **MIMIC-IV Clinical Notes** — 331,794 de-identified notes from Beth Israel
  Deaconess. <https://physionet.org/content/mimic-iv-note/>. Requires a PhysioNet
  credentialed account, CITI "Data or Specimens Only Research" training, and a
  signed data use agreement.
- **i2b2 / n2c2 NLP Challenge datasets** — NER, coreference, and relation-extraction
  tasks in clinical text. <https://n2c2.dbmi.hms.harvard.edu/> (DUA + registration).
- **MTSamples** — 4,999 transcribed medical reports across 40 specialties.
  <https://mtsamples.com/> (check the site's terms before redistributing).
- **MedQA / MedMCQA** — medical question-answering benchmarks for evaluating
  clinical LLMs (verify the current URL and license).

Because none of these may be redistributed here, the committed sample **must** be
synthetic — and it is. To use a real corpus, obtain it legally, tokenize it (a real
pipeline would use a word-piece tokenizer such as the one shipped with
BioClinicalBERT), and write the token-id streams + a learned embedding table into
the loader's format above.
