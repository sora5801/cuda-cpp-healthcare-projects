## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/art_case.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`; no RNG, fully reproducible) |
| License | Public domain (CC0) — it is synthetic, **not** patient-derived |
| Size | ~145 KB (four 64×64 grids, text) |
| Format | line 1: `nx ny`; then `nx*ny` **plan_img**, `nx*ny` **daily_img**, `nx*ny` **plan_dose**, `nx*ny` **daily_dose** |

All values are whitespace-separated, row-major (index `= y*nx + x`). Images are grayscale intensities in
`[0,1]`; doses are in **Gray (Gy)**, `>= 0`. This tiny file lets `demo/run_demo` run **offline, with zero
downloads**, a hard requirement for every project (CLAUDE.md §8).

### What the sample contains (and why it is designed this way)

The four grids describe **one adaptive-radiotherapy fraction**:

- **`plan_img` (planning anatomy, the FIXED image):** a smooth 2-D **Gaussian blob** (σ ≈ 9 px) centered at
  (32, 32) — the tumour on the planning CT.
- **`daily_img` (today's anatomy, the MOVING image):** the **same** blob **translated** by (+5, +3) px **and**
  stretched ~12% wider in x. So the true deformation is *not* a pure translation — it varies across the frame,
  forcing a genuine deformable (per-voxel) field. Registering `daily_img` onto `plan_img` drives the SSD far
  down and recovers a **~4.5 px mean displacement** that matches the built-in shift+stretch, so the result is
  verifiable, not just plausible.
- **`plan_dose` (intended dose, planning frame):** a wider Gaussian **dose cloud** (σ ≈ 11 px, peak 2 Gy)
  centered on the planning target — the dose the plan intends to deliver.
- **`daily_dose` (delivered dose, today's grid):** the dose actually laid down today. The linac fires the
  planned beams, so the delivered cloud sits where the plan put it — but the tumour has **moved** out from
  under it. The **deformable warp** maps this dose back into the planning frame via the DVF; ignoring the
  motion (**rigid** accumulation) mis-places it. The demo prints both hot-spots so the difference is visible.

Because the ground-truth motion is *known*, the demo result is checkable. The GPU pipeline reproduces the
serial CPU pipeline to ~1e-15 (DVF and accumulated dose) and **exactly** (the integer dose-volume histogram).

Regenerate or resize it:

```bash
python scripts/make_synthetic.py                                   # the committed 64x64 case
python scripts/make_synthetic.py --nx 256 --ny 256 --shift 12.0    # a bigger problem
```

## Full dataset

There is **nothing to download** for the demo — this project makes its own synthetic case. For real adaptive-
radiotherapy work, `scripts/download_data.ps1` / `.sh` print pointers to the standard benchmarks. Each
requires you to register / accept a data-use license yourself; the scripts **never** bypass that, and none of
this data is committed here (license + patient privacy).

Catalog dataset notes (verbatim):

> TCIA CT-on-rails / CBCT datasets; DIR-Lab 4D-CT lung dataset (https://www.dir-lab.com/); AAPM TG-132 DIR
> test cases; CREATIS deformable lung phantom (https://www.creatis.insa-lyon.fr/).

- **DIR-Lab** (https://www.dir-lab.com/) — 4D-CT respiratory lung phase pairs with expert landmark points; the
  gold-standard target-registration-error (TRE) benchmark for DIR.
- **AAPM TG-132** (https://www.aapm.org/pubs/reports/RPT_132.pdf) — the clinical QA reference for DIR and
  deformable dose accumulation, with recommended test cases and tolerances.
- **TCIA** (https://www.cancerimagingarchive.net/) — planning-CT + daily-CBCT / CT-on-rails collections used in
  adaptive-radiotherapy studies.
- **CREATIS lung phantom** (https://www.creatis.insa-lyon.fr/) — a deformable lung phantom with a known
  ground-truth motion field.

To use real data with this project, export one 2-D slice per grid (planning image, daily image, planning dose,
daily dose — all the same `nx × ny`), normalize images to `[0,1]` and doses to Gy, and write the text format
above. **Do not commit** the converted data.

## Provenance & field meanings, safety

- Every value in `sample/art_case.txt` is **synthetically generated** and labeled as such here and in the
  generator's stdout. It is **not** derived from any patient.
- `plan_img`, `daily_img`: unitless intensities in `[0,1]`. `plan_dose`, `daily_dose`: absorbed dose in Gray.
- Nothing in this project may be used for diagnosis, treatment planning, or any clinical decision. It is study
  material only (CLAUDE.md §8).
