This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is a **parameter file**, not a
measurement: it defines *one microdosimetry scenario* that the C++ program then
simulates from scratch.

### The one-line format (order matters)

```
box_nm sigma_ion let_spread quantum_eV quanta_per_ion p_delta delta_quanta \
dna_radius_nm n_dna_segments n_y_bins y_max_keV_um n_tracks seed
```

| Field | Sample | Units / meaning |
|---|---|---|
| `box_nm` | 100 | side of the cubic liquid-water scoring volume (nanometres) |
| `sigma_ion` | 1.0 | **mean** ionizations per nm along the track — the particle's LET class |
| `let_spread` | 0.5 | lognormal spread of per-track LET (mixed field / straggling); 0 = mono-LET |
| `quantum_eV` | 30 | energy per deposited quantum (~ mean energy per ionization in water) |
| `quanta_per_ion` | 2 | integer quanta deposited at each primary ionization |
| `p_delta` | 0.5 | probability an ionization emits a short delta-ray cluster |
| `delta_quanta` | 3 | quanta deposited by a delta-ray cluster |
| `dna_radius_nm` | 3.0 | ionization within this distance of the DNA axis scores a strand break |
| `n_dna_segments` | 25 | number of DNA scoring segments along the box y-axis |
| `n_y_bins` | 12 | number of lineal-energy (y) histogram bins |
| `y_max_keV_um` | 480 | upper edge of the y histogram (keV/µm) |
| `n_tracks` | 4000 | number of primary particle tracks to simulate |
| `seed` | 20240517 | base RNG seed (track i uses the reproducible stream (seed, i)) |

The default scenario is a **moderately high-LET, mixed field**, tuned so the
result is interpretable: a broad f(y) with a high-y tail, SSB/DSB ≈ 2.3, and
DSB/track ≈ 0.11. Regenerate or vary it with:

```
python scripts/make_synthetic.py --sigma-ion 0.3 --let-spread 0   # sparse low-LET beam
python scripts/make_synthetic.py --n-tracks 40000                 # smoother spectrum
```

> **Synthetic label.** These parameters are **physically motivated but not
> validated radiobiology**. They exist to make the software demonstrable, not to
> predict real DNA damage yields.

## Full dataset

There is **no single downloadable dataset** for this teaching project — the input
is a handful of parameters, not a measured file. `scripts/download_data.ps1`/`.sh`
therefore print pointers to the real-world resources you would consult if you
extended this toward a research code, and never bypass any registration.

Catalog dataset notes (verbatim):

> Geant4-DNA physics validation data (https://geant4-dna.in2p3.fr/); NIST electron
> stopping powers (https://www.nist.gov/pml/estar); AAPM/NCRP microdosimetry
> benchmark datasets; published DNA damage yield datasets from radiobiology
> experiments.

What each would give you:

- **Geant4-DNA physics validation data** — tabulated liquid-water interaction
  cross-sections (ionization, excitation, elastic) that would replace the single
  scalar `sigma_ion` with real, energy-dependent physics. Open, redistributable.
- **NIST ESTAR** (<https://www.nist.gov/pml/estar>) — electron stopping powers and
  ranges in water; the reference for converting particle energy to LET.
- **AAPM / NCRP microdosimetry benchmarks** — reference y-spectra (TEPC
  measurements and simulations) to validate f(y) against.
- **Published DNA damage yields** — experimental SSB/DSB yields (Gy⁻¹ Da⁻¹) vs.
  LET to calibrate and validate the damage model.

Respect each source's license; none of it is redistributed here. The committed
sample is synthetic so the demo runs with zero external data.
