# Data — 5.8 Linac QA & Machine Performance Assessment

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/qa_planes_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no real patient/machine data |
| Size | ~4 KB |
| Contents | a **planned** (reference) dose plane + a **measured** dose plane, 24×24 px |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is **100% synthetic** and
carries no clinical meaning.

### File layout (whitespace-separated floats, parsed by `load_qa` in `reference_cpu.cpp`)

```
line 1 (header, 6 numbers):  nx  ny  spacing_mm  dd_percent  dta_mm  norm_dose
next nx*ny numbers:          the REFERENCE (planned) dose plane, row-major
next nx*ny numbers:          the MEASURED  (EPID/portal) dose plane, row-major
```

| Field | Meaning | Units | Sample value |
|---|---|---|---|
| `nx`, `ny` | plane dimensions (columns, rows) | pixels | 24, 24 |
| `spacing_mm` | physical size of one (square) pixel | mm | 2.0 |
| `dd_percent` | dose-difference criterion for gamma | % of `norm_dose` | 3.0 |
| `dta_mm` | distance-to-agreement criterion | mm | 3.0 |
| `norm_dose` | normalisation dose; **0** ⇒ use the reference plane's max | dose units | 0 |
| reference plane | the *planned* dose (an ideal open field) | arbitrary dose units | — |
| measured plane | the *delivered* dose = planned + a scripted machine error | arbitrary dose units | — |

**Note:** the file must contain *only* numbers (no `#` comment lines): the loader
does a bare `stream >> float` and stops at the first non-numeric token. All
provenance/labelling lives here in this README, not in the data file.

### How the synthetic sample is built

`scripts/make_synthetic.py` (which mirrors `make_synthetic_qa()` in `src/main.cu`)
generates an **open field** — a flat top of value 100 with a soft 2-pixel penumbra —
as the planned plane, then makes the measured plane by injecting a deterministic,
physically-plausible "healthy machine" error: **1% low output overall** plus the
**right half 2% hotter** (a mild left/right asymmetry). The demo's QA scorecard
recovers exactly these numbers (symmetry ≈ 1.96%, flatness ≈ 0.99%), which is how
you know the analysis is working. Regenerate with:

```bash
python scripts/make_synthetic.py            # defaults: 24x24, 2 mm, 3%/3mm
```

## Full / real dataset

There is no single downloadable "linac QA" file that this demo consumes directly;
real QA data is machine- and vendor-specific and often requires site access. The
public reference material relevant to this project:

- **AAPM TG-119** — a standard set of IMRT QA test *plans* (C-shape, head-and-neck,
  prostate, multi-target) used to commission QA systems.
  <https://www.aapm.org/pubs/reports/RPT_82.pdf>
- **AAPM TG-218** — the report defining the tolerance/action limits (3%/3mm, ≥95%,
  low-dose threshold) this project uses. <https://doi.org/10.1002/mp.12810>
- **OpenMedPhys / awesome-medphys** — a curated index of open medical-physics data
  and tools. <https://github.com/jrkerns/awesome-medphys>
- **Pylinac** — ships example EPID/portal images and log files for its analyses.
  <https://github.com/jrkerns/pylinac>

`scripts/download_data.ps1` / `.sh` print these pointers; they **do not** attempt to
download credentialed or license-restricted data. Respect every dataset's license —
if redistribution is forbidden, keep the committed sample synthetic (as it is here).

## Safety / honesty

All data in this project is **synthetic and labelled synthetic** everywhere it
appears. No output here is clinically valid or intended for any real medical
decision (CLAUDE.md §8).
