# Data — 5.01 Monte Carlo Dose Calculation (simplified slab)

## Committed sample (`sample/mc_params.txt`)

| Field | Value |
|---|---|
| Origin | **Synthetic** simulation parameters (`scripts/make_synthetic.py`) |
| License | Public domain (CC0) |
| Contents | One line of slab + run parameters |

This project's "data" is the **simulation parameters**, not measured input. The
physics is generated by the Monte Carlo itself.

### File format (one line)

```
L  n_bins  mu  p_abs  E0  scatter_dep  n_photons  seed
```

| Field | Meaning |
|---|---|
| `L` | slab thickness (cm) |
| `n_bins` | number of depth bins (the depth-dose resolution) |
| `mu` | attenuation coefficient (1/cm) — sets the mean free path `1/mu` |
| `p_abs` | probability an interaction is absorption (vs. forward scatter) |
| `E0` | starting energy quanta per photon (integer) |
| `scatter_dep` | quanta deposited per scatter event (integer) |
| `n_photons` | number of photon histories to simulate |
| `seed` | base RNG seed (history `i` uses the reproducible stream `(seed, i)`) |

Default sample: `20 40 0.15 0.30 1024 128 262144 12345`.

## "Full dataset" / realistic physics

Real Monte Carlo dose uses measured **cross-section tables** (photon/electron
interaction data) and a **patient CT** for geometry. Reference codes & data:

- **EGSnrc** (<https://github.com/nrc-cnrc/EGSnrc>) — reference photon/electron MC + PEGS cross-sections.
- **GATE / Geant4** (<https://github.com/OpenGATE/opengate>) — full physics, clinical geometries.
- **MC-GPU** (<https://github.com/DIDSR/MCGPU>) — open CUDA photon MC with material tables.

For more histories (smoother statistics): `python scripts/make_synthetic.py --photons 4000000`.

## Provenance & honesty

This is a **deliberately simplified, synthetic** teaching model: 1-D, integer
energy quanta, a single absorption/forward-scatter branch, and **no real cross
sections, no electron transport, no CT geometry**. It is **not** a dose engine
and must never be used for any clinical or dosimetric purpose.
