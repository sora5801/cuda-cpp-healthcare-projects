### Layout (whitespace-separated floats, parsed by `src/main.cu`)

```
line 1 : width height                                    (e.g. 32 32)
line 2 : spacing_mm  dd_percent  dta_mm  dose_threshold_frac   (e.g. 2 3 3 0.1)
line 3 : width*height reference dose values, row-major
line 4 : width*height evaluated dose values, row-major
```

### Field meanings

| Field | Meaning | Units |
|---|---|---|
| `width`, `height` | grid dimensions | voxels |
| `spacing_mm` | physical size of one voxel edge | mm |
| `dd_percent` | dose-difference criterion (global: % of reference max) | % |
| `dta_mm` | distance-to-agreement criterion | mm |
| `dose_threshold_frac` | low-dose analysis cutoff (fraction of max) | — |
| reference map | planned dose `D_r` | arbitrary dose units |
| evaluated map | measured dose `D_e` | same units as reference |

### What the synthetic sample encodes (a KNOWN answer)

The reference is a smooth **Gaussian dose "hill"** (max 100). The evaluated map is
that hill scaled by **+1.5%** everywhere (inside the 3% criterion → should pass),
plus a deliberate **3×3 central hot spot at +12%** (outside 3%/3 mm → should fail).
The demo therefore recovers a realistic IMRT-QA-shaped result: a high pass-rate
(99.7%) with a small, spatially-localized failure region (γ_max = 1.141). This is
engineered so the result is interpretable and verifiable (PATTERNS.md §6).

Regenerate or resize it:

```
python scripts/make_synthetic.py            # the committed 32x32 sample
python scripts/make_synthetic.py --n 64     # a larger 64x64 problem
```

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8).

## Full dataset

Real gamma-index inputs are **plan (reference) + measurement (evaluated) dose
pairs** from clinical QA, which are **not redistributable** (patient-derived, and
vendor-format). `scripts/download_data.ps1` / `.sh` do **not** attempt to fetch or
bypass credentials — they print pointers and instructions only:

- **AAPM TG-218** — the reference methodology and example criteria for
  patient-specific IMRT/VMAT QA. See the TG-218 report (Med Phys 45(4), 2018).
- **Plan + measurement DICOM-RTDOSE pairs** — obtained from your own department's
  TPS + QA system (film / EPID / diode array). Require local access and IRB/ethics
  handling; not public.
- **IROC-Houston phantom datasets** — credentialing phantom irradiations; request
  through IROC.
- **Linac EPID measurement datasets** — device-specific; from your own machine's QA
  archive.

For any of these you must respect the source license and patient-privacy rules.
The committed **synthetic** sample is sufficient to run and understand the demo;
`make_synthetic.py` provides a stand-in of any size.
