## Committed sample (`sample/oart_case.txt`) — **SYNTHETIC**

| Field | Value |
|---|---|
| File | `sample/oart_case.txt` |
| Origin | **Synthetic** (deterministically generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no patient data |
| Size | ~37 KB |
| Contents | one 2-D slice: planning MR (F), daily MR (M), planned dose, GTV mask |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). **It is synthetic and carries no
clinical meaning whatsoever; it exists only to exercise the code path.**

### File layout

```
line 1 : nx ny iters sigma k_norm dose_thresh
block 1: nx*ny doubles  = FIXED  image F (planning MR intensity, ~[0,1]), row-major
block 2: nx*ny doubles  = MOVING image M (daily   MR intensity, ~[0,1]), row-major
block 3: nx*ny doubles  = DOSE          (planned dose on F, Gray)
block 4: nx*ny doubles  = GTV mask      (1.0 inside tumour, 0.0 outside)
```

Row-major means index `i = y*nx + x` (x fast, y slow). All images are the same
`nx`-by-`ny` size.

### Per-field meaning (synthetic scenario)

- **`nx`, `ny`** — slice dimensions in voxels (default 32×32).
- **`iters`** — number of Demons registration iterations (default 60).
- **`sigma`** — Gaussian smoothing width (voxels) applied to the displacement
  field each iteration; the elastic regulariser (default 1.5).
- **`k_norm`** — Thirion force normaliser *K* (squared voxel spacing; default 1.0).
- **`dose_thresh`** — dose level (Gy) used for the "target coverage" report
  (default 30 Gy).
- **`FIXED` (F)** — a smooth elliptical "organ" with a bright round "tumour" disc
  in the centre: the *planning* anatomy.
- **`MOVING` (M)** — the **same** anatomy translated by a known `(dx, dy) = (3, 2)`
  voxels: the *daily* anatomy (e.g. a filling bladder pushed the tumour). This is
  the motion the registration must recover.
- **`DOSE`** — a Gaussian "dose cloud" (peak 60 Gy) planned to cover the tumour on
  F. After warping by the recovered field it should again cover the moved tumour.
- **`GTV`** — the gross-tumour-volume mask on F (the region we score dose over).

Regenerate or resize with:

```bash
python scripts/make_synthetic.py                 # default 32x32
python scripts/make_synthetic.py --nx 48 --ny 48 # larger synthetic slice
```

## Full / real datasets (not committed)

Real MR-Linac images are patient-derived and cannot be redistributed here. The
catalog lists legitimate sources; `scripts/download_data.ps1` / `.sh` print how to
obtain them and **never bypass** any required registration:

- **MR-Linac Consortium** shared datasets — verify the current URL at
  `mrlinac.org` (institutional access).
- **TCIA** (The Cancer Imaging Archive) MR-guided RT collections —
  <https://www.cancerimagingarchive.net/> (per-collection licenses; some require
  a data-use agreement).
- **AAPM MR-Linac Working Group** test cases — via the AAPM task-group pages.
- **MRI-only radiotherapy** cohorts from published papers (per-paper terms).

Catalog dataset notes (verbatim):

> MR-Linac Consortium shared datasets (verify URL at mrlinac.org); TCIA MR-guided
> RT datasets; AAPM MR-Linac WG test cases; MRI-only radiotherapy datasets from
> published cohorts.

**Respect every license.** If a set forbids redistribution, keep the committed
sample synthetic (as it is here). Never present any output as clinically valid —
this is a teaching demonstration only (CLAUDE.md §8).
