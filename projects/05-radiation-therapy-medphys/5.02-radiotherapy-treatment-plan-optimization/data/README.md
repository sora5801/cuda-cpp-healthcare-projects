This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which
is a hard requirement for every project (CLAUDE.md §8). It is **synthetic** — a
1-D geometric toy, *not* a patient dose calculation — and labeled synthetic
everywhere it appears (here, in `make_synthetic.py`, and in `THEORY.md`).

### File format (parsed by `load_problem()` in `src/reference_cpu.cpp`)

Whitespace-separated tokens (newlines are cosmetic):

```
n_vox n_beam nnz iters step d_rx          <- header
kind target weight                        <- one line per voxel (x n_vox)
...
row_ptr[0] ... row_ptr[n_vox]             <- CSR row pointers  (n_vox+1 ints)
col_idx[0] ... col_idx[nnz-1]             <- CSR column indices (nnz ints)
values[0]  ... values[nnz-1]              <- CSR nonzero values  (nnz floats)
```

**Per-field meaning:**

| Field | Meaning | Units |
|---|---|---|
| `n_vox` | number of dose voxels (rows of `D`) | — |
| `n_beam` | number of beamlets (cols of `D`, length of fluence `x`) | — |
| `nnz` | number of nonzeros in `D` | — |
| `iters` | projected-gradient iterations | — |
| `step` | gradient step size η | — |
| `d_rx` | PTV prescription dose | Gy |
| `kind` | voxel structure: `0`=PTV, `1`=OAR, `2`=BODY (see `enum StructKind`) | — |
| `target` | reference dose: prescription (PTV) / tolerance (OAR) / 0 (BODY) | Gy |
| `weight` | penalty weight `w_v` (clinical importance) | — |
| `row_ptr` | CSR: voxel `v`'s nonzeros are `[row_ptr[v], row_ptr[v+1])` | index |
| `col_idx` | CSR: beamlet index `j` of each nonzero | index |
| `values` | CSR: `D[v,j]` = dose in voxel `v` per unit fluence of beamlet `j` | Gy/fluence |

The default sample has `n_vox=48`, `n_beam=16`, a central PTV band, an OAR band
beside it, and Gaussian beamlet corridors (truncated → sparse). Regenerate or
scale it with:

```bash
python scripts/make_synthetic.py                       # the tiny committed sample
python scripts/make_synthetic.py --n-vox 4096 --n-beam 256   # a larger problem
```

## Full dataset

There is no single "download" for this project — real inverse-planning inputs are
**dose-influence matrices** computed by a treatment-planning system from a patient
CT, plus contoured structures. To study or benchmark with realistic data, use the
open research platforms and datasets from the catalog:

- **matRad** (https://github.com/e0404/matRad) and **pyRadPlan**
  (https://github.com/e0404/pyRadPlan) — open-source planning systems that
  **generate the dij matrix** for photons/protons/carbon and export it; ideal for
  producing a real `D` to feed a CSR optimizer like this one.
- **OpenKBP** (https://github.com/ababier/open-kbp) — 340 head-and-neck IMRT
  plans (CT, structures, dose) for knowledge-based planning research.
- **CERR** (https://github.com/cerr/CERR) — MATLAB RT research platform with
  DICOM-RT import.
- **OpenTPS** (https://opentps.org/) — Python/GPU treatment-planning system.
- **TCIA** RT collections and the **AAPM TG-263** structure-naming standard.

`scripts/download_data.ps1` / `.sh` print these pointers and do **not** attempt to
bypass any registration/license. **Respect every license**; several datasets
forbid redistribution, which is exactly why the committed sample is synthetic.

## Provenance & field meanings

The committed `sample/plan_sample.txt` is 100% **synthetic**, generated
deterministically (no RNG) by `scripts/make_synthetic.py` from an analytic 1-D
Gaussian-corridor model. It contains **no patient-derived data** and implies **no
clinical validity** — it exists only so the demo runs offline and the GPU result
can be verified against the CPU reference. Field meanings are tabulated above.
