This tiny plan lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **synthetic** — a made-up
proton pencil-beam plan chosen so the result is easy to interpret, not a real
treatment.

### File format (line-oriented; `#` starts a comment)

```
nx ny nz dx ox oy oz                       # dose grid: voxel counts, spacing (cm), origin (cm)
sigma0 sigma_grow peak_width p_exp z_entry # beam model + patient surface depth (cm)
n_spots                                    # number of pencil-beam spots
x0 y0 range weight                         # one line per spot (positions & range in cm)
```

### Field meanings & units

| Symbol | Meaning | Units |
|---|---|---|
| `nx, ny, nz` | dose-grid voxel counts along x, y, z (z = beam/depth axis) | — |
| `dx` | isotropic voxel edge length | cm |
| `ox, oy, oz` | world coordinates of the grid's `(0,0,0)` corner | cm |
| `sigma0` | lateral pencil-beam width at the surface | cm |
| `sigma_grow` | growth of lateral width per cm of depth (multiple scattering) | cm/cm |
| `peak_width` | Bragg-peak regularisation width (broader ⇒ lower, wider peak) | cm |
| `p_exp` | Bragg exponent `p` (Bortfeld ≈ 1.77); shapes the distal rise | — |
| `z_entry` | world z of the patient surface; the beam enters here | cm |
| `x0, y0` | lateral position of a spot's pencil-beam axis | cm |
| `range` | spot range `R`: depth where the protons stop (the Bragg-peak depth) | cm |
| `weight` | spot weight (MU-like); scales that spot's whole contribution | arbitrary |

The committed default is a **single 12 cm spot** — one pristine Bragg peak.
Passing `--ranges` to the generator stacks several spots into a **spread-out
Bragg peak (SOBP)** (see `scripts/make_synthetic.py` and the README exercises).

> **Dose units are arbitrary teaching units**, normalised to the Bragg peak in
> the demo output. They are **not** calibrated Gray and carry **no clinical
> meaning** (CLAUDE.md §1).

## Full / real dataset

This project needs **no** external dataset to run. If you want to study or
commission an analytic engine against real physics, the relevant sources are:

- **TOPAS / GATE benchmark proton beams** — Monte-Carlo integral depth-dose and
  lateral profiles used to fit pencil-beam kernels. Distributed with the
  toolkits: TOPAS (<https://www.topasmc.org>), GATE
  (<http://www.opengatecollaboration.org>). These are simulation outputs, not
  patient data, but follow each toolkit's license.
- **POPI-model** 4D CT for treatment planning:
  <https://www.creatis.insa-lyon.fr/rio/popi-model>.
- **TCIA** proton treatment-response collections (registration / data-use
  agreement required): <https://www.cancerimagingarchive.net>.

`scripts/download_data.ps1` / `.sh` print these pointers and **never** bypass any
registration or data-use agreement (CLAUDE.md §8). Patient-derived data is **not**
committed to this repo.

## Provenance & honesty

Everything in `sample/` is **synthetic** and labeled synthetic wherever it
appears. No output of this project may inform a real treatment decision.
