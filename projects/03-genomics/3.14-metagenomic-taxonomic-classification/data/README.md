### File layout

A line-oriented text format (`#` comment lines and blank lines are ignored):

```
# optional comment lines
T <num_taxa>
REF  <taxon_name>      <reference_genome_sequence>      # × num_taxa, taxon ids 1..num_taxa
R <num_reads>
READ <true_taxon_id>   <read_sequence>                  # × num_reads
```

| Token | Meaning |
|---|---|
| `T <num_taxa>` | how many reference taxa (species) follow |
| `REF name seq` | a reference genome for one taxon; the *n*-th `REF` line is taxon id *n* |
| `R <num_reads>` | how many reads follow |
| `READ truth seq` | one read; `truth` is the taxon id it was **simulated from** (0 = contaminant / none). `truth` is ground truth for measuring accuracy — the classifier never reads it. |

The synthetic community is deliberately **uneven** (taxon 1 ≈ 40 % of reads down to
taxon 5 ≈ 5 %) so the recovered abundance profile is interesting, and ~10 % of reads
are pure-random "contaminants" that should come back **unclassified**.

Regenerate it (deterministic) or make a larger stress set:

```bash
python scripts/make_synthetic.py                  # the committed sample
python scripts/make_synthetic.py --reads 100000   # a bigger run
```

## Full / real datasets

Real metagenomic classification matches reads against a database built from
reference genomes. The catalog points to (verbatim):

> NCBI RefSeq complete microbial genomes — standard Kraken2 database
> (https://ftp.ncbi.nlm.nih.gov/refseq/); CAMI challenge benchmark datasets —
> simulated metagenomes (https://data.cami-challenge.org/); HMP (Human Microbiome
> Project) reads (https://www.hmpdacc.org/); SRA metagenomics projects
> (https://www.ncbi.nlm.nih.gov/sra).

`scripts/download_data.ps1` / `.sh` print the recipe (download genomes → build a
k-mer→taxon map → classify reads). They download nothing automatically and never
bypass any registration; for HMP/SRA they print the access instructions only.

- **License:** NCBI RefSeq and CAMI data are openly available; respect each
  resource's terms. We do **not** redistribute any of it here — the committed
  sample is synthetic so this project carries no third-party data license.
- **Provenance of the committed file:** 100 % generated locally by
  `scripts/make_synthetic.py`; no real reads or genomes are included.

## Why this is a faithful (if reduced) stand-in

A real Kraken2 database is the same object as our `RefDatabase`: a hash map from
canonical k-mers to taxon ids, queried once per read k-mer. Our table is small and
uses exact k-mers (not minimizers) and a "first writer wins" rule instead of a true
lowest-common-ancestor over the NCBI taxonomy tree — both simplifications are
explained in `../THEORY.md` under "Where this sits in the real world".
