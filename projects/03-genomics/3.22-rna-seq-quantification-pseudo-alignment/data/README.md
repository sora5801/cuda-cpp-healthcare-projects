### File format

Comment lines start with `#` and are ignored. The token stream is:

```
T  M                         # number of transcripts, number of equivalence classes
eff_len[0] ... eff_len[T-1]  # effective length of each transcript (length units)
count  k  m0 m1 ... m_{k-1}  # ec line: read count, #members, member transcript ids
... (M ec lines total) ...
TRUTH  rho[0] ... rho[T-1]   # OPTIONAL ground-truth abundances (fractions, sum~1)
```

Field meanings:

- **transcript** `t` — one isoform of a gene; the unit we quantify.
- **effective length** `eff_len[t]` — transcript length minus mean fragment length
  + 1; the number of positions a read could start. Reads are generated in
  proportion to `abundance / eff_len`, so the EM divides length back out.
- **equivalence class (ec)** — a set of transcripts that some reads are *equally*
  compatible with. A singleton ec `{t}` is a read from a region unique to `t`; a
  multi-member ec is a read from a region shared by several isoforms.
- **count** — how many reads fell into that ec (the observed data).
- **TRUTH** `rho[t]` — the abundance (fraction of all reads) used to *generate* the
  sample. The demo reports how close the EM's estimate comes (it recovers it to
  ~1e-5 because the counts were generated by the same model the EM inverts).

### How the sample was built (so it is meaningful)

`scripts/make_synthetic.py` picks round abundances `rho = [.30,.10,.05,.20,.15,.20]`
and effective lengths, then sends a chosen number of reads into each *shared* ec,
splitting them among the ec's members in proportion to their length-normalised
weights `rho[t]/eff_len[t]` — exactly the proportion the EM assumes. Each
transcript's remaining reads become its unique-region singleton ec. Because the
generation matches the model, the true abundances are (up to integer rounding) the
EM's fixed point, so recovery is essentially exact — a clean teaching signal.

To regenerate or scale it:

```bash
python scripts/make_synthetic.py                 # the committed sample
python scripts/make_synthetic.py --reads 1000000 # same structure, more reads
```

## Full / real datasets

Real pseudo-alignment runs on a reference transcriptome + RNA-seq reads, and a
tool such as **kallisto** or **Salmon** produces the equivalence classes this
project consumes. `scripts/download_data.{ps1,sh}` print the canonical sources and
the exact `kallisto index` / `kallisto quant` commands to reproduce ec counts; they
never bypass credentials.

Catalog dataset notes (verbatim):

> GENCODE human transcriptome — reference transcript index (https://www.gencodegenes.org/); ENCODE RNA-seq FASTQs — diverse cell-type transcriptomes (https://www.encodeproject.org/); GTEx v9 — tissue RNA-seq compendium (https://gtexportal.org/); SRA RNA-seq studies (https://www.ncbi.nlm.nih.gov/sra).

- **GENCODE** — the reference transcriptome FASTA (the transcripts we quantify).
- **ENCODE / SRA** — open RNA-seq FASTQs to quantify; mostly downloadable directly.
- **GTEx v9** — a tissue RNA-seq compendium; **requires registration** (controlled
  access for individual-level data). Respect it; do not commit GTEx data here.

## Honesty
