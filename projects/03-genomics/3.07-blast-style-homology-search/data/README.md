### How the sample is engineered (so the result is interpretable)

The dataset has **known homology relationships by construction**, so the top-K
ranking is meaningful and deterministic (PATTERNS.md §6):

| Record | What it is | Expected outcome |
|---|---|---|
| `QUERY` | a random 120-residue "query" protein | (the query itself) |
| `hit_close` | the query with ~8% of residues mutated | **strongest** HSP score |
| `hit_medium` | the query with ~25% mutated | medium HSP score |
| `hit_domain` | unrelated scaffold with the query's middle 40 residues spliced in | one strong **local** HSP over the shared "domain" |
| `decoy_1…6` | fully random proteins | low / zero HSP score |

So the search **should** rank `hit_close > hit_medium > hit_domain > decoys`,
which is exactly what `demo/expected_output.txt` records. That is the whole point
of seed-extend: it recovers local homology even when flanks are unrelated
(`hit_domain`).

### Field meanings

- Each FASTA `>header` line: the first whitespace-delimited token is the sequence
  **name** shown in the demo output; the rest is a human description (including
  the word **synthetic** and the designed relationship).
- Residue lines: single-letter amino-acid codes (`ACDEFGHIKLMNPQRSTVWY`). The
  loader uppercases input, encodes each residue to an index `0..23`, and maps any
  unknown byte to `X` ("any residue"). The synthetic generator emits only the 20
  standard amino acids (no ambiguity codes), so every k-mer is a valid seed.

Regenerate (and optionally enlarge) the sample:

```bash
python scripts/make_synthetic.py                 # default: 120-aa query, 6 decoys
python scripts/make_synthetic.py --decoys 50     # a bigger DB
```

## Full / real datasets

This teaching project searches a **tiny synthetic** database. Real homology search
runs against large public protein databases. `scripts/download_data.{ps1,sh}`
print instructions and links for these — they **never** bypass any registration:

- **UniRef50 / UniRef90** — clustered UniProt sequences (the database AlphaFold2's
  MSA step searches): <https://www.uniprot.org/help/uniref>
- **NCBI nr** — the non-redundant protein database:
  <https://ftp.ncbi.nlm.nih.gov/blast/db/>
- **PDB70** — representative PDB sequences (used by HHsearch / structure pipelines):
  <https://www.rcsb.org/downloads>
- **Pfam** — protein-family HMM database (profile search, beyond this demo):
  <https://www.ebi.ac.uk/interpro/download/>

**License note:** UniProt/UniRef are CC BY 4.0; NCBI nr is public domain; PDB data
is freely available. Always check the current terms before redistributing. Because
those sets are large and their terms vary, **we do not commit any of them** — the
committed sample is synthetic.

> **Not for clinical use.** This is educational. Synthetic sequences are labeled
> synthetic everywhere; no output here implies a real biological or medical finding.
