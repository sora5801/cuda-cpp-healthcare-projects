| Size | < 1 KB (6 reads × 60 bases) |
| Format | FASTA-like: a `>` header line per read, then one sequence line of `A/C/G/T` |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **synthetic** — the sequence
is arbitrary and carries **no biological meaning**.

### How the sample is constructed (and why the answer is known)

`make_synthetic.py` (and the identical built-in fallback in `src/main.cu`) tile a
fixed **120-base pseudo-genome** with overlapping reads:

- **read length** = 60 bases, **step** = 12 bases ⇒ 6 reads starting at
  positions 0, 12, 24, 36, 48, 60.
- Consecutive reads overlap by **48 bases**, so they share many *minimizers*;
  reads far apart (e.g. read 0 vs read 5) overlap little or not at all.

Because we know the layout, we know the **expected overlap graph**: a single
chain `0–1–2–3–4–5`, i.e. **one connected component = one contig** spanning all
six reads. The demo recovers exactly this (11 edges at threshold ≥ 3 shared
minimizers, 1 component, largest component = 6 reads), which is how we know the
pipeline is correct on data with a ground truth (see `THEORY.md` §Verification).

### FASTA field meanings

- **Header** `>readN pos=P len=L SYNTHETIC`: `N` = read index, `P` = its start in
  the pseudo-genome (synthetic ground-truth coordinate, used only for the human
  to reason about the expected graph — the program never reads it), `L` = length.
- **Sequence**: `L` upper-case DNA bases over the alphabet `{A,C,G,T}`. `N` /
  ambiguous bases are tolerated by the loader (they just break the k-mer run).

### Regenerate / scale up

```bash
python scripts/make_synthetic.py                       # the committed 6-read sample
python scripts/make_synthetic.py --read-len 80 --step 8
python scripts/make_synthetic.py --genome-len 5000 --read-len 500 --step 100 --error-rate 0.02
```

A larger genome with more, longer reads makes the GPU's `O(n²)` pairwise stage
the clear bottleneck — the regime where the kernel matters.

## Full / real datasets

This is a **reduced-scope teaching version** of de-novo assembly (the all-vs-all
read-**overlap** stage). The real benchmark datasets named in the catalog are
large and are fetched, never committed:

- **CHM13 (T2T)** — telomere-to-telomere human reference, the assembly gold
  standard: <https://github.com/marbl/CHM13>
- **GenomeArk** — vertebrate genome assembly data: <https://genomeark.github.io/>
- **Human Pangenome Reference Consortium**: <https://humanpangenome.org/>
- **SRA PacBio HiFi / ONT reads** for species-specific de-novo projects:
  <https://www.ncbi.nlm.nih.gov/sra>

`scripts/download_data.ps1` / `.sh` print the links and the recommended access
path (e.g. the SRA Toolkit) and **never bypass any registration**. None of these
require credentials for download, but they are gigabytes in size, so the scripts
only guide you rather than auto-fetching. The committed synthetic sample is
sufficient for the demo.

## Honesty / safety

- The committed data is **synthetic** and labelled synthetic in the FASTA headers,
  in `make_synthetic.py`, and here.
- Nothing in this project is clinically valid or usable for diagnosis; it is a
  didactic illustration of the overlap-detection kernel only (CLAUDE.md §8).
