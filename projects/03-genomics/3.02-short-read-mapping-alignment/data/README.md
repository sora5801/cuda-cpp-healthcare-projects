## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/reads_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, fixed PRNG seed) |
| License | Public domain (CC0) — it is synthetic, no biological content |
| Size | < 1 KB |
| Layout | line 1: the reference sequence (ACGT); each later line: one read (ACGT), all reads the same length |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which
is a hard requirement for every project (CLAUDE.md §8).

### What the sample contains (the embedded known answer)

- A **240 bp synthetic reference** — a random ACGT string from a fixed seed
  (so the committed file is byte-stable and reproducible).
- **10 "mapped" reads** (40 bp each), copied from the reference at **known,
  evenly spaced start positions** (`0, 22, 44, 66, 88, …, 200`). Each read gets
  `i mod 4` point substitutions (0, 1, 2, 3, 0, …), **never** within its first
  `SEED_K = 12` bases — so the exact-match seed is guaranteed to hit and the
  aligner must recover the true position with score `40 − 2·mismatches`.
- **1 "unmappable" read** — pure random noise whose leading 12-mer is (with
  overwhelming probability) absent from the reference, so it correctly maps
  **nowhere**. This exercises the "no seed hit" path.

Because the true positions and mismatch counts are known by construction, the
demo's output is both **interpretable** (you can see the recovered positions) and
**verifiable** (the GPU and CPU must agree exactly). Regenerate with:

```bash
python scripts/make_synthetic.py                       # default tiny sample
python scripts/make_synthetic.py --ref-len 400 --n-reads 16   # a bigger one
```

> **Synthetic, not real.** This data is generated and carries **no biological
> meaning**. It must never be presented as real sequencing data or used for any
> clinical purpose.

## Full dataset

Real short-read data is large and/or requires registration, so it is **not**
committed. `scripts/download_data.ps1` / `.sh` print instructions and links for
the catalog datasets (they never bypass any registration):

- **1000 Genomes Project** — 2504 human WGS samples, short reads.
  <https://www.internationalgenome.org/data> (open; FTP/HTTP).
- **Genome in a Bottle (GiaB)** — NA12878 / HG002 benchmark short-read WGS, with
  truth-set variant calls for accuracy evaluation.
  <https://www.nist.gov/programs-projects/genome-bottle>.
- **SRA FASTQ archives** — petabyte-scale short reads; fetch with the SRA Toolkit.
  <https://www.ncbi.nlm.nih.gov/sra>.
- **ENCODE ChIP/RNA-seq FASTQs** — curated short-read functional-genomics data.
  <https://www.encodeproject.org/>.

To use real data with this teaching program you would: pick a small reference
(e.g. one contig / a region), extract equal-length reads as plain ACGT lines, and
feed the file as `argv[1]`. Real pipelines instead consume FASTQ + a full genome
index — see THEORY §7.

## Provenance & field meanings

The sample's only "fields" are nucleotide characters `A/C/G/T` (encoded to base
codes `0/1/2/3` by the loader). There are no units, no patient-derived content,
and no labels beyond position-by-construction. Respect every dataset's license
when you bring in real data; **never imply clinical validity**.
