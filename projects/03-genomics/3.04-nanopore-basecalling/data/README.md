# Data — 3.4 Nanopore Basecalling (CTC greedy decode)

## Committed sample (`sample/reads_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/reads_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, contains no real data |
| Size | < 4 KB |
| Contents | 4 reads' posterior matrices over `C = 5` CTC classes `{blank, A, C, G, T}` |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8).

### Why it is synthetic (and labeled as such)

Real posterior matrices are the **output of a trained neural network** run on raw
squiggle signal. Producing them is the research-grade stage this project
deliberately omits (CLAUDE.md §13). So we synthesize the network's output directly:
for each read we **plant a known DNA sequence** and build a posterior matrix that
greedily decodes back to it. This makes the demo *recoverable* — you can read the
planted sequence straight off the output and confirm the decoder is correct
(PATTERNS.md §6). The probabilities are **idealized** (one class clearly dominant per
step) and are **not** claimed to resemble a real model's outputs.

Planted sequences in the committed sample:

| Read | Planted sequence | Note |
|---|---|---|
| 0 | `ACGTACGT` | all neighbours distinct |
| 1 | `AACCGGTT` | every base doubled (homopolymers) |
| 2 | `GATTACA`  | contains the `TT` homopolymer |
| 3 | `TTTTGGGG` | long homopolymer runs |

### File format (consumed by `load_reads` in `src/reference_cpu.cpp`)

```
<n_reads> <C>            # C must equal CTC_NUM_CLASSES = 5
# then, for each read:
<T>                      # this read's number of time steps
<C floats>               # step 0 posteriors (probabilities over the C classes)
<C floats>               # step 1
 ...                     # T rows total
```

Class order is fixed: column `0 = blank`, `1 = A`, `2 = C`, `3 = G`, `4 = T`
(matching `src/ctc_core.h`). Each row's floats are the per-class probabilities at
that time step; the decoder only uses their `argmax`, so they need not sum to exactly
1 (though the synthetic generator makes them approximately normalized for realism).

Regenerate (deterministic, fixed seed):

```bash
python scripts/make_synthetic.py            # the committed 4-read sample
python scripts/make_synthetic.py --reads 64 # a larger synthetic batch
```

## Full dataset (real nanopore data)

Real ONT data is **raw signal** (`.pod5` / legacy `.fast5`), not posteriors. To go
from real signal to the input this project consumes, you must run a basecaller's
*network* (the out-of-scope stage). `scripts/download_data.ps1` / `.sh` document this
and point to the sources below — they **do not** download or run anything that needs
credentials.

Catalog dataset notes (verbatim):

> ONT Open Dataset (PromethION human WGS) — available via SRA / ENA (https://www.ncbi.nlm.nih.gov/sra); R9.4.1 and R10.4.1 benchmark datasets released by ONT (https://github.com/GoekeLab/awesome-nanopore); GIAB ONT ultra-long reads — NA12878/HG002 nanopore truth sets (https://www.nist.gov/programs-projects/genome-bottle); ENA Project PRJNA594038 — public multi-species ONT data (https://www.ebi.ac.uk/ena).

| Source | What it is | License / access |
|---|---|---|
| ONT Open Dataset (SRA/ENA) | PromethION human WGS signal | Public; via SRA/ENA accessions |
| R9.4.1 / R10.4.1 benchmarks | ONT pore-chemistry benchmark reads | Public; see awesome-nanopore |
| GIAB / NIST (NA12878, HG002) | Truth genomes to benchmark accuracy | Public domain (US Gov) |

**Respect every license.** None of the above is redistributed in this repo; the
committed sample is synthetic so the demo is self-contained.

## Provenance & field meanings

- The committed `reads_sample.txt` is **100% synthetic**, generated locally by
  `scripts/make_synthetic.py` with a fixed RNG seed (`20260628`) for byte
  reproducibility. It contains **no patient-derived or real sequencing data**.
- Each numeric value is a **class probability** at one time step (dimensionless,
  range `[0, 1]`); the only thing the decoder reads from them is which class is
  largest. Nothing here implies clinical validity — this is study material only.
