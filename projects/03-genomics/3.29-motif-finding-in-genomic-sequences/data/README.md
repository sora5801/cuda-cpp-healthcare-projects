| Size | 12 sequences × 60 bp (~1 KB) |
| Why synthetic | A known motif (`TGACGTCA`) is **planted** at a random offset in each sequence (with a 10% per-base mutation rate), so the demo can show that MEME EM **recovers** it — a meaningful, deterministic, offline result. |

### File format (standard FASTA)

```
>seq_0  synthetic planted_at=52 motif=TGACGTCA     # header (the 'planted_at' is the TRUTH, for study)
AAAGGTTCAACTTAATGATTTTCAGACGTACAGGAAGCTTGGAATAATCCCCTGACGTCA   # one sequence (may wrap)
>seq_1  synthetic planted_at=18 motif=TGACGTCA
...
```

The loader (`src/reference_cpu.cpp::load_sequences`) reads each `>`-delimited
record, concatenates the bases into one flat buffer, and encodes
**A=0, C=1, G=2, T=3, other=4**. Any non-ACGT character (N, gap, lowercase mask)
encodes to 4 and excludes the windows that overlap it. The `planted_at` field in
each header is the ground-truth offset; the program does **not** read it — it
discovers the motif from the sequence alone, and you compare afterward.

## Full dataset

Real motif discovery runs on **ChIP-seq peak sequences** — the DNA under the
binding peaks of a transcription factor — in FASTA. To assemble such input:

- **ENCODE** ChIP-seq peak BED files (thousands of TF experiments):
  <https://www.encodeproject.org/>
- **GEO** ChIP-seq datasets: <https://www.ncbi.nlm.nih.gov/geo/>
- **ReMap 2022** (regulatory elements from ~5k ChIP-seq experiments):
  <https://remap.univ-amu.fr/>
- **JASPAR 2024** (curated PWM database — known motifs to validate against):
  <https://jaspar.elixir.no/>

The standard pipeline: download a peak BED, extract the peak summit ±N bp from a
reference genome FASTA (e.g. with `bedtools getfasta`), and feed the resulting
multi-FASTA to a motif finder. `scripts/download_data.ps1` / `.sh` print this
recipe and the tool links; they download nothing and need no credentials. For a
large synthetic stand-in instead:

```
python scripts/make_synthetic.py --n 5000 --len 200
```

## Provenance & honesty

The committed sample is **synthetic** and labeled as such in every FASTA header
and in this file. The recovered motif, information content, and predicted sites
are meaningful only for this toy input — **no biological or clinical conclusion**
may be drawn from it. The planted consensus `TGACGTCA` is a real CRE/AP-1-like
element, used purely as a known target so the recovery is checkable.
