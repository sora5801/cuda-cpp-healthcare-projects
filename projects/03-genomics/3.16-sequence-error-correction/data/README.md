## Committed sample (`sample/reads_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/reads_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (fixed seed `20260628`) |
| License | Public domain (CC0) — it is synthetic |
| Size | ~15 KB (120 reads × 60 bp, plus ground truth) |
| Reads | 120 short reads sampled from a random 400 bp "genome" at ~18× coverage |
| Error model | 2% per-base substitution noise (an isolated-substitution model) |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8).

### File format (parsed by `src/reference_cpu.cpp::load_reads`)

```
# comment lines (start with '#') and blank lines are ignored
<n> <has_truth>          # n reads; has_truth = 1 if a truth line follows each read
<raw read 1>             # the observed (noisy) read, ASCII A/C/G/T (and possibly N)
<truth read 1>           # the error-free read (present only when has_truth == 1)
<raw read 2>
<truth read 2>
...
```

- **raw read** — what a sequencer would report: the true substring with a few
  bases flipped. This is the *input* to error correction.
- **truth read** — the error-free substring of the synthetic genome. It exists
  **only because the data is synthetic**; with real reads the truth is unknown.
  The program uses it purely to report the science metric *errors-before vs
  errors-after* (it is **not** part of the GPU-vs-CPU correctness check, which
  compares the GPU and CPU spectra/corrections to each other).

Why these parameters: at ~18× coverage every true 9-mer recurs ~18 times (far
above the trust threshold `T=3`), while a 9-mer carrying a sequencing error is
essentially random and appears once or twice. That separation is exactly the
signal the k-mer spectrum exploits. Regenerate or scale up with:

```bash
python scripts/make_synthetic.py                  # the committed 120-read sample
python scripts/make_synthetic.py --reads 200000   # a larger, GPU-friendlier set
```

## Full / real datasets

Real benchmark reads with known error profiles come from public archives. The
download scripts (`scripts/download_data.ps1` / `.sh`) print instructions and
links; they download nothing automatically and never bypass any registration.

Catalog dataset notes (verbatim):

> GAGE short-read datasets — benchmark reads with known errors
> (http://gage.cbcb.umd.edu/); GiaB HG001-HG007 — truth-set comparison for
> corrected reads (https://www.nist.gov/programs-projects/genome-bottle); ONT
> long-read SRA archives (https://www.ncbi.nlm.nih.gov/sra); PacBio CLR SRA
> datasets — high-error long reads (https://www.ncbi.nlm.nih.gov/sra).

| Dataset | Use | License / access |
|---|---|---|
| **GAGE** | Short reads with a known reference → measurable error rates | Public, per-dataset terms |
| **GIAB** (NIST) | High-confidence truth sets to score corrected reads | Public domain (US Gov) |
| **SRA** (ONT / PacBio CLR) | High-error long reads (different correction regime) | Public; respect submitter terms |

To use real reads with this teaching program, convert a FASTA/FASTQ slice into
the simple format above (drop the truth line and set `has_truth = 0`; the
*errors-before/after* metric is then unavailable but correction still runs and
CPU==GPU verification still applies).
