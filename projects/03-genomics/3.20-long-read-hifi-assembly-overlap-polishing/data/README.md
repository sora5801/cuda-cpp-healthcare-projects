# Data — 3.20 Long-Read HiFi Assembly Overlap & Polishing

## Committed sample (`sample/reads_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/reads_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, RNG seed 20) |
| License | Public domain (CC0) — it is synthetic, no real organism |
| Size | ~33 KB |
| Content | 12 reads' **minimiser sketches** (1193 minimisers total), `K=15`, `W=5` |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard
requirement for every project (CLAUDE.md §8). It is **synthetic**: there is no
real DNA here. The "genome" is a random ACGT string; reads are sampled from it
end-to-end with a 150 bp overlap between neighbours (every other read reverse-
complemented), then point-mutated at a HiFi-like 0.4% rate. The sketch is the
per-read list of minimisers, which is exactly what the overlap kernel consumes.

### File format

```
<n_reads>
<read_len> <cnt>          # repeated per read r = 0 .. n_reads-1
<pos> <hash>              #   cnt lines, positions ascending
...
```

| Field | Meaning | Units / type |
|---|---|---|
| `n_reads` | number of reads in the set | integer |
| `read_len` | length of read `r` | bases |
| `cnt` | number of minimisers in read `r`'s sketch | integer |
| `pos` | start offset of a minimiser's k-mer in the read | bases (0-based) |
| `hash` | canonical, mixed minimiser hash | 8-hex-digit 32-bit value |

The hashes are produced by the **same** canonical k-mer hashing as
`src/overlap_core.h` (forward/reverse-complement min, then a splitmix32 mix), so
two reads that overlap on the genome — on **either strand** — share minimiser
hashes. That shared-seed structure is the entire input to the GPU overlap kernel.

### Regenerate it

```bash
python scripts/make_synthetic.py                 # the committed 12-read sample
python scripts/make_synthetic.py --n-reads 2000  # a bigger overlap graph
```

If you regenerate with different parameters, re-capture
`demo/expected_output.txt` from a real run (the scores depend on the sample).

## Full dataset

Real PacBio **HiFi** reads are large public archives — no credentials are needed
for the ones below, but they are gigabytes, so the committed synthetic sample is
the default for the demo. `scripts/download_data.{ps1,sh}` print these pointers:

- **Human Pangenome / HG002 HiFi** (SRA): <https://www.ncbi.nlm.nih.gov/sra>
- **Vertebrate Genomes Project** HiFi assemblies: <https://vertebrategenomesproject.org/>
- **GenomeArk** HiFi datasets (AWS open data): <https://genomeark.github.io/>
- **CHM13 T2T** HiFi reads: <https://github.com/marbl/CHM13>

License: these are open research datasets; respect each archive's terms. We do
**not** redistribute any of them here — only the synthetic sample is committed.

To convert raw reads (FASTQ/BAM) into this project's minimiser-sketch format, use
a real sketcher such as **minimap2** (`-k 15 -w 5`, its minimiser index) or the
sketch stage of **hifiasm**; this project re-implements that seeding didactically
rather than copying it.

## Honesty / safety

The sample is **synthetic** and labelled as such everywhere. Nothing here is a
real genome, and no output is clinically meaningful — this is study material only
(CLAUDE.md §8).
