This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard

```
python scripts/make_synthetic.py            # default: k=11 w=5 s=16, seed 2026

```

### File format (parsed by `src/reference_cpu.cpp::load_reads`)

```
k w s                <- k-mer length, minimiser window (in k-mers), MinHash sketch size
>A                   <- section header for read set A
ACGTACGTACG...       <- one read per line (ACGT, optionally N for ambiguous bases)
...
>B                   <- section header for read set B
TCGATCCGTT...
...
```

### How the sample is engineered (so the result recovers a known answer)

The generator is **not** random noise — it plants a verifiable structure
(PATTERNS.md §6):

- A fixed-seed "genome" string is sampled to produce reads.
- A distinctive **motif** (`ACGTACGTACG` at k=11) is inserted near the start of
  the first few reads of set A, so its canonical k-mer is the unambiguous
  **top-count** k-mer the demo prints (count = 7).
- Set B is sampled from a genome window that **overlaps** A's window by ~50%, so
  the MinHash **Jaccard estimate** between A and B is a non-trivial number that
  reflects the true sequence overlap (≈ 0.125 for the committed sample).

Everything here is **synthetic** and for teaching only — no clinical meaning.

## Provenance & field meanings

- `k` — k-mer length in bases (1..31; packs into a 64-bit word at 2 bits/base).
- `w` — minimiser window measured in *consecutive k-mers* (not bases).
- `s` — MinHash sketch size (number of smallest distinct minimiser hashes kept).
- read characters — nucleotides `A/C/G/T`; any other character (e.g. `N`) makes
  the loader/kernels **skip** k-mer windows that span it, exactly like real tools.

Never imply clinical validity. Synthetic data is labelled synthetic everywhere.
