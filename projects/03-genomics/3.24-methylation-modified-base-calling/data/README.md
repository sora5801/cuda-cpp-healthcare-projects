| Field | Value |
|---|---|
| File | `sample/methylation_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (seed 7) |

| Size | ~10 KB (12 CpG sites × 8 reads, plus two 64-entry pore models) |
| Clinical validity | **None.** Educational only; not a real methylome. |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is engineered to be
**interpretable** (PATTERNS.md §6): each site carries a planted ground-truth label,
so the program can report calling accuracy (it recovers 12/12).

### On-disk format (whitespace/line text)

Parsed by `load_meth_data` in [`../src/reference_cpu.cpp`](../src/reference_cpu.cpp).
All values are space-separated; the layout is positional:

```
num_sites num_reads coverage                 # header (here: 12 96 8)
<64 lines>  canon_mean canon_stdv            # canonical pore model, k-mer code order 0..63
<64 lines>  meth_mean  meth_stdv             # methylated pore model, same order
<num_sites lines>  site_pos truth            # CpG reference coordinate + ground truth (1=5mC, 0=C)
<num_jobs lines>   read_id site_id  b0..b11  e0..e9
                                             # per (read, site): 12 reference base codes
                                             # (A=0,C=1,G=2,T=3) then 10 observed event currents
```

`num_jobs = num_sites × coverage` (here 96). Each job's 12 base codes are the
reference window centered on the CpG; the 10 event currents are the read's observed
mean currents over that window.

| Field | Meaning | Units |
|---|---|---|
| `canon_mean` / `meth_mean` | expected k-mer current (canonical / methylated model) | normalized pA-like |
| `*_stdv` | expected current spread for that k-mer (> 0) | normalized pA-like |
| `site_pos` | synthetic reference coordinate of the CpG | base index |
| `truth` | planted label: `1` = methylated (5mC), `0` = canonical C | — |
| `b0..b11` | 12 reference base codes of the window (A=0,C=1,G=2,T=3) | — |
| `e0..e9` | 10 observed event mean currents for this read | normalized pA-like |

The two pore models are **identical except** on the k-mer codes that contain the
CpG cytosine — in the methylated model those means are shifted by `+6.0`. That
shift is the *only* signal distinguishing 5mC from C, exactly as in a real pore.

## Full (real) dataset

This project ships **synthetic** data; no real dataset is required to learn the
algorithm. To experiment with real nanopore methylation signal:

- **Oxford Nanopore open datasets** — R10.4.1 reads with 5mC/6mA labels, indexed at
  https://github.com/GoekeLab/awesome-nanopore . Raw signal is `POD5`/`FAST5`;
  you would basecall (Dorado), align to a reference, and run f5c-style event
  alignment to produce the per-site event windows this project consumes.
- **ENCODE WGBS** — https://www.encodeproject.org/ — whole-genome bisulfite
  sequencing, the orthogonal **ground-truth methylation** calls nanopore results
  are benchmarked against (`bedMethyl`).
- **NCBI GEO methylation studies** — https://www.ncbi.nlm.nih.gov/geo/ .

`scripts/download_data.ps1` / `.sh` print these pointers and the format mapping;
they never bypass any registration. Respect each dataset's license; none of it is
redistributed here.

## Provenance & honesty

Everything in `sample/` is **synthetic and labeled synthetic**. The pore models are
hand-constructed, the events are Gaussian draws, and the methylation labels are
planted by the generator. Nothing here is a real measurement and nothing may imply
clinical validity (CLAUDE.md §8).
