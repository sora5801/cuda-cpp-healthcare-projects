### What the synthetic sample encodes (and why it is interpretable)

The reference is a 3-exon "gene" built as `E1(30) · I1(40) · E2(24) · I2(48) · E3(30)`
(172 bases total). Each intron `I1`,`I2` is **canonical**: it begins with the `GT`
donor dinucleotide and ends with the `AG` acceptor (the "GT-AG rule" obeyed by
~99% of human introns). The intron interiors are drawn only from `{A,C}` so no
spurious `GT`/`AG` site appears inside them — that keeps the true splice
boundaries the *only* canonical sites, so every junction read has **one
unambiguous best alignment** (a clean demo).

The reads are fragments of the **mature mRNA** (`E1·E2·E3`, introns already
spliced out). When mapped back onto the genomic reference, a read that straddles
an exon-exon boundary must **skip the intron** — producing a CIGAR with an `N`
("skipped region") operation. The engineered reads:

| Read | Origin in mRNA | Expected CIGAR | Teaches |
|---|---|---|---|
| `exon1_internal` | inside E1 | `20M` | no junction → pure match |
| `exon3_internal` | inside E3 | `22M` | no junction → pure match |
| `junc_E1E2_a` | spans E1\|E2 | `12M40N12M` | one intron skip (40 bp) |
| `junc_E1E2_b` | spans E1\|E2 | `8M40N16M` | same junction, different offset |
| `junc_E2E3_a` | spans E2\|E3 | `10M48N14M` | second junction (48 bp) |
| `junc_double` | spans both | `6M40N24M48N6M` | **two** introns in one read |

Because the answer is engineered in, the demo's output is *checkable by eye*.

## Full / real datasets

This is a **reduced-scope teaching** project: it aligns short reads against one
short reference "gene", not a whole genome. To experiment with **real** RNA-seq,
`scripts/download_data.(sh|ps1)` print pointers (they do **not** auto-download —
several sources are large and/or credentialed):

Catalog dataset notes (verbatim):

> ENCODE RNA-seq FASTQs (https://www.encodeproject.org/); GENCODE annotation (https://www.gencodegenes.org/); SRA RNA-seq benchmarks (SEQC/MAQC) (https://www.ncbi.nlm.nih.gov/sra); GTEx tissue RNA-seq (https://gtexportal.org/).

- **ENCODE / GENCODE** — open-access; FASTQ reads + the GTF gene annotation you
  would use to *know* the true exon/intron structure.
- **SRA SEQC/MAQC** — community benchmark RNA-seq for evaluating aligners.
- **GTEx** — tissue RNA-seq; **controlled-access** for individual-level data
  (dbGaP registration). The script prints the registration link and never
  attempts to bypass it (CLAUDE.md §8).

**Respect every license.** None of these are redistributed here; the committed
sample is synthetic. Real RNA-seq is **not for clinical use** in this study repo.

## Honesty

Everything in `sample/` is **synthetic and labeled synthetic**. It is engineered
to make the splice-aware DP legible; real reads carry sequencing errors,
non-canonical (e.g. `GC-AG`, `AT-AC`) splice sites, variable intron lengths into
the kilobases, and multi-mapping ambiguity — see `THEORY.md §Where this sits in
the real world`.
