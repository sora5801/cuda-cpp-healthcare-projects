### What the sample encodes (and why it is verifiable)

It is a **12-bin** synthetic Hi-C contact matrix (a "genome" of 12 loci) built so
the right answer is **known in advance** — the trick from `docs/PATTERNS.md §6`:

- **Three TADs** (topologically associating domains): bins `0–3`, `4–7`, `8–11`.
  Pairs **inside** a domain contact strongly (distance-decaying); pairs **across**
  a domain border contact only weakly. So the **insulation score** should dip at
  the two domain borders (bins **4** and **8**), and the boundary caller should
  recover exactly those — which the demo confirms.
- **Per-bin coverage bias.** Each bin carries a multiplicative "visibility"
  factor (some bins are over-sequenced); every raw count is multiplied by
  `bias_i * bias_j`. ICE balancing exists to *undo* exactly this, so embedding a
  known bias lets us watch ICE flatten the row sums (convergence variance falls
  toward 0).

### Field meanings

| Token | Meaning | Units / range |
|---|---|---|
| `n` (line 1) | number of genomic bins; the matrix is `n × n` | integer > 0 |
| `nnz` (line 1) | number of stored nonzeros (upper triangle only) | integer ≥ 0 |
| `i`, `j` | bin indices of a contact, **0-based**, `0 ≤ i ≤ j < n` | integer |
| `count` | observed contact count between bins `i` and `j` | non-negative integer |

The matrix is **symmetric**; only the upper triangle (`i ≤ j`) is stored. A real
genome bin spans a fixed genomic length (e.g. 1 kb–100 kb); here bins are abstract.

Regenerate the sample at any time:

```bash
python scripts/make_synthetic.py        # writes data/sample/hic_sample.txt
```

## Full dataset

Real Hi-C contact maps are large and distributed as **binary `.hic` (Juicer)** or
**`.mcool` (cooler)** files. `scripts/download_data.ps1` / `.sh` print the exact,
vetted steps to fetch one and convert a single resolution to the plain `i j count`
text this project reads (via `cooler dump`). Public sources (catalog "Datasets"):

> 4DN (4D Nucleome) Data Portal — Hi-C across cell types and time
> (https://data.4dnucleome.org/); ENCODE Hi-C datasets — cell-line 3D contacts
> (https://www.encodeproject.org/); GEO Hi-C studies (GSE63525, Rao et al. 2014)
> (https://www.ncbi.nlm.nih.gov/geo/); OpenChromatin Consortium ATAC/Hi-C.

- **License:** 4DN/ENCODE/GEO data are openly available for research; respect each
  dataset's specific terms. We do **not** redistribute any real map here — the
  committed sample is synthetic, so there is nothing license-encumbered in the repo.
- **No credentials required** for these portals (unlike MIMIC/UK Biobank); the
  download script still never bypasses any access control.

## Honesty

The sample is **synthetic** and labelled as such everywhere it appears. Nothing
here is patient-derived, and **no output is clinically valid** — this is study
material for the GPU compute pattern (ICE balancing + insulation TAD calling),
not a genomics pipeline you would publish from.
