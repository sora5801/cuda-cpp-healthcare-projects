# Data — 3.21 Structural Variant (SV) Calling

> **Everything here is SYNTHETIC and for education only. Not for clinical use.**

## Committed sample (`sample/sv_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/sv_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (fixed seed `3`) |
| License | Public domain (CC0) — it is synthetic, no real genome |
| Size | < 2 KB |
| Contents | one 240 bp reference window + 24 candidate split reads with a planted heterozygous-style **deletion** at breakpoint 120, length 50 |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is engineered so the result is
*interpretable* (PATTERNS.md §6): a single deletion is planted at a known
breakpoint, and the demo reports whether the caller recovered it.

### File format (parsed by `src/reference_cpu.cpp::load_dataset`)

```
REF   <reference_sequence>                 # ACGT string, the reference window
TRUTH <true_breakpoint> <true_del_length>  # planted ground truth (-1 = unknown)
N     <num_reads>
<raw_guess> <del_len> <flank_sequence>     # one line per read; repeated N times
...
```

Per-field meaning:

| Field | Meaning | Units |
|---|---|---|
| `reference_sequence` | the reference DNA window the reads map to | bases (A/C/G/T) |
| `true_breakpoint` | reference coordinate of the planted deletion's left edge | 0-based bp |
| `true_del_length` | number of bases deleted in the variant haplotype | bp |
| `raw_guess` | a read's **noisy** estimate of its breakpoint (as a split-read aligner would emit) | 0-based bp |
| `del_len` | a read's estimate of the deleted length | bp |
| `flank_sequence` | the `SV_FLANK` (=24) read bases ending at the breakpoint — the input to banded SW realignment | bases |

The 18 *supporting* reads all carry the **true** left flank (the 24 reference
bases ending at position 120) with `raw_guess` jittered by ±6 bp; the 6 *noise*
reads carry random flanks and scattered guesses. Re-alignment pulls every
supporting read back to bp 120, they pile into one histogram bin, clear the
support floor, and yield exactly one DEL call. The noise reads scatter below the
floor and are (correctly) not called.

Regenerate (or scale up) the sample with:

```bash
python scripts/make_synthetic.py                 # the committed tiny sample
python scripts/make_synthetic.py --reads 200000  # a large synthetic stress run
```

## Full / real datasets

This teaching project does **not** ship real genomic data. To experiment with
gold-standard benchmarks, see `scripts/download_data.ps1` / `.sh`, which print
instructions and links (and never bypass any registration). Catalog dataset
notes (verbatim):

> GiaB SV benchmark (HG002) — gold-standard deletion/insertion/inversion calls (https://www.nist.gov/programs-projects/genome-bottle); PacBio SV benchmark (https://github.com/PacificBiosciences/sv-benchmark); 1000 Genomes SV catalog (https://www.internationalgenome.org/data); ENCODE long-read SV studies (https://www.encodeproject.org/).

Real SV calling consumes aligned reads (BAM/CRAM) and emits VCF; mapping our toy
text format onto real BAM pileups is left as an exercise (see the project README).
Respect every dataset license; never present synthetic output as clinically valid.
