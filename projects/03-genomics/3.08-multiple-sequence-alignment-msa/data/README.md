### What the sample contains (and why it is interpretable)

Six **synthetic** DNA sequences (`seq0 … seq5`), all descended from one ancestral
motif: a fixed conserved core (`ACGTACGTTGCAACGT`) flanked by short random arms.
Each descendant was produced by applying a few point substitutions and small
insertions/deletions (`make_synthetic.py`, fixed PRNG seed → byte-identical every
run). Because they share the core, a **correct** multiple alignment lines that
core up into fully-conserved (`*`) columns — which you can see directly in the
demo output. That visible recovery is the dataset's built-in "known answer"
(PATTERNS.md §6).

Each residue is one of `A`, `C`, `G`, `T` (the loader rejects anything else; this
teaching project is DNA-only — see THEORY.md for the protein/BLOSUM upgrade).

Regenerate or enlarge:

```bash
python scripts/make_synthetic.py --n 12 --sub 0.10 --indel 0.06
```

## Full dataset

The demo needs no download. To go further, real MSA benchmarks (study their
licenses before redistributing) are listed in `scripts/download_data.ps1` / `.sh`:

- **BAliBASE** — curated reference alignments — <https://www.lbgi.fr/balibase/>
- **HomFam** — large homologous-family benchmark used by Clustal Omega (search
  "HomFam benchmark"; distributed alongside the Clustal Omega papers).
- **Pfam seed alignments** — protein family seed MSAs —
  <https://www.ebi.ac.uk/interpro/download/> (Pfam section).

These ship multi-FASTA the loader reads directly. **Caveat:** the protein sets use
the 20-letter amino-acid alphabet; the DNA-only loader here would reject them
until the substitution-matrix upgrade described in THEORY.md is added. None of
these require credentials, but always confirm the current license/URL — the
download scripts only print links and **never bypass any registration**
(CLAUDE.md §8).

## Provenance & field meanings

- **Header (`>seqN`)** — display label only; not used in scoring.
- **Sequence line** — the residues; encoded internally as `A=0, C=1, G=2, T=3`.

Everything here is **synthetic and labeled synthetic**. No real patient or
organism data is included, and no output implies clinical validity (CLAUDE.md §8).
