This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard

clinical meaning; nothing here may inform a real medical decision.

### File format (read by `src/reference_cpu.cpp::load_dataset`)

Whitespace/newline separated; lines beginning with `#` are comments.

```
# comment lines ...
N  G  k  target_sum          <- header (one line)
<label>  c0 c1 ... c(G-1)     <- one row per cell (N of them)
...
```

- `N` — number of cells; `G` — number of genes; `k` — neighbours per cell to
  find; `target_sum` — the fixed total each cell is normalized to (counts-per-`target_sum`).
- Each data row is a cell: an integer **type label** (the synthetic ground truth,
  `-1` if unknown) followed by `G` non-negative integer **counts** (one per gene).

### How the synthetic data is structured (and why it is interpretable)

`make_synthetic.py` builds **3 cell types**. Each type is defined by a contiguous
block of "marker" genes that are highly expressed in that type and lowly
expressed elsewhere — exactly how real cell types are distinguished by marker
programs. Per cell, counts are drawn from a Poisson around the type's mean
profile and then scaled by a random **sequencing depth** in `[0.5, 2.0]`. That
depth is the nuisance that **library-size normalization removes**: two cells of
the same type can have very different total counts yet normalize to nearby
points, so the KNN graph connects same-type cells. The demo reports a **label
purity** of 100% (every neighbour edge joins two same-type cells), which is how
we know the pipeline recovered the embedded structure.

Regenerate (deterministic):

```bash
python scripts/make_synthetic.py                 # the committed 30×18 sample
python scripts/make_synthetic.py --cells 48 --genes 24 --k 6   # a bigger one
```

## Full / real datasets

The committed sample is synthetic so the demo always runs. Real scRNA-seq count
matrices come from (catalog "Datasets", verbatim):

> Human Cell Atlas — multi-organ scRNA-seq compendium (https://www.humancellatlas.org/); 10× Genomics public datasets (https://www.10xgenomics.com/resources/datasets); CellxGene Census — 50 M+ cells (https://cellxgene.cziscience.com/); NCBI GEO — thousands of scRNA-seq studies (https://www.ncbi.nlm.nih.gov/geo/).

`scripts/download_data.ps1` / `.sh` print pointers to these. They do **not**
download or convert real data automatically: real matrices arrive as `.h5ad` /
`.mtx` / 10x HDF5 and need Scanpy/AnnData to export a dense slice into this
project's plain-text format. The scripts explain that path and never bypass any
registration or license.

### License note

All real sources above have their own licenses (often permissive for the public
10x sets; registration for some HCA/CellxGene assets). **Respect every license.**
Because redistribution terms vary, this repo commits **only** the synthetic
sample. If you export a real slice for local study, keep it out of version
control (`.gitignore` already excludes `data/full/`).
