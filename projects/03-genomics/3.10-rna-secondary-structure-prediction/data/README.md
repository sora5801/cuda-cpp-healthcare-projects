| Field | Value |
|---|---|
| File | `sample/rna_sample.fasta` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, not a real transcript |
| Size | < 1 KB |
| Format | FASTA: one `>` header line + one RNA sequence line (alphabet `ACGU`) |

**The sequence:** `GGGCGCAAAAGCGCCCAU` — a designed 18-nt **hairpin**. The first
six bases (`GGGCGC`) are reverse-complementary to bases 11–16 (`GCGCCC`), so they
zip up into a 6-bp GC-rich **stem**; the `AAAA` in the middle is the unpaired
**hairpin loop**; the trailing `AU` is left dangling. Under the Nussinov model
(maximum base pairs, minimum loop = 3) it folds to exactly **6 base pairs**:

```
GGGCGCAAAAGCGCCCAU
((((((....))))))..      <- '(' pairs with the matching ')'; '.' is unpaired
```

This known answer is encoded in `demo/expected_output.txt`, so the demo is a real
correctness check (CPU and GPU must both recover it), not a tautology. The file
lets `demo/run_demo` run **offline with zero downloads** (CLAUDE.md §8).

Regenerate or vary it:

```bash
python scripts/make_synthetic.py                  # the default hairpin above
python scripts/make_synthetic.py --random 40      # a random length-40 RNA
python scripts/make_synthetic.py --seq GGGAAACCC  # any explicit sequence
```

### Field meanings

- **Header line** (`>...`): free-text label. Our loader (`src/reference_cpu.cpp`,
  `load_rna`) skips any line starting with `>` and reads the first sequence line.
  We mark it `SYNTHETIC` so nobody mistakes it for a real molecule.
- **Sequence line**: ribonucleotides as letters `A C G U`. A `T` is also accepted
  and treated as `U`, so DNA-style files load too. Whitespace is ignored.

## Full dataset (real RNA structures)

Real, curated RNA secondary structures (for comparing predictions against
experimentally/comparatively determined "ground truth") come from public
databases. None require credentials, but they are far larger than this teaching
sample, so we fetch on demand rather than committing them:

- **Rfam** — RNA family alignments + consensus secondary structures.
  <https://rfam.org/> (CC0). Per-family Stockholm files carry a `#=GC SS_cons`
  consensus-structure annotation line.
- **RNAcentral** — a comprehensive non-coding-RNA sequence database.
  <https://rnacentral.org/> (CC0). Good source of raw sequences to fold.
- **PDB** — RNA 3D structures with derived secondary structure.
  <https://www.rcsb.org/> (CC0).
- **ArchiveII** — a classic curated benchmark of single-sequence RNA secondary
  structures used to score folding accuracy. Distributed with the RNAstructure
  package; see <https://rna.urmc.rochester.edu/RNAstructure.html>. (Verify the
  exact mirror/URL and its terms before redistributing — we do **not** commit it.)

`scripts/download_data.ps1` / `.sh` print these links and instructions; they do
not scrape or bypass any site's terms. See those scripts for details.

## Provenance & honesty

The committed sample is **synthetic** and labeled synthetic everywhere it
appears. Nussinov base-pair maximisation is a didactic model: it predicts a
plausible non-crossing folding but **omits the thermodynamic energies** that
real tools (ViennaRNA, RNAstructure) use, so its structures are illustrative, not
clinically or experimentally authoritative. Nothing here is for medical use.
