### File format

```
# comment lines start with '#'
guide  <NAME>  <20-letter ACGT spacer>      # the guide RNA spacer
genome <ACGT string>                        # one long line; loader concatenates
```

Bases are uppercased, then encoded as 2-bit codes (`A=0 C=1 G=2 T=3`); any other
character becomes "invalid" (255) and can never match. The loader
(`src/reference_cpu.cpp::load_problem`) rejects a guide that is not exactly 20
ACGT bases, or a genome too short to hold one 23-base window.

### What is engineered into this sample (so the result means something)

The genome is **PAM-scrubbed random filler** with these sites spliced in, each
followed by an `NGG` PAM:

| Site | Mismatches | Where | Expected CFD |
|---|---|---|---|
| on-target | 0 | exact 20-mer + `CGG` | 1.000000 (recovered at position 40) |
| off-target | 1 | **distal** (5′ end) | ~0.86 — barely penalized |
| off-target | 1 | **seed** (PAM-proximal) | 0.05 — heavily penalized |
| off-target | 2 | mixed | ~0.06 |
| off-target | 2 | both seed | ~0.003 |
| off-target | 3 | spread | ~0.03 |

The wide spread is the whole point: it makes the **seed effect** (mismatches near
the PAM cripple cutting far more than distal ones) visible in the demo's ranked
output, and gives the verification something non-trivial to check.

## Full dataset — real off-target scanning

Real off-target assessment runs a guide against a **reference genome**:

- **hg38 / mm10 reference genomes** — UCSC: <https://genome.ucsc.edu/> (FASTA,
  gigabytes). One human chromosome alone is ~10⁸ candidate windows per guide.
- **CRISPOR benchmark** — validated guide efficiencies + off-targets:
  <https://crispor.gi.ucsc.edu/>
- **GeCKO v2 library** — genome-scale knockout-screen guides (Addgene pooled
  library): <https://www.addgene.org/pooled-library/>
- **Azimuth / Rule Set 2** — published on-target efficiency training data
  (for the efficiency model this teaching project does *not* implement).

`scripts/download_data.ps1` / `.sh` print where to get a reference genome and how
to convert FASTA into the loader's format; they download nothing and need no
credentials. For a larger synthetic genome instead:

```
python scripts/make_synthetic.py --filler 100000
```

## Provenance & honesty

The committed sample is **synthetic** and labeled as such on every line of the
file and here. The CFD weights used to score it are a **synthetic teaching
model**, *not* the published Doench-2016 CFD matrix (see `THEORY.md` §"Where this
sits in the real world"). No score here has biological meaning, and **nothing in
this project may inform a real CRISPR experiment** — it is study material only.
