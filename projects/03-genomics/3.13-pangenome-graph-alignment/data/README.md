| Field | Value |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md В§8). It is **synthetic** and must
never be presented as real genomic data or used for any clinical purpose.

### File format (the grammar `src/reference_cpu.cpp::load_problem` parses)

One record per line; blank lines and `#` comments are ignored:

```
# comment text
Q  <DNA>                 a query read (exactly one).  e.g.  Q GCTAATTAC...
N  <name> <DNA>          a graph node, listed in TOPOLOGICAL order.  e.g.  N a0 GCTAAA
E  <src_name> <dst_name> a directed edge src -> dst.  e.g.  E a0 s0ref
```

**Topological-order rule.** Nodes are given integer indices in declaration order,
and every `E` edge must point *forward* (source declared before destination). That
makes declaration order a valid topological order вҖ” the DP fills node blocks in
ascending index order and relies on every predecessor being finished first
(`THEORY.md` В§The algorithm). The loader rejects any backward edge with a clear
error rather than silently miscomputing.

### What the sample encodes (the embedded "known answer")

The generator builds a chain of **SNP bubbles**. Each bubble is two parallel allele
nodes (`s<k>ref`, `s<k>alt`, differing by one base) between two shared **anchor**
nodes (`a0, a1, вҖҰ`):

```
 a0 в”Җв”Җв–¶ s0ref в”Җв”Җв–¶ a1 в”Җв”Җв–¶ s1ref в”Җв”Җв–¶ a2 в”Җв”Җв–¶ вҖҰ
   в””в”Җв”Җв–¶ s0alt в”Җв”Җв”ҳ   в””в”Җв”Җв–¶ s1alt в”Җв”Җв”ҳ
```

The query read is generated to follow **one specific path** вҖ” `ref` allele on even
bubbles, `alt` on odd вҖ” with ~8 % random point mutations sprinkled in. So there is
a single high-scoring local alignment, and a correct graph aligner must **recover
that exact allele path**. The committed sample's answer is
`a0>s0ref>a1>s1alt>a2>s2ref>a3>s3alt>a4`, which the demo prints and which validates
the science, not just CPU==GPU agreement (PATTERNS.md В§6).

Regenerate (or make a bigger graph) with:

```
python scripts/make_synthetic.py                  # the committed default
python scripts/make_synthetic.py --snps 6 --seg 8 # more/larger bubbles
```

## Full dataset

Real pangenome graphs come as **GFA** files (Graphical Fragment Assembly) from:

- **Human Pangenome Reference Consortium (HPRC)** вҖ” 94 haplotype-resolved
  assemblies. <https://humanpangenome.org/>
- **1000 Genomes Project GVCFs** вҖ” variant calls used to *construct* graphs.
  <https://www.internationalgenome.org/data>
- **Ensembl Pangenome** вҖ” multi-species graphs. <https://www.ensembl.org/>
- **PGGB tutorial data** вҖ” small, runnable example graphs.
  <https://github.com/pangenome/pggb>

These are large and/or governed by data-use agreements; this teaching project does
**not** redistribute them. `scripts/download_data.ps1` / `.sh` print the source
links and the standard build pipeline (`pggb` to build a graph, `vg`/`odgi` to
inspect it) but never bypass any registration. To convert a real GFA to this
project's tiny text format you would emit one `N`/`E` line per GFA `S`/`L` record,
after a topological sort (`vg ids -s`, `odgi sort`); doing that for a full graph is
left as an exercise (`README.md` В§Exercises).

## Provenance & field meanings

- **Q line** вҖ” the read to align (codes A/C/G/T). One per file.
- **N line** вҖ” a node `name` (an arbitrary token used only for printing the path)
  and its DNA `segment`. Segment length is arbitrary; the sample uses 6.
- **E line** вҖ” a directed edge; the graph must be a **DAG** in topological order.

Never imply clinical validity. The committed data is labeled **synthetic**
everywhere it appears; real HPRC/1000G data is human-derived and must be handled
under its own license and ethics terms (CLAUDE.md В§8).
