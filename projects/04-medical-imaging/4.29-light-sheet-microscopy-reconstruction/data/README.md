## Committed sample (`sample/lsfm_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/lsfm_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (a fixed pseudo-random seed, no external data) |
| License | Public domain (CC0) — it is synthetic |
| Size | ~7 KB (a 32 x 32 image) |
| Represents | One optically-blurred, mildly-noisy 2D *plane* of a light-sheet stack |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is **synthetic** and is
labeled as such everywhere; it is **not** real specimen or patient data and
carries no clinical meaning.

### File layout (what the loader `load_lsfm` expects)

```
line 1 :  H  W  SIGMA  ITERS
then   :  H*W  floating-point pixel values (row-major), whitespace-separated
```

| Symbol | Meaning | Units | Sample value |
|---|---|---|---|
| `H` | image height (rows) | pixels | 32 |
| `W` | image width (cols) | pixels | 32 |
| `SIGMA` | Gaussian point-spread-function standard deviation | pixels | 1.6 |
| `ITERS` | number of Richardson-Lucy iterations to run | — | 12 |
| pixel | measured (blurred + noisy) fluorescence intensity | arbitrary (photon-like) | ~0.05 background, brighter at beads |

### How the sample is constructed (fully transparent)

`scripts/make_synthetic.py`:

1. builds a **known ground-truth** image — five bright sub-pixel "beads"
   (point-like fluorophores) on a dim uniform background;
2. **blurs** it by convolving with the *same* origin-centered, unit-sum Gaussian
   PSF (`SIGMA` px) the program deconvolves against;
3. adds **mild, deterministic** multiplicative noise from a fixed
   linear-congruential generator (so the file is byte-identical on every machine).

Because the ground truth is known, the demo can pose a real question — *does
Richardson-Lucy sharpen the beads back up?* — and answer it with the reported
peak/L2 sharpening ratios.

## Full / real datasets

Real light-sheet (LSFM / SPIM) datasets are **terabyte-scale 3D+time volumes**
stored as TIFF / HDF5 / N5 / Zarr — a different world from this didactic text
plane, and several require a registration or data-use agreement. `download_data.*`
therefore prints the public sources and defers to the synthetic generator rather
than fetching TBs. Study these to see production data and formats:

- **OpenOrganelle (Janelia):** https://openorganelle.janelia.org/ — FIB-SEM and
  light-sheet neuroscience volumes.
- **EMBL LSFM public datasets:** https://www.embl.org/ — SPIM acquisitions.
- **BioImage Archive (EBI):** https://www.ebi.ac.uk/biostudies/bioimages — LSFM
  collections.
- **Zebrafish SPIM atlas data:** from the Nature Methods SPIM papers.

**Licensing:** respect every dataset's license. If redistribution is forbidden,
the committed sample stays synthetic (as it is here). For credentialed sets, the
download script prints registration instructions only and never bypasses
authentication (CLAUDE.md §8).

## Honesty

The committed data is synthetic and is used purely to demonstrate the
reconstruction algorithm. Nothing here is a real specimen, and no output implies
diagnostic or clinical validity.
