## Committed sample (`sample/volume_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/volume_sample.txt` |
| Origin | **Synthetic** — generated by `scripts/make_synthetic.py` (NOT patient data) |
| What it is | A 17×17×17 scalar volume holding a signed-distance field of a sphere |
| Field meaning | `value(i,j,k) = radius − distance_from_center(i,j,k)`, in mm |
| Iso-value | `0.0` — the surface `value = 0` is exactly the sphere of radius 6 mm |
| Spacing / origin | 1.0 mm isotropic; grid centred on world `(0,0,0)` |
| Size | ~14 KB, 4913 samples |
| License | CC0 / public domain (it is synthetic; generate it yourself) |

### File format (what `src/reference_cpu.cpp::load_problem` reads)

```
line 1 : nx ny nz spacing origin_x origin_y origin_z iso
rest   : nx*ny*nz floats, row-major with x fastest, then y, then z
```

So `vol[(k*ny + j)*nx + i]` is the sample at grid point `(i,j,k)`. The loader is
forgiving about line breaks (it reads whitespace-separated numbers) and validates
the count, so a truncated file fails loudly.

### Why a sphere?

A sphere implicit field has a **known analytic surface area** `4πr²`, so the demo
can cross-check the extracted mesh against ground truth (see `src/main.cu`'s
`[science]` line and THEORY.md "How we verify"). It is also smooth, so the classic
Marching Cubes table extracts it without ambiguous-case artefacts — ideal for a
first didactic example. Regenerate or vary it:

```bash
python scripts/make_synthetic.py                 # default 17^3, r=6 mm
python scripts/make_synthetic.py --n 33 --radius 12
```

**Everything in `sample/` is synthetic and labeled synthetic. It is not a patient
scan and has no clinical meaning.**

## Real datasets (full pipeline)

The catalog points at real clinical CT collections used to build patient-specific
surgical models. They are **not committed** here — they require registration and/or
forbid redistribution. `scripts/download_data.ps1` / `.sh` print pointers and do
**not** attempt to bypass any credential wall (CLAUDE.md §8).

| Dataset | What | Access |
|---|---|---|
| TCIA body CT collections | Whole-body CT for organ/bone models | https://www.cancerimagingarchive.net/ (per-collection license) |
| OsteoArthritis Initiative (OAI) | Knee MRI/CT for joint models | https://nda.nih.gov/oai/ (registration) |
| VerSe | Vertebral CT + segmentations | https://github.com/anjany/verse (open) |
| TotalSegmentator dataset | CT + 104-structure labels | https://zenodo.org/record/6802614 (CC BY) |

To use a real volume with this project you would resample it to a regular grid and
write it in the text format above (a few lines of Python with `nibabel` /
`pydicom` + `numpy`); the marching-cubes core is identical. That conversion is left
as an exercise — the committed synthetic sphere keeps the demo offline and exactly
verifiable.
