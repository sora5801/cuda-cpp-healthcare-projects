# Data — Project 4.4 Deep-Learning MRI/CT Reconstruction

## What ships here (committed)

`sample/mri_scan_sample.txt` — a **tiny, fully SYNTHETIC** MRI-style acquisition,
generated by `scripts/make_synthetic.py`. It contains **no patient data** of any
kind. It is a 24×24 numerical phantom, its exact 2-D Fourier transform (k-space),
and an under-sampling mask — everything the reconstruction needs to run offline.

> **Synthetic label.** Every value in this file is computer-generated from a
> geometric phantom. It is not derived from any scan, person, or clinical study.
> Nothing here may be used for diagnosis or treatment (this is study material).

### File layout (whitespace-separated floats)

```
ny nx                       # image dimensions (24 24)
truth[0 .. N-1]             # N = ny*nx ground-truth pixels, row-major, in [0.1, 1.0]
mask[0 .. N-1]              # 0/1 per frequency: 1 = this k-space sample was "measured"
kmeas_re[0 .. N-1]          # measured k-space, real part (0 where mask==0)
kmeas_im[0 .. N-1]          # measured k-space, imag part (0 where mask==0)
```

- **`truth`** is the phantom the reconstruction tries to recover: a dim
  background (0.1), a bright central disk (1.0), and a mid-gray square (0.6). In a
  real scan you never have this — it exists here only to *score* how good the
  reconstruction is (`rms_error` in `src/reference_cpu.cpp`).
- **`mask`** encodes the acceleration: we keep a narrow low-frequency band plus
  every other line, retaining **225 / 576 ≈ 39%** of k-space (~2.5× acceleration).
- **`kmeas`** is the phantom's k-space with the unsampled bins zeroed — i.e. what
  the "scanner" reported.

Field meaning and generation live in `scripts/make_synthetic.py`, which mirrors
`make_synthetic_acquisition()` in `src/reference_cpu.cpp` closely, so the file and
the program's built-in fallback describe the same scan.

Regenerate (or resize) it:

```bash
python scripts/make_synthetic.py            # the committed 24x24 sample
python scripts/make_synthetic.py --ny 32 --nx 32
```

## The real dataset (not committed)

The gold-standard data for learned MRI reconstruction is **fastMRI** (NYU
Langone): raw **multi-coil k-space** for knee and brain, with fully-sampled
references.

- **Source / registration:** <https://fastmri.med.nyu.edu/>
- **License:** free for research **after** accepting a data-use agreement. It is
  **not redistributable**, so we commit only the synthetic sample above. Our
  `scripts/download_data.*` prints the registration steps and **never** attempts
  to bypass them.
- **Size / format:** tens–hundreds of GB of HDF5 (`.h5`) files; a single volume is
  256²-ish × up to 32 coils.
- **Annotations:** fastMRI+ (radiologist labels) —
  <https://github.com/StanfordMIMI/fastMRI_plus>.
- **CT alternative:** the **2016 AAPM Low-Dose CT Grand Challenge** for learned CT
  reconstruction.

Consuming `.h5` k-space and training a network is a PyTorch job (see the fastMRI
baseline repo). This C++ project deliberately teaches the *reconstruction
structure* on the synthetic phantom instead — see `../THEORY.md`.
