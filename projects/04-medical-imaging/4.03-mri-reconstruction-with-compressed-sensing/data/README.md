| Field | Value |
|---|---|
| File | `sample/kspace_sample.txt` |

| License | Public domain (CC0) — it is synthetic, no patient data |
| Size | ~30 KB (a 32×32 slice) |
| Contents | one under-sampled k-space slice + its sampling mask + the ground-truth image |

> **This is synthetic data.** It is a mathematical phantom (a few bright Gaussian
> blobs, like a sparse MR angiogram), **not** a real scan of a real person. Nothing
> here is clinical.

### File layout

```
line 1 :  n  lambda  iters  has_truth
n*n rows (row-major, index = row*n + col):
          re  im  mask  [truth]
```

| Symbol | Meaning | Units |
|---|---|---|
| `n` | image side length (a power of two; grid is `n × n`) | pixels |
| `lambda` | FISTA L1 regularization weight (sparsity strength) | dimensionless |
| `iters` | number of FISTA iterations to run | count |
| `has_truth` | 1 if a `truth` column is present, else 0 | flag |
| `re`, `im` | measured k-space value at this position (0 where unsampled) | arbitrary MR units |
| `mask` | 1 = this k-space position was acquired, 0 = skipped (zero-filled) | flag |
| `truth` | ground-truth magnitude image pixel (synthetic only) | arbitrary MR units |

The committed sample is a **32×32** slice with **~34%** of k-space acquired
(a dense low-frequency center plus a random subset of high frequencies — the
standard variable-density CS-MRI pattern). The mask uses a **fixed RNG seed**, so
the sample — and therefore `demo/expected_output.txt` — is reproducible.

Regenerate or resize it with:

```bash
python scripts/make_synthetic.py                 # the committed 32×32 sample
python scripts/make_synthetic.py --n 64 --keep 0.30 --iters 80
```

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8).

## Full (real) datasets

Real raw-k-space MRI datasets exist but **require registration / a data-use
agreement and cannot be redistributed here**. Use them for study by following each
provider's instructions (`scripts/download_data.*` prints these and never bypasses
any credential wall):

- **fastMRI** (NYU / Meta) — 1,500+ knee and 6,970+ brain raw-k-space scans.
  <https://fastmri.med.nyu.edu/> · <https://github.com/facebookresearch/fastMRI>
  (requires accepting the fastMRI data-use agreement).
- **Calgary-Campinas-359** — multi-channel brain MRI k-space.
  <https://sites.google.com/view/calgary-campinas-dataset/>
- **SKM-TEA** (Stanford knee MRI) — quantitative knee k-space + labels.
  <https://github.com/StanfordMIMI/skm-tea>

To adapt this project to real data, export a single slice's k-space and coil-combine
(or take one coil) into the text layout above, or extend the loader to read HDF5.
The reconstruction math is identical; only the loader and the coil dimension change.

## Provenance & honesty

The committed sample is **100% synthetic**, generated deterministically on the fly.
It carries a known ground-truth image *specifically* so the demo can prove that
compressed sensing recovers it from far fewer samples than a full scan — this is a
teaching illustration and **implies no clinical validity** whatsoever.
