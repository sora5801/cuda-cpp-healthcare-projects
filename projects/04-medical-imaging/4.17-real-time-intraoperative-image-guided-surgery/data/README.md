## Committed sample (`sample/surface_pair.txt`)

| Field | Value |
|---|---|
| File | `sample/surface_pair.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, seed 7) |
| License | Public domain (CC0) — it is synthetic, models no real patient |
| Size | < 3 KB (36 moving + 36 fixed points) |
| Units | millimetres (mm) |

This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8). It is engineered so the demo result
is meaningful and self-checking: ICP should drive the alignment error down to the
injected noise floor (~0.24 mm).

### File format

```
np nq
GT  r00 r01 r02 r10 r11 r12 r20 r21 r22  tx ty tz
<np lines>  x y z        # moving cloud P (pre-operative surface)
<nq lines>  x y z        # fixed  cloud Q (intra-operative surface)
```

- **`np nq`** — point counts of the two clouds.
- **`GT ...`** — the ground-truth rigid transform (3×3 rotation row-major, then a
  3-vector translation, mm) used to build `P` from `Q`. Recorded only so a reader
  can see how the sample was synthesized; the demo's correctness metric (RMS
  alignment error) does not use it.
- **`P` block** — the *moving* cloud: `Q` transformed by `GT` plus ~0.15 mm
  Gaussian sensor noise. This is the pre-op surface, deliberately misaligned.
- **`Q` block** — the *fixed* cloud: points on a curved organ-like patch (a
  Gaussian bump on a tilted plane). This is the intra-op surface ICP aligns to.

The parser lives in `src/reference_cpu.cpp::load_clouds`. The `GT` line is
optional — a file with no `GT` block (just `np nq` then the two point blocks) is
also accepted.

### Regenerate / resize

```bash
python scripts/make_synthetic.py                       # default 6x6 grid, seed 7
python scripts/make_synthetic.py --grid 40 --noise 0.3 # 1600 points, more noise
```

Everything it writes is **synthetic** and labeled as such.

## Full / real datasets

This teaching project uses synthetic clouds so it runs anywhere. Real
image-guided-surgery research uses these datasets (from the catalog); most are
video/volume corpora that require registration or credentials, so
`scripts/download_data.*` only prints instructions and links — it never bypasses
any login (CLAUDE.md §8).

- **Cholec80** laparoscopic surgery videos — <https://camma.u-strasbg.fr/datasets>
  (tool/phase annotations; registration required).
- **ReMIND2Reg 2025** brain-resection multimodal (pre-op MRI ↔ intra-op US/MRI)
  registration challenge — <https://arxiv.org/abs/2508.09649>.
- **EndoVis** (MICCAI) surgical-vision challenges —
  <https://endovis.grand-challenge.org/>.
- **SurgT** surgical-tool-tracking benchmark.

To turn a real surface into this project's format, sample 3-D points from the
two surfaces (e.g. mesh vertices from a segmented MRI/CT surface for `Q`, and
digitized/US-derived points for `P`) and write them in the layout above.

## Safety

**Not for clinical use.** Synthetic data only; no patient is represented, and the
recovered transform is an educational artifact, not a surgical navigation output.
