### What the synthetic phantom contains (and why)

`make_synthetic.py` builds a deterministic phantom engineered so the resulting DRR
is *interpretable* (PATTERNS.md §6 — "synthetic data that makes the demo
meaningful"):

| Structure | Hounsfield Unit | Linear attenuation μ (1/mm) | Role in the DRR |
|---|---|---|---|
| Air background | −1000 HU | ≈ 0 | transparent; rays through it add nothing |
| Soft-tissue sphere (large, centered) | +40 HU | ≈ 0.0198 | a faint, broad shadow |
| Bone sphere (small, **offset** from center) | +1000 HU | ≈ 0.0380 | the brightest pixel — deliberately off-axis |

Because the bone ball is offset toward +x/+y, the DRR is **asymmetric** and its
brightest pixel lands off-center. The program prints `max attenuation at (u,v)` so
you can confirm the ray geometry is wired up correctly (the demo reports `(76,61)`,
right of the 64-px center column — exactly the offset direction).

**Field meanings.** `nx,ny,nz` = voxel counts; `sx,sy,sz` = voxel spacing in mm
(here 2 mm isotropic, so the volume is 64×64×64 mm). Each body value is a
Hounsfield Unit; the loader converts HU → μ via `hu_to_mu()` (see
`src/drr_core.h`) so the ray loop integrates attenuation directly.

## Full / real datasets

The committed sample is synthetic so the demo is self-contained. Real CT volumes to
practice on (from the catalog "Datasets" column):

- **TCIA** (The Cancer Imaging Archive, <https://www.cancerimagingarchive.net/>) —
  large public collections of prostate/lung CT in DICOM; most are CC-BY. Convert a
  DICOM series to the `nx ny nz sx sy sz` + HU text format this loader expects (a
  few lines of `pydicom` / `SimpleITK`), or extend the loader to read DICOM.
- **Gold Atlas** male-pelvis MR/CT (verify the current URL; the catalog lists
  `goldenatlasproject.com`) — paired CT used in registration studies.
- **AAPM TG-132** registration test cases — standardized phantoms for validating
  image-registration accuracy.
- **Clinical CBCT + kV portal images** — only under institutional IRB; never
  redistributed here.

`scripts/download_data.ps1` / `.sh` print these pointers and do **not** attempt to
bypass any registration or credentials (CLAUDE.md §8). For credentialed sets the
scripts only print instructions; `make_synthetic.py` is always available as a
license-clean stand-in.

## Honesty & safety

This is **educational**. The phantom is synthetic, the geometry is a single fixed
teaching pose, and nothing here is validated for clinical use. Labeling: the
sample, this README, and the generator all say **synthetic** explicitly.
