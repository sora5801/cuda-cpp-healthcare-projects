# Data — 4.32 GPU-Accelerated Landmark Detection

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/heatmaps_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no patient data |
| Size | ~0.3 MB |
| Content | 5 landmark heatmaps over a 20×20×16 voxel grid (6400 voxels each) |

This file is the **input a landmark decoder consumes**: for each anatomical
landmark, a dense 3D *heatmap* — the kind of tensor a heatmap-regression network
(stacked hourglass, 3D U-Net) emits, one volume per landmark, high near the
landmark and low elsewhere. It lets `demo/run_demo` run **offline, with zero
downloads** (a hard requirement, CLAUDE.md §8).

### File format (whitespace-separated; the loader ignores line breaks)

```
nx ny nz L                     # grid extents + number of landmarks
# then, repeated L times:
cx cy cz                       # the ground-truth centre this heatmap was built around
v0 v1 v2 ... v{nx*ny*nz-1}     # the heatmap intensities, row-major (x fastest, then y, then z)
```

- **`nx ny nz`** — voxel extents of the (shared) grid. Flat index of voxel
  `(x,y,z)` is `((z*ny)+y)*nx + x`. This matches a `[Z,Y,X]` medical volume /
  PyTorch tensor in memory.
- **`L`** — number of landmarks = number of stacked heatmap volumes.
- **`cx cy cz`** — the *planted* ground-truth coordinate (voxel units, may be
  fractional). Used only to report decode error; a real prediction would not
  ship its own answer. Our synthetic blob is
  `H(x,y,z) = exp(-((x-cx)² + (y-cy)² + (z-cz)²) / (2σ²))`, σ = 1.5 voxels.
- **`v…`** — heatmap intensities in `[0,1]` (so the fixed-point weighting in
  `src/landmark.h` is valid).

### Why the centres are fractional

The integer argmax can only return the nearest *whole* voxel; the **soft-argmax**
centroid recovers the fractional part. Planting centres like `5.6` gives the
sub-voxel decoder something real to recover — see `err` in the demo output.

To regenerate or make a larger synthetic set:

```
python scripts/make_synthetic.py                                  # the committed sample
python scripts/make_synthetic.py --nx 64 --ny 64 --nz 64 --landmarks 26
```

## Full / real datasets

This project decodes *heatmaps*, so a "real" run needs a real 3D volume, a
trained landmark network to produce heatmaps, and ground-truth landmark
annotations. The annotated datasets below require **registration / a data-use
agreement**, so we do **not** auto-download them (`scripts/download_data.*`
prints instructions only and never bypasses credentials):

- **VerSe vertebral challenge** — <https://github.com/anjany/verse> — 374 CT
  scans, 26 vertebral landmarks each. License per the challenge terms.
- **RSNA Vertebral Fracture Detection** —
  <https://rsna-vertebral-labeling-level-detection.grand-challenge.org/> —
  challenge registration required.
- **CephaloNet** cephalometric landmark dataset (2D cephalograms).
- **MICCAI 2015 prostate challenge** landmark dataset.

To feed real data through this project, run a network (e.g. via
[MONAI](https://github.com/Project-MONAI/MONAI) or
[nnDetection](https://github.com/MIC-DKFZ/nnDetection)) to obtain per-landmark
heatmap tensors `[Z,Y,X]`, then export them into the format above.

## Honesty

All committed data is **synthetic** and labeled as such. Nothing here is a real
patient image, and no output may be used for diagnosis or treatment (CLAUDE.md §8).
