This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8).

### What the sample is, exactly

`make_synthetic.py` builds a sharp synthetic "specimen" — a dim background plus
five bright point sources (mimicking sub-resolution fluorescent beads/puncta) and
two thin bright lines (sharp edges) — then **blurs it** by circular convolution
with a normalized Gaussian PSF (`radius = 4`, `sigma = 1.5` pixels). **Only the
blurred image is written** to `blurred_image.txt`; the ground truth is withheld so
the C++ program has to recover the sharpness itself, exactly as a real microscope
user would.

Field meanings:
- **Intensity** values are unitless synthetic "photon counts" (non-negative). The
  background is `2.0`; the brightest bead is `120.0`.
- The blur parameters (`radius`, `sigma`) are a **shared contract**: the same
  numbers appear in `src/main.cu` (`PSF_RADIUS`, `PSF_SIGMA`) and in
  `make_synthetic.py`. The program deconvolves assuming this *known* PSF. If you
  regenerate the data with different blur, change both places or the demo's
  expected output will shift.

Everything is **deterministic** (no RNG), so `demo/expected_output.txt` is stable.

## Full dataset

There is no single canonical "full dataset" — deconvolution is applied to whatever
fluorescence stack you have, paired with the microscope's PSF. To study real data,
`scripts/download_data.ps1` / `.sh` point you at the standard sources:

- **EPFL Biomedical Imaging Group deconvolution benchmark** (raw + ground-truth
  stacks and **measured PSFs**): <https://bigwww.epfl.ch/deconvolution/>
- **BioImage Archive** fluorescence datasets: <https://www.ebi.ac.uk/biostudies/bioimages>
- **ImageJ/Fiji** sample images: <https://imagej.net/>

**License:** each source carries its own terms — respect them. We do **not**
redistribute any of them; the committed sample is synthetic. To feed a real image
into this project, export one 2-D slice to the text format above (a few lines of
Python with `tifffile` + `numpy.savetxt` will do it).
