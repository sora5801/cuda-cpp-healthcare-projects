# Data — 4.27 Radiomics Feature Extraction

## Committed sample (`sample/radiomics_sample.txt`)

| Field | Value |
|---|---|
| File | `sample/radiomics_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, deterministic) |
| License | Public domain (CC0) — it is synthetic, no patient data |
| Size | < 2 KB |
| Grid | `6 x 6 x 5` voxels, `Ng = 8` gray levels |
| ROI | a spherical mask of **56 voxels** |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is **SYNTHETIC** — it is
*not* a real scan and carries no clinical meaning.

### File layout (parsed by `load_volume` in `src/reference_cpu.cpp`)

```
line 1        : nx ny nz Ng            (grid dims + number of gray levels)
next nx*ny*nz : intensities            (row-major, x fastest, then y, then z)
next nx*ny*nz : mask flags 0/1         (1 = inside ROI, 0 = background)
```

- **intensities** — arbitrary units (think Hounsfield units on CT). Only voxels
  whose mask flag is `1` contribute to any feature; background voxels are ignored.
- **mask** — the segmentation. In real radiomics this comes from a radiologist's
  or an ML model's contour of the tumour/organ.

### How the sample is engineered (so the result is interpretable)

Inside the spherical ROI the generator lays down a **smooth radial gradient**
(correlated, near-diagonal GLCM structure) **plus a coarse checkerboard ripple**
(high-contrast neighbour pairs, `+20`/`-20`). The result: high GLCM `contrast`,
low `homogeneity`, and a slightly **negative** `correlation` — exactly the
signature of a checkerboard texture. That makes the printed features a *teachable*
readout of the input, not an opaque number.

Regenerate (or make a bigger synthetic volume) with:

```bash
python scripts/make_synthetic.py                       # the committed 6x6x5 sample
python scripts/make_synthetic.py --nx 16 --ny 16 --nz 12 --ng 16
```

## Full dataset (real radiomics cohorts)

The committed sample is synthetic on purpose; real radiomics uses large DICOM
CT/PET/MRI volumes with segmentation masks. See `scripts/download_data.ps1` /
`.sh` for pointers. Catalog dataset notes (verbatim):

> TCIA NSCLC-Radiomics (https://www.cancerimagingarchive.net/collection/nsclc-radiomics/) — 422 lung CTs with survival; RIDER Breast MRI (via TCIA); QIN-HEADNECK (via TCIA) — head and neck RT; TCGA collections (https://portal.gdc.cancer.gov/).

- **License:** TCIA collections carry their own (mostly permissive, some
  restricted) licenses. Respect each one; do **not** redistribute imaging here.
- **Credentialed sets:** the download script must **not** bypass registration —
  it prints instructions and links only, and `make_synthetic.py` provides a
  stand-in so the project always runs.
- **Format gap:** real data is DICOM/NIfTI; converting a real ROI into this
  project's simple text format (dense grid + mask, quantized levels) is left as
  an exercise (see README "Exercises"). Tools like `pydicom` / `SimpleITK` read
  the images and masks.

## Honesty / safety

The committed data is **synthetic** and labeled synthetic everywhere. No output
of this project is clinically valid — it is a teaching artifact only (CLAUDE.md §8).
