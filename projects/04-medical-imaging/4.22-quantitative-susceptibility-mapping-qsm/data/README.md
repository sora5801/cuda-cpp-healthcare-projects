This tiny file lets `demo/run_demo` run **offline, with zero downloads** — a hard
requirement for every project (CLAUDE.md §8).

### What the sample is, exactly

`make_synthetic.py` builds a known **susceptibility phantom** χ(**r**): a zero
background with four compact "sources" — three **paramagnetic** blobs (χ > 0, like
iron-rich deep-brain nuclei) and one **diamagnetic** blob (χ < 0, like a
calcification). It then applies the **dipole forward model** in k-space —
`field = IFFT3( D(k) · FFT3(chi) )` with `D(k) = 1/3 − kz²/|k|²` (B₀ ∥ z) — and
writes **only the resulting field map** to `field_map.txt`. The ground-truth χ is
**withheld** from the file; the C++ program must invert the dipole kernel to
recover it, exactly as a real QSM pipeline does.

Field meanings:
- Each value is a **field shift** (the local tissue field δB relative to B₀), in
  arbitrary units — a clean *local* field map (phase already unwrapped, background
  fields already removed; those stages are out of scope, see THEORY.md).
- The four source voxels and their χ values (`+1.0, +0.6, +0.8, −0.7`) are a
  **shared contract**: the same phantom is rebuilt in `src/main.cu`
  (`make_ground_truth_chi()`) to score recovery, and generated in
  `make_synthetic.py` (`make_ground_truth_chi()`). Change one, change both, or the
  recovery numbers and `demo/expected_output.txt` will shift.
- The reconstruction parameters (`TKD_THRESHOLD`, `TIK_ALPHA`, `TIK_STEP`,
  `TIK_ITERS`) live in `src/main.cu`.

Everything is **deterministic** (no RNG), so `demo/expected_output.txt` is stable
across runs and machines.

Regenerate or enlarge it with:

```bash
python scripts/make_synthetic.py                       # default 16x16x8
python scripts/make_synthetic.py --nx 24 --ny 24 --nz 16
```

(The generator uses a pure-Python O(N²) DFT with no dependencies; keep the grid
small or switch it to `numpy.fft` for larger volumes.)

## Full dataset

Real QSM data (multi-echo GRE magnitude + phase, or preprocessed field maps) is
large and/or requires registration. `scripts/download_data.ps1` / `.sh` print the
sources and instructions; they never bypass credentials.

- **QSM Reconstruction Challenge 2.0** — benchmark data + reference reconstructions
  on Zenodo: <https://doi.org/10.1101/2020.11.25.397695>
- **HCP 7T multi-echo GRE** (Human Connectome Project): <https://db.humanconnectome.org/>
- **AHEAD** (Amsterdam Ultra-high-field Adult lifespan Database) — ultra-high-field
  7T MRI including QSM-ready data.
- **UK Biobank** (credentialed): <https://www.ukbiobank.ac.uk/>

**License:** each source carries its own terms and most require registration —
respect them. We do **not** redistribute any of them; the committed sample is
synthetic. To feed a real local field map into this project, export the 3-D volume
to the text format above (a few lines of Python with `nibabel` + `numpy` will do
it), matching the `nx ny nz` header and x-fastest ordering.

## Provenance & honesty

- The committed sample is **synthetic** and is labeled synthetic everywhere it
  appears. It contains no patient-derived data.
- Recovered χ is in **arbitrary O(1) units**, not physical ppm — a scaling choice
  so the demo prints legibly. The k-space physics is unchanged.
- This is **study material, not a clinical tool**. Nothing here is validated for
  diagnosis or any medical decision (CLAUDE.md §8).
