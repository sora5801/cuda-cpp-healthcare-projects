### Why synthetic?

Every real multi-site imaging dataset (ABIDE, ADNI, UK Biobank, IXI — see
[`scripts/download_data.sh`](../scripts/download_data.sh)) requires registration or a
data-use agreement, and most **forbid redistribution**. Committing any of them would
violate their license (CLAUDE.md §8). So the demo ships a small, clearly-labeled
synthetic table that reproduces the exact phenomenon ComBat fixes.

### How it is generated (interpretable by construction)

For sample *n* on scanner *b*, feature *p*, the generator draws

```
    y = alpha_p  +  beta_p · age_n  +  gamma_{p,b}  +  delta_{p,b} · noise
        └ grand ┘   └── biology ───┘  └ scanner loc ┘ └── scanner scale ──┘
```

so the **scanner location/scale signature** (`gamma`, `delta`) and the **biological
age signal** (`beta`) are *known*. A correct ComBat run collapses the across-scanner
mean gap toward zero **while preserving** the age→feature relationship — which is exactly
what the demo reports. A fixed RNG seed (42) makes the file and
[`demo/expected_output.txt`](../demo/expected_output.txt) reproducible.

## File format (what the loader in `src/reference_cpu.cpp` expects)

Whitespace-separated text; lines beginning with `#` are comments and are skipped.

```
# optional comment/banner lines
N P B C                      # header: samples, features, batches, covariates
b_0 b_1 ... b_{N-1}          # batch (scanner) label per sample, each in [0, B)
c_0 ... c_{C-1}              # covariate row for sample 0   (C values)
...                          # ... N covariate rows total   (omitted if C == 0)
y_0 y_1 ... y_{N-1}          # feature row 0: the feature's value for each sample
...                          # ... P feature rows total
```

Each **feature is a row** (P rows of N values) because ComBat processes features
independently — one GPU thread owns one feature row (see THEORY §GPU mapping).

## Getting / using real data

Run [`scripts/download_data.ps1`](../scripts/download_data.ps1) (or `.sh`) for the official
links and instructions. ComBat works on **extracted features** (e.g. FreeSurfer regional
volumes or cortical thickness, or radiomic features), not raw voxels: export a feature
table per subject, tag it with the scanner/site label and covariates, and write it in the
format above. Compare your output against **NeuroComBat**
(<https://github.com/Jfortin1/ComBatHarmonization>).

> **No clinical use.** Synthetic data is labeled synthetic everywhere; nothing here implies
> diagnostic or clinical validity.
