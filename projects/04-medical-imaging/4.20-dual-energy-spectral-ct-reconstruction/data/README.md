# Data — 4.20 Dual-Energy / Spectral CT Reconstruction

## Committed sample (`sample/dect_sinogram_sample.txt`)

| Field | Value |
|---|---|
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`, seed 4200) |
| License | Public domain (CC0) — it is synthetic, no patient data |
| Size | 24 dual-energy sinogram bins with ground-truth path lengths (~1 KB) |
| Why synthetic | Bins are generated by the **same polychromatic forward model** the code inverts, with **known** basis-material path lengths, so the demo can report a real recovery error against a known answer — and it is fully **deterministic**. |

### File format

```
# comment lines (ignored) may appear anywhere
<n> <has_truth>                       # e.g. "24 1"  (has_truth = 1 -> truth columns follow)
<m_lo> <m_hi> <true_t1> <true_t2>     # one line per bin (truth columns only if has_truth=1)
...                                   # n data lines total
```

- `m_lo`, `m_hi` — the measured **log-attenuation** of the bin at the low-kVp
  (~80 kVp) and high-kVp (~140 kVp) spectra, i.e. `-ln(I/I0)`. Dimensionless.
- `true_t1`, `true_t2` — the ground-truth **basis-material path lengths** in cm:
  `t1` = water-equivalent tissue thickness, `t2` = iodine-equivalent thickness
  along the ray. Present because the sample is synthetic; a real scan has no
  ground truth, so `has_truth = 0` and those columns are absent.

The loader is `src/reference_cpu.cpp::load_sinogram`. The scanner physics (the
two spectra and the two attenuation curves) is **not** in this file — it is built
deterministically in code by `build_spectral_model()`, and `make_synthetic.py`
mirrors those exact formulae so the data is self-consistent.

## Full / real datasets

This teaching sample stands in for real dual-energy and photon-counting CT data.
To work with real spectral CT (raw sinograms or DICOM), see:

- **AAPM Spectral CT / DL-Spectral-CT challenge data** — verify the current URL
  at <https://www.aapm.org/> (grand-challenge pages).
- **MARS photon-counting CT datasets** — <https://www.marsbioimaging.com/>.
- **TCIA** (The Cancer Imaging Archive) DECT collections —
  <https://www.cancerimagingarchive.net/>.
- **XCAT phantom** simulated DECT (widely used in the literature) — requires a
  license from Duke.

`scripts/download_data.ps1` / `.sh` print pointers and instructions for these;
they do **not** attempt to bypass any registration/credential requirement
(CLAUDE.md §8). For more synthetic bins instead:

```
python scripts/make_synthetic.py --n 100000
```

Realistic attenuation curves come from **NIST XCOM** cross-sections
(<https://physics.nist.gov/PhysRefData/Xcom/html/xcom1.html>) and real tube
spectra from tools like **SpekPy**; the analytic curves here are simplified
stand-ins (see `THEORY.md` → "Where this sits in the real world").

## Provenance & honesty

The committed sample is **synthetic** and labeled as such everywhere. The numbers
are physically plausible but not from any scanner or patient. **Not for clinical
use** — this is study material only. No diagnostic or material-quantification
conclusion may be drawn from this sample.
