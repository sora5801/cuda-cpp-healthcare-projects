# Data — 4.5 PET Image Reconstruction

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/sinogram_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic |
| Size | ~4 KB |
| Contents | A noisy PET **sinogram** (coincidence counts) forward-projected from a known two-disc emission phantom, with Poisson noise baked in at a fixed seed |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). Because we generated it from
a *known* image, the reconstruction is interpretable: MLEM must recover a bright
central disc plus a smaller off-center hot spot (see `demo/expected_output.txt`).

### File format

```
K D ds N W iters          # header (one line)
<row 0: D count values>   # sinogram row for angle theta_0
<row 1: D count values>   # ... angle theta_1
...                       # K rows total
```

| Symbol | Meaning | Units |
|---|---|---|
| `K` | number of projection angles (`theta_k = k·π/K`, parallel-beam over 180°) | — |
| `D` | detector bins per angle | — |
| `ds` | detector bin spacing | world units |
| `N` | reconstructed image side length (image is `N×N`) | pixels |
| `W` | image half-width; the image covers `[-W, +W]²` | world units |
| `iters` | advisory MLEM iteration count (main.cu uses it if > 0) | — |
| counts | `counts[k·D + j]` = measured coincidences on LOR (angle `k`, bin `j`) | counts |

Counts are non-negative integers-as-floats (Poisson samples). The mapping from a
pixel to its detector bin at each angle is defined once in
[`../src/pet_geometry.h`](../src/pet_geometry.h) and used by both the generator
and the C++ solver, so the committed data is exactly consistent with the model
the reconstruction inverts.

Regenerate (or make a larger synthetic problem) with:

```bash
python scripts/make_synthetic.py                                  # the committed tiny sample
python scripts/make_synthetic.py --N 64 --K 60 --D 91 --iters 40  # bigger, still synthetic
```

## Full / real datasets

The committed sample is enough to run and study the demo. Real PET data is large
and often needs registration, so it is **not** committed. `scripts/download_data.*`
prints pointers (it never bypasses credentials). Catalog dataset notes (verbatim):

> OpenNEURO PET datasets (https://openneuro.org/); TCIA PET collections
> (https://www.cancerimagingarchive.net/); PETRIC challenge datasets
> (https://github.com/SyneRBI/PETRIC); Siemens mMR phantom datasets (publicly
> available through STIR/SIRF).

- **PETRIC / SIRF-Exercises** ship openly-usable phantom sinograms in
  Interfile/STIR format — the most direct path to real (non-clinical) data.
- **TCIA / OpenNEURO** host reconstructed clinical PET volumes; using them for
  *reconstruction* requires the raw sinogram or list-mode data, which is less
  commonly public. Respect every collection's license and de-identification terms.

**Not for clinical use.** This project is educational (CLAUDE.md §8). The
committed data is synthetic and labeled synthetic everywhere it appears; nothing
here implies clinical validity.

## Provenance & honesty

- The sample is a **digital phantom**, not a real patient scan.
- Poisson noise is generated with a fixed seed and stored in the file, so the
  program itself is deterministic (it never draws random numbers at run time).
- The geometry is a simplified **2-D parallel-beam** rebinning; a real scanner is
  a 3-D ring with fan/oblique LORs, attenuation, scatter, and randoms (see
  [`../THEORY.md`](../THEORY.md) "Where this sits in the real world").
