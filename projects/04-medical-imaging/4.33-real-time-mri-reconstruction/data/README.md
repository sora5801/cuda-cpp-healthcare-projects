### File layout

```
line 1 :  n  n_spokes  n_ro  win  stride  n_frames  kb_w  has_truth
n_spokes*n_ro rows (spoke-major, readout-minor; index = spoke*n_ro + readout):
          re  im
n*n rows (iff has_truth; row-major image order):
          truth
```

| Symbol | Meaning | Units |
|---|---|---|
| `n` | reconstructed image / gridding side length (power of two) | pixels |
| `n_spokes` | total golden-angle radial spokes acquired | count |
| `n_ro` | readout samples per spoke (along the spoke diameter) | count |
| `win` | sliding-window size: spokes used to reconstruct ONE frame | count |
| `stride` | spokes advanced between consecutive frames | count |
| `n_frames` | number of sliding-window frames (the movie length) | count |
| `kb_w` | Kaiser-Bessel gridding kernel width | grid cells |
| `has_truth` | 1 if a `truth` block is present, else 0 | flag |
| `re`, `im` | measured complex k-space value of a (spoke, readout) sample | arbitrary MR units |
| `truth` | ground-truth magnitude image pixel (synthetic; at the last frame's time) | arbitrary MR units |

The committed sample is a **32×32** reconstruction from **64 golden-angle spokes**
of **64 readout** samples each. The sliding window uses **21 spokes/frame**, advances
**8 spokes/frame**, and produces **6 frames**. The Kaiser-Bessel kernel is **width 4**.
Because the spoke geometry and the phantom's analytic Fourier transform are both
computed exactly (no RNG), the sample — and therefore `demo/expected_output.txt` — is
fully reproducible.

Regenerate or resize it with:

```bash
python scripts/make_synthetic.py                                  # the committed sample
python scripts/make_synthetic.py --n 64 --spokes 128 --win 34 --frames 8
```

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is a
hard requirement for every project (CLAUDE.md §8).

### Why radial (and not Cartesian) data?

Real-time cardiac/interventional MRI acquires k-space along **radial spokes at the
golden angle** (≈111.25° apart), streaming spoke by spoke. Any contiguous run of
golden-angle spokes covers k-space near-uniformly, so a **sliding window** of the most
recent spokes forms a fresh image at every time step — that is what makes the movie
"real-time". Radial samples do **not** lie on the FFT grid, so the reconstruction is a
**gridding NUFFT** (density-compensate → Kaiser-Bessel grid → inverse FFT → deapodize),
which is exactly what this project teaches. (Sibling project **4.03** covers the
*Cartesian* compressed-sensing case; this one is the *non-Cartesian* streaming case.)

## Full (real) datasets

Real dynamic-MRI raw-k-space datasets exist but **require registration / a data-use
agreement and cannot be redistributed here**. Use them for study by following each
provider's instructions (`scripts/download_data.*` prints these and never bypasses any
credential wall):

- **CMRxRecon 2023** — cardiac MRI reconstruction challenge (multi-coil raw k-space).
  <https://cmrxrecon.github.io/>
- **ACDC** — Automated Cardiac Diagnosis Challenge (cine cardiac MRI).
  <https://www.creatis.insa-lyon.fr/Challenge/acdc/>
- **OCMR** — open cardiovascular MRI raw data, including real-time free-breathing
  acquisitions. <https://ocmr.info/>

To adapt this project to real data, export a single slice's radial k-space (or resample
Cartesian k-space onto a radial trajectory), coil-combine (or take one coil), and write
it in the layout above. The gridding math is identical; only the loader, the trajectory,
and the coil dimension change.

## Provenance & honesty

The committed sample is **100% synthetic**, generated deterministically on the fly. It
carries a known ground-truth image *specifically* so the demo can prove that gridding
recovers the anatomy from sparse radial samples and that the sliding window tracks
motion — this is a teaching illustration and **implies no clinical validity** whatsoever.
