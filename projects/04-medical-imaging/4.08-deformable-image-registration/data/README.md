# Data — 4.8 Deformable Image Registration

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/dir_pair.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`; no RNG, fully reproducible) |
| License | Public domain (CC0) — it is synthetic, **not** patient-derived |
| Size | ~72 KB (64×64 fixed + moving images, text) |
| Format | line 1: `nx ny`; then `nx*ny` **fixed** intensities; then `nx*ny` **moving** intensities |

All values are whitespace-separated grayscale intensities in `[0,1]`, row-major (index `= y*nx + x`).
This tiny file lets `demo/run_demo` run **offline, with zero downloads**, a hard requirement for every
project (CLAUDE.md §8).

### What the sample contains (and why it is designed this way)

- **Fixed image `F`:** a smooth 2-D **Gaussian blob** (σ ≈ 9 px) centered at (32, 32). It stands in for an
  anatomical structure.
- **Moving image `M`:** the **same** blob, but **translated** by (+5, +3) px **and** stretched ~12% wider in
  x. So the true deformation is *not* a pure translation — it varies across the image, which forces a genuine
  deformable (per-pixel) field, not a single global shift.

Both images are smooth, so the intensity **gradient is defined everywhere** — exactly what the Demons force
needs (a flat image carries no direction to move along). A correct registration warps `M` onto `F`, driving
the SSD from ~51.8 to ~0.06 (a 99.9% reduction) and recovering a displacement field whose magnitude (~4.5 px
mean) matches the built-in shift+stretch. Because the ground-truth motion is *known*, the demo result is
verifiable, not just plausible.

Regenerate or resize it:

```bash
python scripts/make_synthetic.py                          # the committed 64x64 pair
python scripts/make_synthetic.py --nx 128 --ny 128 --shift 8.0   # a bigger problem
```

## Full dataset

There is **nothing to download** for the demo — this project makes its own synthetic pair. For real
registration work, `scripts/download_data.ps1` / `.sh` print pointers to the standard benchmarks below. Each
requires you to register / accept a data-use license yourself; the scripts **never** bypass that, and none of
this data is committed here (license + patient privacy).

Catalog dataset notes (verbatim):

> OASIS brain MRI (https://www.oasis-brains.org/) — used in Learn2Reg challenge; Learn2Reg 2022 challenge
> (https://learn2reg.grand-challenge.org/) — lung, brain, abdominal; DIR-Lab lung CT deformation dataset
> (https://dir-lab.com/); 4D-CT lung datasets for respiratory motion.

- **Learn2Reg** (https://learn2reg.grand-challenge.org/) — CT/MR lung, brain, abdominal pairs with an
  evaluation server; the standard modern DIR challenge.
- **OASIS** (https://www.oasis-brains.org/) — brain MRI used in the Learn2Reg inter-subject task.
- **DIR-Lab** (https://dir-lab.com/) — 4D-CT respiratory lung pairs with expert landmark points; the
  gold-standard target-registration-error (TRE) benchmark.

To use real data with this project, convert one 2-D slice of a fixed/moving pair into the text format above
(normalize intensities to `[0,1]`). **Do not commit** the converted data.
