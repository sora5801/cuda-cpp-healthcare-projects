**This is SYNTHETIC data**, generated deterministically — **not** a real patient
CT, and the densities are **not** Hounsfield Units. It exists so `demo/run_demo`
runs **offline with zero downloads** (a hard requirement, CLAUDE.md §8), and so the
demo has a *known answer* (a planted polyp) to recover.

### File format

```
nx ny nz iso step max_steps width height      <- header (8 numbers)
<nx*ny*nz density floats>                      <- body (x fastest, then y, then z)
```

| Field | Meaning | Units |
|---|---|---|
| `nx ny nz` | volume dimensions | voxels |
| `iso` | iso-value of the lumen↔wall surface we render | density (0..1) |
| `step` | ray-march step length | voxels (`<1` oversamples) |
| `max_steps` | safety cap on march steps per ray | count |
| `width height` | output frame size | pixels |
| density `d[(z·ny+y)·nx+x]` | tissue density: **air-filled lumen ≈ 0**, **wall ≈ 1** | unitless |

### What the synthetic scene contains

- An **air-filled lumen**: a cylinder along +z (the fly-through axis) with a gentle
  sideways bend, density ≈ 0 inside.
- A **soft-tissue wall**: a smooth density ramp from 0 (lumen) to 1 (tissue) across
  ~3 voxels, crossing `iso = 0.5` at the wall surface. The smooth ramp makes the
  gradient (and thus the shading) well behaved.
- One **polyp**: a sphere bulging into the lumen on the −y wall, partway down the
  tube. It is the recoverable known answer — it reads ~2× brighter than the wall.

Regenerate or enlarge it:

```bash
python scripts/make_synthetic.py                       # default 32x32x48 sample
python scripts/make_synthetic.py --nx 96 --width 256   # a larger problem
```

## Full dataset

Real CT colonography volumes (to replace the synthetic stand-in) are public but
large and patient-derived — we do **not** commit them. `scripts/download_data.*`
prints how to obtain them and never bypasses any registration:

- **TCIA CT Colonography** —
  <https://wiki.cancerimagingarchive.net/display/Public/CT+Colonography> — the
  canonical public CTC collection (supine/prone DICOM series). License: TCIA Data
  Usage Policy / per-collection terms; respect them. Requires accepting terms /
  using the NBIA Data Retriever.
- **MICCAI 2018 colon challenge**, **ACR Lung-RADS CT**, **NLST CTC subsets** — see
  the catalog notes; several require registration.

To use a real volume you would: segment the air-filled lumen (a 3-D CNN, e.g. via
MONAI), resample to a dense grid, write it in the format above (or adapt the
loader to read DICOM/NIfTI), and place the camera on the extracted centerline.
That segmentation/centerline work is the rest of the pipeline (THEORY §7), out of
scope for this rendering-focused teaching project.
