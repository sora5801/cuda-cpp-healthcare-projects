| Field | Value |

| Size | ~7 KB (32×32 grayscale) |
| Layout | line 1: `<w> <h>`; then `w*h` floats in `[0,1]`, row-major (one image row per text line) |

This tiny file is a **ground-truth HIGH-RES image**. The program derives the
low-res input from it (a 2×2 box-average downsample), super-resolves it back to
high-res, and scores PSNR against this file. That lets `demo/run_demo` run
**offline, with zero downloads** (a hard requirement, CLAUDE.md §8).

### What the phantom contains (and why)

It is a deliberately **interpretable** SR target, not a real scan:

- a mid-gray background (0.15) — the "air/soft-tissue" baseline;
- two overlapping **Gaussian disks** — smooth, low-frequency blobs (easy to
  upscale, like a soft-tissue window);
- a band of **vertical bars** near the top — a classic resolution target whose
  sharp edges test whether SR recovers high frequencies;
- a thin bright **ring** — a one-pixel-wide sharp edge (the hardest thing for a
  degrade→upscale round-trip to recover).

Because it mixes smooth and sharp content, the demo can show the learned network
beating naive nearest-neighbour upscaling in PSNR. **It is synthetic** — no
clinical meaning, no diagnosis.

Regenerate or resize it with:

```bash
python scripts/make_synthetic.py --size 64
```

## Full (real) datasets

The catalog lists real CT/MRI SR datasets. This teaching project does **not**
redistribute them — they require registration and/or forbid redistribution.
`scripts/download_data.ps1` / `.sh` print the links and instructions but never
bypass credentials (CLAUDE.md §8):

| Dataset | URL | Notes |
|---|---|---|
| HCP 7T/3T paired brain MRI | https://db.humanconnectome.org/ | Registration + data-use terms; paired high/low-field MRI is the canonical SR training set. |
| fastMRI | https://fastmri.med.nyu.edu/ | Registration; k-space + images, used implicitly for SR evaluation. |
| IXI brain MRI | https://brain-development.org/ixi-dataset/ | CC BY-SA 3.0; multi-sequence brain MRI, commonly used for SR. |
| MSD (Medical Segmentation Decathlon) | http://medicaldecathlon.com/ | CC BY-SA 4.0; CT/MRI tasks repurposed for resolution enhancement. |

To use a real volume, extract one axial slice, normalize intensities to `[0,1]`,
crop/pad both dimensions to a multiple of `SR_SCALE`, and write it in the text
format above. The program then treats that slice as the ground-truth HR image.

## Provenance & field meanings

- **Provenance:** 100% synthetic, produced on-machine by `make_synthetic.py`
  (deterministic — no RNG seed dependence). No patient-derived data is committed.
- **Field meaning:** each number is a normalized pixel intensity in `[0,1]`
  (0 = black, 1 = white). There are no physical units; a real CT slice would map
  Hounsfield units into `[0,1]` before this pipeline.
- **Safety:** educational only. Nothing here is a real image, and the SR output
  must not be used for any medical decision (CLAUDE.md §8).
