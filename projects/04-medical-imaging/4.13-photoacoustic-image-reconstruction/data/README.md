### File format (parsed by `src/reference_cpu.cpp::load_pa`)

Whitespace-separated text:

```
line 1 (header):  n_sensors  n_samples  dt  c  img  world_half
next n_sensors lines:  sx  sy                     (sensor position, metres)
next n_sensors lines:  g0 g1 ... g{n_samples-1}   (that sensor's pressure trace)
```

Field meanings and units:

| Field | Meaning | Units |
|---|---|---|
| `n_sensors` | number of point sensors on the ring | — |
| `n_samples` | time samples recorded per sensor | — |
| `dt` | sample period (here 5e-8 s = 20 MHz) | seconds |
| `c` | speed of sound (here 1500, soft tissue) | m/s |
| `img` | reconstruction image side (image is `img × img`) | pixels |
| `world_half` | image spans `[−world_half, +world_half]²` (here ±10 mm) | metres |
| `sx, sy` | sensor position on the ring (radius 20 mm) | metres |
| `g…` | recorded pressure at each time sample | arbitrary pressure units |

### How the sample is built (and its known answer)

`scripts/make_synthetic.py` places three **point absorbers** at known world
coordinates (the strongest at the origin) and, for each sensor, superposes a short
Gaussian pulse arriving at the physical travel time `|sensor − absorber| / c`,
scaled by `1/distance`. This is a simple analytic **forward model**, not a full
acoustic simulation. Because delay-and-sum reconstruction reads each trace at
exactly those travel times, the reconstruction must peak at the planted absorber
locations — a built-in, verifiable ground truth (see `demo/expected_output.txt`).

Regenerate (deterministic — identical every run):

```bash
python scripts/make_synthetic.py                       # the committed sample
python scripts/make_synthetic.py --sensors 256 --samples 1024 --img 256   # bigger
```

## Full / real datasets

This project needs **no** download to run. To study real photoacoustic data, run
`scripts/download_data.ps1` / `.sh` (they print the pointers below and **never**
bypass any registration). Verify each URL and **respect every license** — if a set
forbids redistribution, keep the committed sample synthetic.

Catalog dataset notes (verbatim):

> k-Wave simulation datasets (generated locally); USCT (Ultrasound Computed Tomography) benchmark data (verify URL); in vivo photoacoustic datasets from Nature Communications publications (open access); PASCAA challenge datasets (verify URL at photoacoustics.org).

- **k-Wave** (http://www.k-wave.org/) — the standard toolbox; generate physically
  faithful traces with its k-space solver, then reconstruct here.
- **k-Wave-Fluid-CUDA** (https://github.com/klepo/k-Wave-Fluid-CUDA) — CUDA forward solver.
- **PyTomography** (https://github.com/lukepolson/pytomography) — GPU tomography incl. PA.
- **In-vivo PA datasets** accompanying open-access Nature Communications papers.
- **PASCAA / IPASC challenge data** — verify URL at photoacoustics.org.
