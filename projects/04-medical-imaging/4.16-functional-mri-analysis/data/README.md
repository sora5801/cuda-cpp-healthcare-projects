# Data — 4.16 Functional MRI Analysis

## Committed sample (`sample/`)

| Field | Value |
|---|---|
| File | `sample/fmri_sample.txt` |
| Origin | **Synthetic** (generated by `scripts/make_synthetic.py`) |
| License | Public domain (CC0) — it is synthetic, no real subjects |
| Size | ~32 KB |
| Contents | 48 voxels × 80 scans of a simulated task-fMRI BOLD experiment |

This tiny file lets `demo/run_demo` run **offline, with zero downloads**, which is
a hard requirement for every project (CLAUDE.md §8). It is **synthetic** — it does
**not** come from any human subject and carries no clinical meaning.

### File layout (plain text, whitespace-separated)

```
V  T  TR_seconds  block_scans          <- header (one line)
active_flag  y_0 y_1 ... y_{T-1}        <- voxel 0
active_flag  y_0 y_1 ... y_{T-1}        <- voxel 1
...                                      (V rows total)
```

| Symbol | Meaning | Units |
|---|---|---|
| `V` | number of voxels (simulated brain locations) | count |
| `T` | number of scans / time points | count |
| `TR_seconds` | repetition time = seconds between scans | s |
| `block_scans` | task boxcar half-period (ON for this many scans, then OFF) | scans |
| `active_flag` | ground truth: `1` if task signal was injected into this voxel, else `0` | — |
| `y_t` | the BOLD signal of this voxel at scan `t` | arbitrary BOLD units (~100 baseline) |

The `active_flag` column is **labels only** — the GLM never reads it. `main.cu` uses
it *after* the fit to report how many of the top-t voxels were truly active, which
recovers the planted answer (docs/PATTERNS.md §6) and proves the pipeline works.

### How the sample is built (so it is meaningful and reproducible)

Every 6th voxel is "active." An active voxel's series is
`baseline + drift·ramp(t) + amplitude·task_regressor(t) + noise`, where
`task_regressor` is the **same** task-boxcar-convolved-with-HRF the C++ GLM fits
(`src/glm.h`). Inactive voxels get only baseline + drift + noise. Noise is a small
deterministic LCG, so re-running `make_synthetic.py` reproduces the file
byte-for-byte and `demo/expected_output.txt` stays stable.

Regenerate (or make a bigger synthetic brain) with:

```bash
python scripts/make_synthetic.py                 # the committed tiny sample
python scripts/make_synthetic.py --V 200 --T 240 # larger synthetic problem
```

## Full (real) dataset

The committed sample is synthetic on purpose. Real task/resting fMRI is large,
distributed as NIfTI volumes, and usually needs preprocessing and/or credentials.
`scripts/download_data.ps1` / `.sh` print pointers to the standard public sources
and **never bypass registration**:

- **HCP** — Human Connectome Project, resting-state + task fMRI (3T/7T):
  <https://db.humanconnectome.org/> (registration required).
- **OpenNeuro / OpenFMRI** — thousands of BIDS-format datasets, many openly
  downloadable: <https://openneuro.org/>.
- **ABIDE** — Autism Brain Imaging Data Exchange:
  <http://fcon_1000.projects.nitrc.org/indi/abide/>.
- **UK Biobank** — imaging cohort (application + approval required):
  <https://www.ukbiobank.ac.uk/>.

**License / safety:** respect every dataset's license. Credentialed sets (HCP,
UK Biobank) must not be redistributed here; the synthetic sample stands in so the
demo always runs. Never present any output as clinically valid — this is study
material only (CLAUDE.md §8).

## Provenance & honesty

- The committed `sample/fmri_sample.txt` is **100% synthetic**, generated locally
  with a fixed seed. No patient data is included anywhere in this project.
- The BOLD "units" are arbitrary (centred near 100). They are not calibrated to any
  scanner and imply nothing physiological beyond the toy HRF response.
