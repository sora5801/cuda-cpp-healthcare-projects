### What it represents (and why)

The file stores **two blocks** in row-major `(z, y, x)` order (x fastest), so a
voxel `(z,y,x)` is at flat index `((z*H)+y)*W + x` in each block — exactly the
layout `Volume::idx` and the GPU kernels use, so a voxel maps to the same memory
address on both sides:

1. **Intensities** (floats ≈ `[0.2, 1.1]`) — a stand-in for a small CT/MRI crop:
   a gentle "soft-tissue" background gradient (~0.3) with a single **bright
   spherical "lesion"** (smoothed intensity ~0.5–1.0) at a known, slightly
   off-center location, plus mild Gaussian noise and a soft (cosine-tapered)
   lesion edge that mimics a partial-volume boundary.
2. **Ground-truth labels** (`0` = background, `1` = lesion) — `1` for every voxel
   inside the sphere of radius 3. The demo scores the network's predicted mask
   against this with the **Dice coefficient** (`2|A∩B| / (|A|+|B|)`), so the
   number it prints is a real accuracy, not just CPU==GPU agreement.

Because the lesion's position is known, the result is **interpretable**: the demo
prints the central-slice mask and a Dice of ≈ 0.96, proving the (hand-set) 3D
conv head actually finds the lesion.

> **Synthetic, not clinical.** This volume is generated math, labelled synthetic
> everywhere it appears. Nothing here is a real scan and no output is a diagnosis.

## Full dataset (real medical segmentation data)

Real volumetric segmentation uses large, labelled CT/MRI archives. `scripts/
download_data.{ps1,sh}` print pointers and instructions only; several require
registration and **must not be bypassed** (CLAUDE.md §8). Catalog sources
(verbatim):

> Medical Segmentation Decathlon (http://medicaldecathlon.com/) — 10 tasks,
> ~2,500 volumes total; TotalSegmentator training set (Zenodo, ~1,200 CT with
> 117 structure labels; https://zenodo.org/record/6802614); KiTS23 kidney tumor
> challenge (https://kits-challenge.org/kits23/); BraTS brain tumor dataset
> (https://www.synapse.org/#!Synapse:syn27046444).

- **License:** respect each dataset's terms. Most forbid redistribution, so the
  committed sample here is **synthetic** and `make_synthetic.py` is the stand-in.
- **Format:** real volumes ship as NIfTI (`.nii.gz`) or DICOM with a paired label
  map; loading those needs a reader (e.g. NIfTI/ITK) — out of scope for this
  teaching build, which uses a plain text volume so the loader stays readable.

## Provenance & field meanings

| Block | Meaning | Units / range |
|---|---|---|
| `D H W` | volume dimensions (depth, height, width) | positive integers |
| intensities | per-voxel image intensity | dimensionless, ≈ `[0.2, 1.1]` (rescaled) |
| labels | ground-truth class | `0` background, `1` lesion |

Never imply clinical validity; synthetic data is labelled synthetic everywhere.
