# Push 2026-06-29 #05 -- flagship 13.02 pbpk

> Push-note (CLAUDE.md §7.1). Twelfth Phase 1 flagship — pharmacology / quantitative modeling.

## 1. Summary

The pharmacology flagship is done: **13.02 PBPK at Scale**, a complete, verified GPU virtual-population
pharmacokinetic simulator. Each of thousands of virtual patients is an independent multi-compartment PBPK
ODE solved with RK4 (one thread per patient), with per-patient physiology sampled log-normally. Double
precision keeps GPU and CPU in lock-step (~1e-14), and the population mean AUC reproduces the textbook
dose/CL identity.

## 3. New projects (didactic blurb)

**13.02 PBPK** applies the ensemble-ODE pattern (cf. 9.02) to a multi-compartment pharmacokinetic model with
**per-patient parameter sampling**: each thread seeds a reproducible RNG, draws its physiology log-normally,
runs the full RK4 loop in registers, and reports Cmax/Tmax/AUC. The standout file is `src/pbpk.h` + THEORY
§4: on-device log-normal sampling via a shared RNG so CPU and GPU build the identical population, and why the
population mean AUC must equal dose/CL.
