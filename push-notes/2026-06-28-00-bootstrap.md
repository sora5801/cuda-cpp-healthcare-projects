## 1. Summary

This push bootstraps **`cuda-cpp-healthcare-projects`**: a didactic collection of **301 self-contained CUDA
C++ projects** across 14 biomedical domains. It lays down the contract, the tooling, a **validated**
project template, the standards docs, and **301 scaffolded project skeletons** — each a buildable,
heavily-commented SAXPY placeholder ready to be filled in. The build pipeline was proven end-to-end on real
hardware before scaffolding, so every project compiles and runs on day one.

## 3. New projects (didactic blurb)

No project is *done* yet — this push delivers **skeletons**. Each skeleton is the template's **SAXPY**
(`out = a·x + y`) placeholder: the canonical first CUDA kernel, used to demonstrate the full required
shape — a `__global__` kernel with a one-thread-per-element mapping and a ragged-last-block guard, a host
wrapper performing the five canonical CUDA steps (alloc → H2D → launch → D2H → free), a CPU reference for
verification, CUDA-event timing, and a demo that diffs deterministic stdout against `expected_output.txt`.
The single most interesting thing to look at is [`docs/PROJECT_TEMPLATE/src/kernels.cu`](../docs/PROJECT_TEMPLATE/src/kernels.cu)
— it is the bar every real kernel must clear for comment density and clarity.

## 4. How to build & run

```powershell
# regenerate everything from the catalog (idempotent)
python tools/catalog.py
python tools/scaffold.py
python tools/status.py

# build + run any skeleton (example: the drug-discovery flagship slot)
cd projects/01-drug-discovery/1.12-molecular-fingerprint-similarity-search
msbuild build/molecular-fingerprint-similarity-search.sln /p:Configuration=Release /p:Platform=x64
./demo/run_demo.ps1            # -> RESULT: PASS (GPU matches CPU)

# check a project against the Definition of Done
python tools/verify_project.py projects/01-drug-discovery/1.12-molecular-fingerprint-similarity-search
```

## 7. Known limitations / TODOs

- **All 301 projects are skeletons** (SAXPY placeholder); none meets the full Definition of Done yet.
- **Toolchain ratified, not original:** the contract specified CUDA 12.x + VS 2022; the owner's machine has
  **CUDA 13.3 + VS 2026 (v145)**, which the owner approved as the standard. `CLAUDE.md` §5 + `BUILD_GUIDE.md`
  reflect this. Consequence: `sm_75` (Turing) is the floor (CUDA 13 dropped pre-Turing).
- **Design note (resolved):** `CLAUDE.md` §10's cardinal rule (agents never edit outside their own folder)
  slightly conflicts with "claim by editing a row in the shared `STATUS.md`." Resolved in favor of the
  cardinal rule: status lives in each project's own `.project_status.json`; `status.py` aggregates them into
  the generated `STATUS.md`. No worker touches the shared file.
- CI is not set up yet (Phase 3); hosted runners can compile but not run kernels.
